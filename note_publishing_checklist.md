# note記事公開チェックリスト

## 📋 投稿前の確認事項

### ✅ 記事の内容チェック

#### 1. タイトル
- [ ] 40文字以内で興味を引く内容か
- [ ] キーワード（ICT支援員、GIGAスクールなど）が含まれているか
- [ ] 具体的な数字や価値が入っているか（例：「全50回」「無料」）

#### 2. 冒頭文（リード）
- [ ] 最初の3行で読者の興味を引けるか
- [ ] 読者の悩みや課題に共感しているか
- [ ] 記事を読むメリットが明確か

#### 3. 本文
- [ ] 見出しで構造化されているか（H2、H3の活用）
- [ ] 段落が長すぎないか（3-4行で改行）
- [ ] 具体例やエピソードが入っているか
- [ ] 専門用語は分かりやすく説明しているか
- [ ] 絵文字やアイコンで読みやすくしているか

#### 4. 締めくくり
- [ ] 明確なCTA（行動喚起）があるか
- [ ] 講座URLが正しく記載されているか
- [ ] メールアドレスが正しいか（school.ict.info@gmail.com）
- [ ] 関連記事へのリンクがあるか

#### 5. ハッシュタグ
- [ ] 10個以上のハッシュタグを設定したか
- [ ] メインハッシュタグ（#ICT支援員 #GIGAスクール）が入っているか
- [ ] 関連性の高いタグを選んでいるか

## 📊 note記事の評価基準

### 良好な数値の目安（初期段階）

| 指標 | 目標値（公開後1週間） | 優秀な値 |
|------|---------------------|---------|
| PV数 | 500 PV | 1,000 PV以上 |
| スキ数 | 50スキ | 100スキ以上 |
| 読了率 | 30% | 50%以上 |
| コメント | 5件 | 10件以上 |

### 改善すべき場合の対策

#### PV数が低い
→ タイトルとSNS告知を見直す
→ ハッシュタグを追加・変更
→ 関連記事に内部リンクを貼る

#### スキ数が低い
→ 内容の価値が伝わっていない可能性
→ 具体例やエピソードを追加
→ CTAを強化

#### 読了率が低い
→ 冒頭で離脱されている
→ リード文を改善
→ 見出しで構造を明確に
→ 文章を短く、読みやすく

## 💡 記事作成のTips

### タイトルの作り方

✅ 良いタイトルの例
- 「GIGAスクール時代のICT支援員に必要なスキルとは？｜全50回の体系的教育プログラムを無料公開」
- 「ICT支援員の1日に密着｜朝8時から17時まで、何をしている？」
- 「ICT支援員1年目の大失敗｜あの時知っておきたかった5つのこと」

❌ 避けるべきタイトル
- 「ICT支援員について」（抽象的すぎる）
- 「教育ICTの話」（漠然としている）
- 「私の経験」（読者のメリットが不明）

### 記事構成のコツ

**黄金の構成**
1. **問題提起**：読者の悩みを言語化
2. **共感**：「分かります、私もそうでした」
3. **解決策**：具体的な方法を提示
4. **根拠**：なぜその方法が有効か
5. **行動喚起**：今すぐできることを提案

### 読みやすくする工夫

- ✅ 一文は60字以内
- ✅ 3-4行で改行
- ✅ 箇条書きを活用
- ✅ 絵文字で視覚的に
- ✅ 太字・マーカーで強調
- ✅ 引用・吹き出しで変化をつける
