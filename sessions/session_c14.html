中級編第14回（最終回）

実践的な課題解決演習

所要時間: 120分 | レベル: 中級

📘 学習目標

  • これまでの学習内容を統合し、実践的な課題に対応できる
  • 複雑な状況下での優先順位判断と問題解決能力を習得する
  • チームでの協働と情報共有の重要性を理解する
  • ICT支援員としての総合的な実践力を確認する

1. ケーススタディの進め方

1.1 課題解決の基本プロセス

🎯 PDCA サイクルでの課題解決

  1. Plan（計画）：
    • 現状分析：何が問題か
    • 目標設定：どうなればよいか
    • 対策立案：どのように解決するか
  2. Do（実行）：
    • 計画の実施
    • 記録の作成
  3. Check（評価）：
    • 結果の確認
    • 目標達成度の評価
  4. Act（改善）：
    • 成功要因の分析
    • 課題の抽出
    • 次への改善策

1.2 演習の評価ポイント

  • 問題の本質を見抜けているか
  • 優先順位の判断は適切か
  • 実現可能な解決策か
  • 関係者への配慮があるか
  • 再発防止策を考えているか

2. ケーススタディ1：新学期のトラブル続出

2.1 状況設定

💡 状況

時期：4月2日目（入学式翌日）、午前8時30分

あなたの役割：ICT支援員（勤務開始2年目）

学校：小学校（全18クラス、児童数600人）

発生した問題（同時多発）：

  1. 【緊急】 1年生の教室（3クラス）で、タブレットが全く起動しない（120台）
  2. 【重要】 5年生の担任から「Google Classroom にログインできない児童が10人いる」と連絡
  3. 【通常】 6年生の担任から「昨年度の写真データをUSBメモリで渡したい」と依頼
  4. 【情報】 校長から「来週の授業参観でICT活用授業を公開したい。準備を手伝ってほしい」
  5. 【確認】 教頭から「タブレットの保険更新手続きの書類を今日中に」と指示

2.2 課題

以下の問いに答えてください：

  1. 対応の優先順位をつけ、理由を説明してください
  2. 最優先の問題について、具体的な対応手順を示してください
  3. 同時並行で対応できること、後回しにできることを整理してください
  4. 情報担当教員や他のスタッフとどのように連携しますか

2.3 解答例と解説

🎯 優先順位の考え方

1位：1年生のタブレット起動問題（緊急度：高、影響範囲：大）

  • 理由：授業開始時刻が迫っており、3クラス全員に影響
  • 対応：
    1. 現場に急行、状況確認（充電されていない可能性）
    2. 充電カートの電源確認
    3. 数台を別の教室の充電カートに接続してテスト
    4. 原因判明まで代替案（デジタル教科書は後回し、紙のプリント使用）を教員に提案

2位：5年生のログイン問題（緊急度：中、影響範囲：中）

  • 1年生の問題が落ち着いたら対応
  • 休み時間に個別パスワードリセット

3位：保険更新書類（期限：今日中）

  • 昼休みまたは放課後に対応

4位：授業参観準備（期限：来週）

  • 放課後に打ち合わせ設定

5位：写真データの受け渡し（通常業務）

  • 授業のない時間帯に対応

3. ケーススタディ2：ICT活用授業の支援

3.1 状況設定

💡 状況

依頼内容：小学4年生の理科「電池のはたらき」でタブレットを使った実験記録を行いたい

教員：ICT活用経験が少ない、不安を感じている

授業予定：2週間後

使用ツール：タブレットのカメラ、Google スライド

3.2 課題

  1. 授業までの準備スケジュールを立ててください
  2. 授業当日の支援計画を作成してください
  3. 想定されるトラブルと対策を3つ挙げてください
  4. 授業後の振り返りで確認すべきポイントは何ですか

3.3 解答例

🎯 準備スケジュール（2週間）

1週間前

  • 教員と打ち合わせ（授業の流れ、使用機能の確認）
  • 児童向け操作手順書の作成
  • タブレットの動作確認（カメラ、Google スライド）
  • Google Classroom に課題を設定

3日前

  • 教員向けリハーサル（実際の操作を一緒に確認）
  • 想定Q&Aの準備

前日

  • タブレット充電確認
  • 教室のWi-Fi接続確認
  • 実験器具との配置確認

想定トラブルと対策

トラブル 対策
写真がぼやける 事前に撮影のコツを指導（明るい場所、手ブレ注意）
スライドに写真を挿入できない デモ動画を用意、巡回サポート
Wi-Fiが遅い 写真は一時的にカメラロールに保存、後でアップロード

4. ケーススタディ3：予算要望と環境改善

4.1 状況設定

💡 状況

現状の課題：

  • タブレット導入から6年経過、動作が遅い、バッテリー劣化
  • 3階の教室でWi-Fiが不安定
  • プロジェクターのランプ寿命が近い（5台）
  • デジタルドリル教材の契約更新時期

次年度予算要望の時期が来た

4.2 課題

  1. 優先順位をつけて予算要望リストを作成してください
  2. 最優先項目について、予算要望書（概要）を作成してください
  3. 予算が削減された場合の代替案を考えてください

4.3 解答例

🎯 優先順位と理由

1位：タブレット更新（最重要）

  • 理由：全校での使用、教育活動の根幹
  • 概算：600台 × 5万円 = 3,000万円

2位：Wi-Fi環境改善（重要）

  • 理由：授業の円滑な実施に不可欠
  • 概算：アクセスポイント追加 5台 × 10万円 = 50万円

3位：プロジェクターランプ交換（必要）

  • 理由：消耗品、計画的交換
  • 概算：5台 × 3万円 = 15万円

4位：デジタルドリル更新（検討）

  • 理由：効果を見ながら継続判断
  • 概算：年間 200万円

予算削減時の代替案

  • タブレット：全台更新できない場合は学年ごとに段階更新
  • Wi-Fi：混雑時間帯の使用制限、授業時間の分散
  • デジタルドリル：無料教材（eboard等）への切り替え検討

5. グループワーク演習

5.1 ロールプレイ

シナリオ：教員研修の実施

  • 役割1：ICT支援員（研修講師）
  • 役割2：ベテラン教員（ICT苦手）
  • 役割3：若手教員（ICT得意）
  • 役割4：情報担当教員（進行管理）

課題：「Google Classroom 基礎研修」（30分）を実施してください

6.3 継続的な成長のために

  • 実践：学んだことを現場で試す
  • 振り返り：うまくいったこと、改善点を記録
  • 情報収集：最新技術、教育動向をキャッチアップ
  • ネットワーク：他の支援員、教員との交流
  • 自己研鑽：資格取得、専門書の読書

📝 最終課題

レポート課題：ICT支援員としての成長計画

以下の項目について、A4用紙2枚程度（1,200字程度）でまとめてください。

  1. 現在の自己評価：中級編で学んだ内容のうち、得意な領域と苦手な領域
  2. 今後1年間の目標：どのようなスキルを身につけたいか（具体的に3つ）
  3. 実践計画：目標達成のために何をするか
  4. 理想のICT支援員像：あなたが目指すICT支援員とは

📌 まとめ

  • 実践的な課題解決には優先順位判断、計画的対応、柔軟な思考が必要
  • 複数の問題が同時発生した際は、緊急度と影響範囲で優先順位を決定する
  • 授業支援は事前準備、当日サポート、事後振り返りの3段階で行う
  • 予算要望は現状課題、導入目的、効果を明確にし、代替案も用意する
  • 中級編修了後も継続的な学習と実践で、プロフェッショナルとして成長し続ける
