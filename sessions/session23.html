トラブルシューティングの実践

所要時間: 90分 | レベル: 基礎

📘 学習目標

本講座を修了することで、以下のスキルを習得できます：

  • 学校現場で頻発するICTトラブルに迅速かつ適切に対応できる
  • 問題の原因を論理的に切り分け、効率的に解決に導くことができる
  • トラブル対応の記録を体系的に管理し、組織的な知識として共有できる
  • 予防保全の観点から、トラブルを未然に防ぐ仕組みを構築できる

2. よくあるトラブルと対処法

2.1 起動・ログインの問題

学校現場で最も頻繁に発生するトラブルの一つです。授業開始時に発生すると大きな支障となるため、迅速な対応が求められます。

主な症状と対処法

電源が入らない場合
  • バッテリー残量の確認（充電ランプの状態を確認）
  • ACアダプターの接続確認（コネクタの抜き差し）
  • 電源ボタンの長押し（10秒以上）による強制再起動
  • バッテリーの取り外し・再装着（可能な機種の場合）
起動途中で止まる場合
  • セーフモードでの起動を試行
  • 周辺機器（USBメモリ、外付けHDDなど）を全て外して起動
  • BIOSの起動順位の確認
  • システム復元ポイントへの復帰
ログインできない場合
  • CapsLockの状態確認（パスワード入力時の大文字・小文字）
  • NumLockの状態確認（テンキーでの数字入力）
  • アカウント名の入力ミス確認（全角・半角、スペースの有無）
  • パスワードリセットツールの利用
  • 管理者アカウントでのログイン試行

実践的対応フロー

  1. 症状の正確な聞き取り（「いつから」「何をしたら」「どうなった」）
  2. 最も可能性の高い原因から順に確認
  3. 各手順の結果を記録
  4. 解決後、再発防止のための指導

2.2 ネットワーク接続トラブル

Wi-Fi接続やインターネットアクセスの問題は、オンライン学習の普及により特に重要度が増しています。

症状別対処法

症状 考えられる原因 対処方法
Wi-Fiアイコンに×印 Wi-Fi機能がOFF 機内モード解除、Wi-Fi ON確認
SSIDが表示されない 電波が届いていない APに近づく、障害物確認、AP再起動
接続はできるがインターネットに繋がらない DHCP、DNS、ゲートウェイの問題 「ネットワーク診断」実行、IPアドレス確認、DNS設定確認
接続が頻繁に切れる 電波干渉、AP負荷過多 チャンネル変更、AP増設検討、接続端末数制限
速度が遅い 帯域不足、同時接続多数 時間帯をずらす、不要なアプリ終了、回線増強検討

チェックリスト

個別端末の問題か、全体の問題かの切り分け
  • 1台だけ：端末側の設定問題の可能性大
  • 複数台/全台：ネットワーク機器やサーバー側の問題の可能性大
基本確認項目
  1. Wi-FiがONになっているか
  2. 正しいSSIDに接続しているか
  3. パスワードは正しいか
  4. 他の端末は繋がっているか
  5. ブラウザのキャッシュクリア
  6. 端末の再起動

2.3 アプリケーションの問題

起動しない・フリーズする

対処手順
  1. タスクの強制終了：タスクマネージャー（Ctrl+Shift+Esc）で該当プロセスを終了
  2. アプリの再起動：完全に終了してから再度起動
  3. 端末の再起動：メモリリフレッシュ
  4. アップデート確認：最新版へのアップデート
  5. 再インストール：アンインストール後、再インストール
  6. 互換性モード：古いアプリの場合、互換性設定を変更

データが保存できない・開けない

考えられる原因と対処
  • 保存先の容量不足：ストレージ容量確認、不要ファイル削除
  • ファイル名の問題：使用禁止文字（/\:*?"<>|）確認
  • 権限の問題：管理者権限で実行、保存先変更
  • ファイル破損：バックアップからの復元、修復ツール使用
  • 互換性の問題：ファイル形式の変換、別アプリで開く

2.4 プリンター・印刷トラブル

症状 原因 対処法
プリンターが見つからない 接続問題、ドライバー問題 ケーブル確認、プリンター電源ON、ドライバー再インストール
印刷ジョブが詰まっている エラー発生、処理待ち 印刷キューのクリア、プリンタースプーラー再起動
印刷できるが文字化けする フォント問題、ドライバー不具合 PDFで印刷、ドライバー更新、フォント埋め込み
用紙が詰まる 用紙設定ミス、湿気 用紙トレイ確認、用紙種類設定、湿気対策
色がおかしい インク不足、ヘッド詰まり インク残量確認、ヘッドクリーニング、ノズルチェック

2.5 タブレット・iPad特有のトラブル

タッチ操作が効かない

  • 画面の清掃（指紋、汚れ）
  • 保護フィルムの確認（気泡、浮き）
  • 再起動（強制再起動も試す）
  • タッチ調整（設定→アクセシビリティ）
  • ケースが干渉していないか確認

アプリがダウンロードできない

  • Apple ID / Google アカウントのサインイン状態確認
  • ストレージ容量確認
  • ネットワーク接続確認
  • App Store / Google Play のキャッシュクリア
  • MDM（モバイルデバイス管理）の制限確認

バッテリーの減りが早い

  • バックグラウンドアプリの終了
  • 画面の明るさ調整
  • 位置情報サービスの見直し
  • 自動更新の設定見直し
  • バッテリーの劣化確認（設定→バッテリー）

2.6 プロジェクター・電子黒板トラブル

症状 対処法
映像が映らない 入力切替確認、ケーブル接続確認、ランプ切れ確認、PC側の出力設定
色がおかしい ケーブル接触不良、色温度設定、RGB調整
画面がぼやける フォーカス調整、解像度設定確認、レンズ清掃
音が出ない 音量確認、ミュート解除、オーディオケーブル確認、出力デバイス選択
リモコンが効かない 電池交換、受光部の障害物除去、リモコンリセット
電子黒板のタッチが反応しない キャリブレーション実行、専用ペンの電池確認、センサー清掃

3. トラブルシューティングの基本プロセス

3.1 問題の切り分け手法

効率的にトラブルを解決するには、論理的な切り分けが重要です。

切り分けの基本軸

① ハードウェア vs ソフトウェア
  • ハードウェア：物理的な故障（画面割れ、水濡れ、コネクタ破損等）
  • ソフトウェア：OS、アプリ、設定の問題
  • 切り分け方法：別の端末で同じ操作をしてみる
② 個別 vs 全体
  • 個別：特定の端末・ユーザーだけの問題
  • 全体：複数台・全体で発生している問題
  • 切り分け方法：他の端末・ユーザーで試す
③ 端末側 vs サーバー・ネットワーク側
  • 端末側：端末設定、アプリの問題
  • サーバー・ネットワーク側：インフラの問題
  • 切り分け方法：オフラインで動作するか、他のサービスは使えるか

3.2 トラブル対応の7ステップ

ステップ1: 状況把握

  • いつから、どの端末で、誰が使っている時に発生したか
  • 何をしようとして、どうなったか
  • エラーメッセージの内容
  • 再現性はあるか（毎回起きるか、時々か）

ステップ2: 仮説立案

  • 症状から考えられる原因を列挙
  • 可能性の高い順に優先順位付け
  • 過去の類似事例を参照

ステップ3: 検証

  • 仮説に基づいて確認・テスト
  • 一度に複数の変更をしない（原因特定のため）
  • 各手順の結果を記録

ステップ4: 対処実施

  • 原因が特定できたら、適切な対処を実施
  • 変更前の状態をメモ（元に戻せるように）
  • ユーザーに作業内容を説明

ステップ5: 動作確認

  • 問題が解決したか確認
  • 他の機能に影響が出ていないか確認
  • ユーザーに正常動作を確認してもらう

ステップ6: 記録

  • トラブルの内容、原因、対処方法を記録
  • 対応時間、担当者を記録
  • ナレッジベースに登録

ステップ7: 予防策の検討

  • 同じトラブルが再発しないための対策
  • 横展開（他の端末でも同様の設定が必要か）
  • 教員・児童生徒への周知事項

3.3 エスカレーションの判断

自分で解決できない場合は、適切にエスカレーション（上位者・専門家への依頼）することが重要です。

⚠ エスカレーションすべき状況

  • 30分試しても解決しない
  • ハードウェアの物理的故障が疑われる
  • サーバー・ネットワーク機器の問題
  • 複数台・全校規模で発生している
  • セキュリティに関わる問題
  • データ消失のリスクがある
  • 自分の権限では対応できない

✓ エスカレーション時に伝える情報

  • 発生日時、場所、端末・ユーザー情報
  • 症状の詳細（エラーメッセージ、スクリーンショット）
  • これまで試した対処とその結果
  • 影響範囲（何台、どのクラス等）
  • 緊急度（授業への影響度）

エスカレーション先

問題の種類 エスカレーション先
端末の故障 保守業者、メーカーサポート
ネットワーク障害 ネットワーク管理者、業者
校務支援システム システム管理者、ベンダー
アカウント・権限 情報管理担当教員、教育委員会
セキュリティ問題 情報セキュリティ担当、教育委員会

4. トラブル対応の記録と共有

4.1 トラブルチケットの管理

トラブル対応を体系的に管理するために、トラブルチケット（記録票）を活用します。

記録すべき項目

項目 内容
チケット番号 一意の管理番号
発生日時 いつ発生したか
報告者 誰から報告があったか
発生場所 どこで発生したか（教室、PC室等）
端末情報 機種、シリアル番号、管理番号
症状 何が起きたか（詳細に記述）
原因 特定できた原因
対処内容 実施した対処（手順も記載）
対応者 誰が対応したか
対応時間 対応開始～完了までの時間
ステータス 未対応/対応中/完了/保留
備考 その他特記事項

4.2 ナレッジベースの構築

トラブル事例を蓄積し、検索可能な形で共有することで、組織全体の対応力が向上します。

ナレッジベースの構成例

カテゴリ分類
  • ハードウェア（PC、タブレット、プリンター等）
  • ソフトウェア（OS、アプリ別）
  • ネットワーク
  • アカウント・認証
  • 周辺機器
記事のフォーマット
  • タイトル：「○○ができない」など、症状を端的に
  • 症状：詳細な症状の説明
  • 原因：なぜ起きるか
  • 対処法：ステップバイステップで解決手順
  • 予防策：再発防止の方法
  • 関連記事：類似トラブルへのリンク

ナレッジベースの運用

  • 定期的な更新：新しいトラブル事例を追加
  • レビュー：古い情報は更新・削除
  • 検索性：タグ付け、全文検索可能に
  • アクセス権限：教員も閲覧できるようにする
  • フィードバック：「役に立った」ボタン等で評価

4.3 FAQ（よくある質問）の作成

頻出する質問・トラブルをFAQとしてまとめ、教員が自己解決できるようにします。

FAQ例

Q1. Wi-Fiに繋がらなくなりました

A. 以下の手順をお試しください：

  1. 機内モードがONになっていないか確認
  2. Wi-FiをOFF→ONにしてみる
  3. 端末を再起動する
  4. それでも繋がらない場合は ICT支援員にご連絡ください
Q2. プロジェクターに映像が映りません

A. 以下を確認してください：

  1. プロジェクターの電源が入っているか
  2. 正しい入力（HDMI1等）が選択されているか
  3. ケーブルがしっかり接続されているか
  4. PC側で「画面複製」または「拡張」が選択されているか（Windowsキー+P）
Q3. パスワードを忘れました

A. 情報管理担当の○○先生、またはICT支援員にご連絡ください。本人確認後、リセットいたします。

5. 予防保全とトラブル予防

5.1 定期メンテナンスの重要性

トラブルが起きてから対処するより、未然に防ぐことが重要です。

日常点検（毎日）

  • 充電カートの充電状態確認
  • ネットワーク機器のランプ状態確認
  • プロジェクターの動作確認
  • 教室環境チェック（破損、異常音等）

週次点検

  • 端末のOS・アプリ更新状況確認
  • ウイルス定義ファイル更新確認
  • ストレージ容量確認
  • バックアップ実行状況確認

月次点検

  • 端末の清掃（キーボード、画面等）
  • プリンターのメンテナンス（ヘッドクリーニング等）
  • ケーブル類の断線チェック
  • バッテリーの劣化状況確認

学期毎・年次点検

  • 全端末の棚卸し（紛失・破損確認）
  • 不要ファイルの削除、初期化
  • ハードウェアの動作確認
  • 保守契約の更新確認

5.2 トラブル予防の啓発活動

教員向け

  • 使い方講習：正しい操作方法の周知
  • 注意喚起：よくある誤操作の事例共有
  • マニュアル配布：基本操作のクイックガイド
  • Q&A掲示：よくある質問を掲示板に

児童生徒向け

  • 使用ルールの徹底：持ち方、扱い方の指導
  • ログイン・ログアウトの習慣化：セキュリティ意識
  • トラブル時の報告方法：すぐ教員に報告
  • デジタルシティズンシップ教育：責任ある使用

5.3 環境整備

物理的環境

  • 適切な温湿度管理（機器の寿命延長）
  • ほこり対策（定期清掃）
  • 配線整理（断線・転倒防止）
  • 盗難・破損対策（保管場所の施錠）

ソフトウェア環境

  • OS・アプリの計画的アップデート
  • セキュリティパッチの適用
  • 不要ソフトのアンインストール
  • 定期的なウイルススキャン

運用ルール

  • 端末持ち出しルールの明確化
  • 故障時の報告フローの整備
  • 予備機の確保と管理
  • 定期的な利用状況確認

6. 緊急時の対応

6.1 授業中のトラブル対応

授業中のトラブルは、児童生徒の学びに直接影響するため、迅速な対応が求められます。

⚠ 授業中対応の原則

  • 授業を止めない：すぐ解決できない場合は代替手段を提案
  • 静かに対応：他の児童生徒の学習を妨げない
  • 教員の指示を尊重：教員の授業進行を最優先
  • 記録：対応後に詳細を記録

代替手段の提案例

トラブル 代替手段
1台の端末が起動しない 予備機と交換、ペア学習に変更
プロジェクターが映らない 電子黒板使用、印刷資料配布、口頭説明
ネットワークが繋がらない オフラインで作業、テザリング、後日オンライン作業
アプリが起動しない 別のアプリで代用、紙の教材使用

6.2 大規模障害への対応

全校的なネットワーク障害など、影響範囲が大きいトラブルの場合

対応手順

  1. 即座に管理職・情報管理担当に報告
  2. 影響範囲の特定：どの教室、何台に影響しているか
  3. 全体アナウンス：校内放送・メール等で周知
  4. 復旧作業：業者・専門家と連携
  5. 進捗共有：定期的に状況を更新
  6. 復旧確認：全端末の動作確認
  7. 原因究明と再発防止：事後報告書作成

6.3 緊急連絡先リストの整備

整備すべき連絡先

  • 校内：管理職、情報管理担当教員
  • 教育委員会：情報教育担当
  • 保守業者：端末、ネットワーク、サーバー
  • メーカーサポート
  • インターネットプロバイダ
  • 校務支援システムベンダー

連絡先は常に最新の状態に更新し、すぐアクセスできる場所に保管しましょう。

📝 演習問題

演習1：トラブル診断

以下の症状から、考えられる原因と対処法を答えてください。

ケース1：

「タブレットの電源ボタンを押しても何も反応しません。昨日まで普通に使えていました。」

ケース2：

「PC教室の全てのパソコンでインターネットに繋がりません。他の教室は正常です。」

ケース3：

「プリンターから印刷できません。他の先生は印刷できているようです。」

演習2：対応フロー作成

「プロジェクターに映像が映らない」というトラブルに対する対応フローチャートを作成してください。

  • 確認すべき項目
  • 対処手順
  • エスカレーションの判断基準

演習3：トラブルチケット記入

以下の状況について、トラブルチケットを記入してください。

状況：
2024年10月15日 10:30、3年2組の授業中に、児童A君のタブレット（管理番号：T-302-05）が突然フリーズしました。何を操作しても反応がなく、電源ボタンを長押しして強制再起動したところ、正常に起動しました。その後は問題なく使用できています。

  • よくあるトラブル：起動・ログイン、ネットワーク、アプリ、プリンター、タブレット、プロジェクター
  • 切り分け手法：ハードvsソフト、個別vs全体、端末vsサーバー
  • 対応プロセス：7ステップ（状況把握→仮説→検証→対処→確認→記録→予防）
  • エスカレーション：判断基準、伝えるべき情報
  • 記録と共有：トラブルチケット、ナレッジベース、FAQ
  • 予防保全：定期メンテナンス、啓発活動、環境整備
  • 緊急時対応：授業中の対応原則、大規模障害対応

重要ポイント

  1. 冷静に、論理的に：焦らず、順序立てて原因を切り分ける
  2. 記録の徹底：トラブル対応は必ず記録し、組織の知識に
  3. 予防が最善：トラブルが起きる前に防ぐ
  4. 適切なエスカレーション：自分で抱え込まず、必要なら専門家に
  5. ユーザー視点：教員・児童生徒の立場で考え、寄り添う
  6. 継続的改善：トラブルから学び、システムを改善

次のステップ

  • 自分の担当校でよく起きるトラブルをリストアップする
  • トラブルチケットのフォーマットを作成する
  • FAQ集を作り始める
  • 緊急連絡先リストを整備する
  • 定期メンテナンスのチェックリストを作成する
  • トラブル対応の実践を積み、経験値を高める
  • 技術系コミュニティで情報交換する
