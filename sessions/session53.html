第2章：システムコールとAPI

2.1 システムコール（System Call）とは

システムコールは、アプリケーションプログラムがオペレーティングシステムのカーネルに対して、ハードウェアリソースの利用やシステム機能を要求するための直接的なインターフェースです。

システムコールの例

  • ファイル操作: open(), read(), write(), close()
  • プロセス管理: fork(), exec(), wait(), kill()
  • メモリ管理: mmap(), brk()
  • ネットワーク通信: socket(), connect(), send(), recv()

システムコールはカーネルモードで実行され、アプリケーションのユーザーモードから特権的な操作を安全に行うための仕組みです。

2.2 API（Application Programming Interface）とは

APIは、システムコールを含む、より高レベルで抽象化されたインターフェースです。開発者は通常、APIを通じてシステム機能にアクセスします。

APIとシステムコールの関係

  • システムコール：OSカーネルへの直接的な要求（低レベル、OS固有）
  • API：システムコールをラップした抽象化レイヤー（高レベル、移植性が高い）
  • 例：Python の open() 関数 → C言語ライブラリの fopen() → システムコール open()

主なAPI例

  • UNIX/Linux: POSIX API（システムコール群を標準化）
  • Windows: Windows API（WindowsAPI、システムサービスへのアクセス）
  • Python: 標準ライブラリ（os, sys, pathlib, subprocess等）

2.3 本講座での実践的な理解

本講座では、システムコールやAPIを直接プログラミングすることはありません。代わりに、以下を学びます：

  • シェルスクリプト：コマンドライン経由でOSの機能（ファイル操作、プロセス管理）を利用
  • Python標準ライブラリ：os, pathlib, subprocess 等を用いてAPI経由でシステム機能を活用

3.2 実行ポリシー（PowerShell）

PowerShellでは、悪意あるスクリプトの実行を防ぐため、実行ポリシー（Execution Policy）が設定されています。

3.2.1 実行ポリシーの種類

ポリシー 説明 推奨度（教育現場）
Restricted 全てのスクリプト実行を禁止（デフォルト） 初期状態
AllSigned 信頼された発行元の署名済みスクリプトのみ実行可 高セキュリティ環境
RemoteSigned ローカルスクリプトは実行可、ダウンロードスクリプトは署名必須 ✅ 推奨
Unrestricted 全てのスクリプト実行可（警告あり） ⚠️ 非推奨
Bypass 全てのスクリプト実行可（警告なし） ⚠️ 危険

3.2.2 実行ポリシーの確認と設定

# 現在の実行ポリシーを確認
Get-ExecutionPolicy

# 実行ポリシーを RemoteSigned に設定(管理者権限が必要)
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

# 一時的にポリシーを回避してスクリプト実行(⚠️非推奨)
PowerShell -ExecutionPolicy Bypass -File script.ps1

⚠️ セキュリティ上の注意

  • 実行ポリシーは誤操作防止が目的：悪意あるスクリプトに対する完全な防御機能ではありません
  • RemoteSigned推奨：教育現場では、ローカル開発のスクリプトは実行可能だが、外部スクリプトは検証が必要な RemoteSigned が最適
  • Unrestricted/Bypass は避ける：セキュリティリスクが高く、推奨されません

3.4 実践例：ファイルバックアップスクリプト

3.4.1 Bash スクリプト例

#!/bin/bash
# ファイルバックアップスクリプト
# 使い方: ./backup.sh [元ファイルのパス] [バックアップ先ディレクトリ]

SOURCE=$1
DEST_DIR=$2
TIMESTAMP=$(date +"%Y%m%d_%H%M%S")
BASENAME=$(basename "$SOURCE")
DEST_FILE="${DEST_DIR}/${BASENAME}_${TIMESTAMP}.bak"

# 引数チェック
if [ -z "$SOURCE" ] || [ -z "$DEST_DIR" ]; then
    echo "使い方: $0 [元ファイル] [バックアップ先ディレクトリ]"
    exit 1
fi

# 元ファイルの存在確認
if [ ! -f "$SOURCE" ]; then
    echo "エラー: $SOURCE が見つかりません"
    exit 1
fi

# バックアップ先ディレクトリの作成(存在しなければ)
mkdir -p "$DEST_DIR"

# バックアップ実行
cp "$SOURCE" "$DEST_FILE"

if [ $? -eq 0 ]; then
    echo "✅ バックアップ成功: $DEST_FILE"
else
    echo "❌ バックアップ失敗"
    exit 1
fi

3.4.2 PowerShell スクリプト例

# ファイルバックアップスクリプト
# 使い方: .\backup.ps1 -Source "C:\data.csv" -DestDir "C:\backup"

param(
    [Parameter(Mandatory=$true)]
    [string]$Source,
    [Parameter(Mandatory=$true)]
    [string]$DestDir
)

# タイムスタンプ取得
$Timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
$BaseName = [System.IO.Path]::GetFileName($Source)
$DestFile = Join-Path $DestDir "${BaseName}_${Timestamp}.bak"

# 元ファイルの存在確認
if (-not (Test-Path $Source)) {
    Write-Error "エラー: $Source が見つかりません"
    exit 1
}

# バックアップ先ディレクトリの作成(存在しなければ)
if (-not (Test-Path $DestDir)) {
    New-Item -ItemType Directory -Path $DestDir | Out-Null
}

# バックアップ実行
try {
    Copy-Item -Path $Source -Destination $DestFile -ErrorAction Stop
    Write-Output "✅ バックアップ成功: $DestFile"
} catch {
    Write-Error "❌ バックアップ失敗: $_"
    exit 1
}

5.1 名簿データの整形

複数のCSVファイルから名簿を統合し、重複を除外する実践例です。

import csv
from collections import defaultdict

def merge_csv_files(file_list, output_file):
    """複数のCSVファイルを統合し、重複を除外"""
    students = {}  # {学籍番号: {名前, 学年, ...}}
    
    for file_path in file_list:
        with open(file_path, 'r', encoding='utf-8') as f:
            reader = csv.DictReader(f)
            for row in reader:
                student_id = row['学籍番号']
                students[student_id] = row  # 重複は上書き
    
    # 統合結果を出力
    with open(output_file, 'w', encoding='utf-8', newline='') as f:
        if students:
            fieldnames = list(students[next(iter(students))].keys())
            writer = csv.DictWriter(f, fieldnames=fieldnames)
            writer.writeheader()
            writer.writerows(students.values())
    
    print(f"✅ 統合完了: {len(students)}件のデータを出力")

# 実行例
files = ['class_1a.csv', 'class_1b.csv', 'class_1c.csv']
merge_csv_files(files, 'all_students.csv')

        reader = csv.DictReader(f)
        students = list(reader)
    
    # 点数を数値リストに変換
    scores = [int(s['点数']) for s in students]
    
    # 統計計算
    avg = statistics.mean(scores)
    median = statistics.median(scores)
    max_score = max(scores)
    min_score = min(scores)
    std_dev = statistics.stdev(scores)
    passed = sum(1 for score in scores if score >= passing_score)
    pass_rate = (passed / len(scores)) * 100
    
    # 結果出力
    print("=" * 40)
    print(f"受験者数: {len(scores)}名")
    print(f"平均点: {avg:.1f}点")
    print(f"中央値: {median:.1f}点")
    print(f"最高点: {max_score}点")
    print(f"最低点: {min_score}点")
    print(f"標準偏差: {std_dev:.2f}")
    print(f"合格者数: {passed}名（合格率: {pass_rate:.1f}%）")
    print("=" * 40)
    
    # 成績分布
    ranges = [(90, 100, 'A'), (80, 89, 'B'), (70, 79, 'C'), 
              (60, 69, 'D'), (0, 59, 'F')]
    
    print("\n成績分布:")
    for min_s, max_s, grade in ranges:
        count = sum(1 for s in scores if min_s <= s <= max_s)
        percentage = (count / len(scores)) * 100
        print(f"{grade} ({min_s}-{max_s}点): {count}名 ({percentage:.1f}%)")

# 実行例
analyze_scores('exam_results.csv', passing_score=60)

5.3 ファイルの一括リネーム

複数のファイルを一括で整理・リネームするスクリプトです。

import os
from pathlib import Path

def batch_rename(directory, prefix, extension):
    """指定ディレクトリ内のファイルを連番でリネーム"""
    
    # 指定拡張子のファイルを取得
    path = Path(directory)
    files = sorted(path.glob(f'*.{extension}'))
    
    if not files:
        print(f"⚠️ {extension}ファイルが見つかりません")
        return
    
    # リネーム実行
    for index, file in enumerate(files, start=1):
        new_name = f"{prefix}_{index:03d}.{extension}"
        new_path = file.parent / new_name
        
        # 既存ファイルと重複しないか確認
        if new_path.exists() and new_path != file:
            print(f"⚠️ スキップ: {new_name} (既に存在)")
            continue
        
        file.rename(new_path)
        print(f"✅ {file.name} → {new_name}")
    
    print(f"\n✅ リネーム完了: {len(files)}ファイル")

# 実行例
# カレントディレクトリの全てのpngファイルを "photo_001.png", "photo_002.png", ... にリネーム
batch_rename('.', 'photo', 'png')

5.4 自動バックアップシステム

重要なファイルを定期的に自動バックアップするスクリプトです。

import shutil
import os
from datetime import datetime
from pathlib import Path

def auto_backup(source_dir, backup_dir, max_backups=7):
    """ディレクトリを自動バックアップ（世代管理機能付き）"""
    
    source = Path(source_dir)
    backup = Path(backup_dir)
    
    # 元ディレクトリの存在確認
    if not source.exists():
        print(f"❌ エラー: {source_dir} が見つかりません")
        return
    
    # バックアップ先ディレクトリ作成
    backup.mkdir(parents=True, exist_ok=True)
    
    # タイムスタンプ付きバックアップ名
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    backup_name = f"backup_{source.name}_{timestamp}"
    backup_path = backup / backup_name
    
    # バックアップ実行
    try:
        shutil.copytree(source, backup_path)
        print(f"✅ バックアップ成功: {backup_path}")
    except Exception as e:
        print(f"❌ バックアップ失敗: {e}")
        return
    
    # 古いバックアップを削除(世代管理)
    backups = sorted(backup.glob(f"backup_{source.name}_*"))
    if len(backups) > max_backups:
        for old_backup in backups[:-max_backups]:
            shutil.rmtree(old_backup)
            print(f"🗑️ 古いバックアップを削除: {old_backup.name}")
    
    print(f"✅ バックアップ完了（保持世代数: {min(len(backups), max_backups)}）")

# 実行例
auto_backup('C:/SchoolData/Documents', 'C:/Backup', max_backups=7)

  • Linux/macOS: cron で毎日実行
  • 推奨: 始業前（例: 午前8時）に自動実行

第6章：セキュリティの考慮

6.1 スクリプト実行時のリスク

自動化スクリプトは強力なツールですが、不適切に使用するとセキュリティリスクやデータ損失のリスクが伴います。

⚠️ 主なリスク

  • ファイルの誤削除：スクリプトのバグでデータを失う可能性
  • 権限の不適切な設定：誰でもアクセス可能な状態になる危険
  • 外部スクリプトの実行：信頼できないソースからのスクリプト実行
  • 個人情報の漏洩：ログファイルに機密情報が残る可能性
  • コマンドインジェクション：外部入力を適切に検証しないことによる攻撃

6.2 安全なスクリプトの作成

6.2.1 入力検証の実施

import os
from pathlib import Path

def safe_delete_file(file_path):
    """安全にファイルを削除（検証機能付き）"""
    
    path = Path(file_path)
    
    # 存在確認
    if not path.exists():
        print(f"❌ エラー: ファイルが見つかりません")
        return False
    
    # ファイルタイプ確認（ディレクトリは削除しない）
    if not path.is_file():
        print(f"❌ エラー: ディレクトリは削除できません")
        return False
    
    # 重要なファイルでないか確認（例: システムファイル、設定ファイル）
    dangerous_extensions = ['.sys', '.dll', '.exe', '.bat', '.ps1']
    if path.suffix.lower() in dangerous_extensions:
        print(f"⚠️ 警告: 重要なファイルの可能性があります")
        response = input("本当に削除しますか? (yes/no): ")
        if response.lower() != 'yes':
            print("キャンセルしました")
            return False
    
    # 削除実行
    try:
        path.unlink()
        print(f"✅ 削除成功: {file_path}")
        return True
    except Exception as e:
        print(f"❌ 削除失敗: {e}")
        return False

# 使用例
safe_delete_file('temp_data.txt')

6.4 個人情報保護の実践

教育現場では、児童生徒の個人情報を扱うため、特に慎重な対応が必要です。

⚠️ 個人情報保護のポイント

  1. 不要な個人情報は扱わない：必要最小限のデータのみ処理
  2. ログに個人情報を記録しない：名前、学籍番号などを出力しない
  3. 一時ファイルの削除：処理後は確実に削除する
  4. 暗号化の検討：機密性の高いデータは暗号化して保存
  5. アクセス記録の保持：誰が、いつ、何をしたかを記録

6.4.1 安全なデータ処理の例

import csv
import logging
from pathlib import Path

def process_sensitive_data(input_file, output_file):
    """個人情報を安全に処理（匿名化と仮名化）"""
    
    # ログには個人情報を記録しない
    logging.info(f"処理開始（ファイル数: 1件）")
    
    try:
        with open(input_file, 'r', encoding='utf-8') as f:
            reader = csv.DictReader(f)
            students = list(reader)
        
        # データ処理（個人情報を仮名化）
        processed = []
        for i, student in enumerate(students, start=1):
            processed.append({
                '生徒ID': f'S{i:04d}',  # 匿名ID
                '学年': student['学年'],
                '点数': student['点数']
                # 名前は含めない
            })
        
        # 結果を出力
        with open(output_file, 'w', encoding='utf-8', newline='') as f:
            writer = csv.DictWriter(f, fieldnames=['生徒ID', '学年', '点数'])
            writer.writeheader()
            writer.writerows(processed)
        
        logging.info(f"処理完了（処理件数: {len(processed)}件）")
        
    except Exception as e:
        logging.error(f"処理エラー（詳細は記録しない）")
        raise

# 使用例
process_sensitive_data('students_raw.csv', 'students_anonymized.csv')
