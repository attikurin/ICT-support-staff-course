中級編第9回

アカウント管理とアクセス制御

所要時間: 90分 | レベル: 中級

📘 学習目標

  • 学校におけるアカウント管理の重要性と方法を理解する
  • Google Workspace / Microsoft 365のアカウント管理機能を説明できる
  • 適切なパスワード管理とセキュリティ対策を習得する
  • MDM（モバイルデバイス管理）の役割と機能を理解する

1. アカウント管理の基本

1.1 アカウント管理の重要性

💡 なぜアカウント管理が重要か

  • 個人情報保護：児童生徒・教員の情報を守る
  • 不正アクセス防止：なりすまし、情報漏洩の防止
  • 適切な権限管理：必要な人に必要な権限のみ付与
  • 利用状況の把握：誰がいつ何をしたか記録
  • コンプライアンス：法令・ガイドラインの遵守

1.2 学校で管理するアカウントの種類

種類 対象 用途
児童生徒アカウント 全児童生徒 学習用端末ログイン、Google / Microsoft 365
教員アカウント 全教職員 校務用PC、校務支援システム、学習系システム
管理者アカウント 情報担当教員、ICT支援員 アカウント作成・削除、設定変更
デジタル教材アカウント 利用者 デジタルドリル、学習アプリ

2. Google Workspace for Education

2.1 組織構造（OU：組織単位）

🎯 組織単位（OU）の設計例

階層 組織単位 説明
○○市立○○小学校 教職員 教職員全体のグループ
└ 管理職 校長、副校長、教頭
└ 教員 担任、専科教員
└ 事務職員 事務、用務員等
児童生徒 児童生徒全体のグループ
└ 1年生～6年生 学年ごとに分類

OUごとに異なる設定が可能：

  • 利用可能なアプリの制限
  • 共有設定（外部共有の可否）
  • YouTubeの制限モード

2.2 主要な管理機能

  • ユーザー管理
    • アカウントの一括作成（CSVインポート）
    • パスワードのリセット
    • アカウントの停止・削除
  • グループ管理
    • クラスごと、学年ごとのグループ作成
    • メーリングリスト機能
    • グループでの共有設定
  • アプリの制御
    • Gmail、Google ドライブ、YouTube等の利用制限
    • Marketplace アプリの許可・ブロック
  • セキュリティ設定
    • 2段階認証の強制
    • パスワードポリシー（長さ、複雑さ）
    • ログイン試行回数制限

2.3 Google Classroom との連携

  • クラスへの児童生徒の自動追加
  • 教員権限の管理
  • クラスデータのアーカイブ

3. Microsoft 365 Education

3.1 管理センター

主要な管理画面

  • Microsoft 365 管理センター：ユーザー・ライセンス管理
  • Azure Active Directory：認証・セキュリティ管理
  • Intune 管理センター：デバイス管理（MDM）
  • Teams 管理センター：Teams の設定

3.2 ライセンス管理

ライセンス 対象 主な機能
A1（無料） 学生・教員 Web版Office、Teams、OneDrive（100GB以上）
A3（有料） 学生・教員 デスクトップ版Office、高度なセキュリティ
A5（有料） 学生・教員 A3のすべて＋電話システム

3.3 条件付きアクセス

  • 場所による制限：学校外からのアクセス制限
  • デバイスによる制限：登録端末のみアクセス許可
  • 多要素認証（MFA）：パスワード＋SMSコード等

4. パスワード管理

4.1 強固なパスワードの条件

💡 推奨されるパスワード要件

  • 長さ：8文字以上（12文字以上推奨）
  • 複雑さ：大文字、小文字、数字、記号を組み合わせる
  • 推測されにくい：誕生日、名前、辞書の単語を避ける
  • 使い回ししない：サービスごとに異なるパスワード

4.2 児童生徒のパスワード管理

年齢に応じた方法

  • 小学校低学年：
    • 覚えやすい8文字程度
    • パスワードカードを配布（保管注意）
    • ブラウザの保存機能を活用
  • 小学校高学年〜中学生：
    • 自分で設定・管理させる
    • 定期的な変更を指導
    • パスワードマネージャーの利用検討

4.3 パスワード忘れへの対応

🎯 ICT支援員の対応手順

  1. 本人確認：氏名、学年、クラスを確認
  2. リセット実施：管理コンソールからパスワードリセット
  3. 仮パスワード発行：初回ログイン時に変更を促す
  4. 記録：対応日時、対象者を記録
  5. 指導：パスワード管理の重要性を説明

4.4 多要素認証（MFA）

  • 知識要素：パスワード
  • 所持要素：スマートフォン、セキュリティキー
  • 生体要素：指紋、顔認証

※教員アカウントには多要素認証の導入を推奨

5. MDM（モバイルデバイス管理）

5.1 MDMとは

💡 MDM（Mobile Device Management）

スマートフォンやタブレットなどのモバイル端末を一元管理するシステム。設定配布、アプリ管理、セキュリティ対策を遠隔で実施できる。

5.2 MDMの主要機能

  • デバイス登録・管理
    • 端末の一括登録
    • 資産管理（シリアル番号、利用者記録）
  • 設定の一括配布
    • Wi-Fi設定の自動配布
    • メール設定
    • 壁紙、ホーム画面のカスタマイズ
  • アプリ管理
    • アプリの一括インストール・削除
    • アプリの利用制限
    • アプリのアップデート管理
  • セキュリティ機能
    • パスコードの強制
    • 遠隔ロック・ワイプ（データ消去）
    • 位置情報の取得（紛失時）
    • 不適切なサイトのブロック
  • 利用制限
    • カメラ、スクリーンショットの無効化
    • アプリ内課金の禁止
    • 利用時間の制限

5.3 主要なMDM製品

製品名 対応OS 特徴
Google Workspace（MDM機能） Android、iOS、Chrome OS Google Workspace に統合
Microsoft Intune Windows、iOS、Android Microsoft 365 と統合
SKYSEA Client View Windows、iPad、Android 国内シェア大、教育機関向け
MobiConnect iOS、Android、Windows 教育機関での導入実績多数

6. アカウントのライフサイクル管理

6.1 アカウント作成（入学・転入時）

  • 名簿データからアカウント一括作成
  • 初期パスワードの設定・配布
  • 適切なOUへの配置
  • 必要なグループへの追加

6.2 年度更新時

  • 学年・クラスの変更
  • OU の移動
  • グループの再編成
  • 前年度データのアーカイブ

6.3 アカウント削除（卒業・転出時）

⚠️ アカウント削除の注意点

  • 即時削除は避ける：まず停止してデータ移行期間を設ける
  • データのバックアップ：必要なデータは保存
  • 共有データの確認：他のユーザーと共有しているファイルがないか
  • 記録の保持：削除日時、対象者を記録

7. ICT支援員の役割

7.1 日常的なアカウント管理業務

  • パスワードリセット対応
  • アカウントロック解除
  • 新規アカウント作成（転入生）
  • アカウント停止（転出生、休学）

7.2 セキュリティ啓発

  • パスワード管理の指導
  • なりすまし防止の注意喚起
  • ログアウトの習慣化
  • 不審なメール・リンクへの警戒

7.3 トラブル対応

  • ログインできない問題の切り分け
  • アカウント乗っ取りへの対応
  • 不正利用の報告・対処

7.4 記録・報告

  • アカウント作成・削除の記録
  • パスワードリセット対応履歴
  • セキュリティインシデントの報告

📝 演習問題

演習1: アカウント構造の設計

300人規模の小学校（各学年2クラス）で Google Workspace を導入します。組織単位（OU）をどのように設計しますか？理由とともに説明してください。

演習2: パスワード忘れ対応

小学2年生の児童が「パスワードを忘れてログインできない」と相談してきました。ICT支援員としてどのように対応しますか？手順を説明してください。

演習3: セキュリティインシデント

ある児童のアカウントで、本人が知らない間に大量のメールが送信されていることが発覚しました。どのような原因が考えられますか？また、どのように対応すべきですか？

📌 まとめ

  • アカウント管理は個人情報保護と不正アクセス防止のために重要
  • Google Workspace / Microsoft 365 は組織単位（OU）で階層的にアカウントを管理
  • 強固なパスワードは長さ・複雑さ・使い回ししないことが重要
  • MDMによりモバイル端末の設定、アプリ、セキュリティを一元管理できる
  • ICT支援員はパスワードリセット、セキュリティ啓発、トラブル対応を担う
