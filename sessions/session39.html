第39回：ICT支援員A領域試験対策③（状況判断・問題解決編）

試験対策講座 第3回 | 学習時間目安：90分

🎯 学習目標

  • 学校現場で発生する典型的なICTトラブルに対し、適切な状況判断ができる
  • 限られた時間の中で優先順位をつけ、最適な対応策を選択できる
  • 教員や児童生徒に分かりやすく説明し、円滑なコミュニケーションができる
  • ハード/ソフト/ネットワークの三層診断モデルを使った切り分けができる
  • 適切なタイミングでエスカレーション判断ができる

📋 本講座の位置づけ

試験範囲：ICT支援員認定試験 A領域「c) 現場で生じる問題に対する状況判断や対応」

既存講座との関連：

  • 第23回「トラブルシューティングの実践」の応用編
  • 中級編第10回「トラブルシューティング実践」の試験対策版

第1部：典型的なトラブルシナリオと対応

1-1. 授業開始直前の機器トラブル

📝 シナリオ例：

1時間目開始5分前、教員が「電子黒板が起動しない」と駆け込んできた。
今日は公開授業で、デジタル教材を使った授業を予定している。

状況判断のポイント：

  • 時間的制約：授業開始まで5分 → 根本解決は困難
  • 授業への影響度：公開授業 → 高優先度
  • 代替手段の有無：デジタル教材の重要度を確認

対応手順：

⏱️ 即座の状況確認（30秒以内）

  • 電源ランプの確認
  • ケーブル接続の確認
  • 電源コンセントの確認

⏱️ 迅速な判断（1分以内）

  • 再起動で解決可能か → 時間不足
  • 代替機器は利用可能か → プロジェクター、タブレット
  • アナログ代替は可能か → プリント配布

⏱️ 教員への提案（2分以内）

「電子黒板の起動に時間がかかりそうです。まず○○の方法で授業を開始していただき、並行して復旧作業を進めます」

  • 具体的な代替案を3つ程度提示
  • 復旧見込み時間を伝える

🔄 並行対応

  • 授業開始後も復旧作業を継続
  • 復旧したら教員に小声で報告
  • 原因を記録し、再発防止策を検討

✅ 試験でのポイント：

  • ❌「すぐに直します」と安易に約束
  • ❌ 原因究明に時間をかけすぎる
  • ✅ 代替手段を複数提示
  • ✅ 授業への影響を最小化する判断

1-2. ネットワーク接続不良

📝 シナリオ例：

3年2組の教室で、児童用タブレット30台のうち15台がインターネットに接続できない。
デジタルドリルを使った学習の時間。

切り分け診断の手順：

STEP1: 影響範囲の特定

  • 特定の端末のみ？ → 端末側の問題
  • 特定の教室のみ？ → アクセスポイントの問題
  • 学校全体？ → ネットワーク全体/回線の問題

STEP2: 接続状態の確認

Wi-Fiアイコンの表示は？

  • ❌マーク → 接続失敗
  • ❗ビックリマーク → 認証失敗/制限付きアクセス
  • 正常だがネット不通 → DNS/プロキシの問題

STEP3: 基本的な対処

  • Wi-Fiのオン/オフ
  • 機内モードのオン/オフ
  • 端末の再起動
  • ネットワーク設定の再接続

STEP4: 教員への説明

✅ 良い説明例：

「現在、この教室の半数のタブレットでインターネット接続に問題が発生しています。
Wi-Fiの電波は届いていますが、認証に問題があるようです。
まず、接続できている児童でペア学習を始めていただき、
その間に設定を確認いたします。5分程度で復旧できる見込みです」

❌ 悪い説明例：

「DHCPサーバーからIPアドレスの払い出しができていない可能性があります」

→ 専門用語で教員が理解できない

📝 記録すべき情報：

  • 発生日時
  • 影響範囲（教室、台数）
  • 症状の詳細
  • 実施した対処
  • 結果
  • 所要時間

1-3. アカウントロック・パスワード忘れ

📝 シナリオ例：

生徒がパスワードを忘れ、3回間違えてアカウントがロックされた。
次の授業で提出課題がある。

対応フロー：

1️⃣ 緊急度の判断：

  • 即時必要 → 管理者権限で仮パスワード発行
  • 当日中に必要 → 通常手続きで対応
  • 余裕あり → 本人確認を厳密に実施

2️⃣ セキュリティ配慮：

本人確認の実施

  • 学籍番号、氏名、クラスの確認
  • 担任教員への確認
  • 可能であれば生徒証の提示

仮パスワードの発行

  • 推測されにくいランダム文字列
  • 初回ログイン時に変更を強制
  • 発行記録の保管

3️⃣ 再発防止の指導：

  • パスワード管理の重要性を説明
  • 適切なパスワードの作り方を助言
  • パスワード再設定方法の確認

✅ 試験でのポイント：

  • ✅ 本人確認を適切に実施
  • ✅ セキュリティポリシーに従った対応
  • ❌ 本人確認なしでパスワードを開示
  • ❌ 簡単すぎるパスワードの設定

1-4. 印刷できない

問題の切り分けマトリクス：

症状 原因候補 確認方法
全員が印刷できない プリンター本体/ネットワーク プリンター状態確認
特定のPCのみ ドライバー/設定 テストページ印刷
特定のファイルのみ ファイル形式/容量 別ファイルで試行
印刷指示後反応なし プリンターの選択ミス 既定のプリンター確認
エラーメッセージ表示 用紙切れ/紙詰まり/トナー プリンター表示確認

対応チェックリスト：

□ 物理的確認

  • 電源は入っているか
  • 用紙はセットされているか
  • トナー/インクは十分か
  • 紙詰まりはないか
  • エラーランプは点灯していないか

□ 接続確認

  • USBケーブルは接続されているか（ローカル接続の場合）
  • ネットワークケーブルは接続されているか
  • Wi-Fi接続は有効か（無線プリンターの場合）
  • 印刷キューにジョブが溜まっていないか

□ 設定確認

  • 正しいプリンターが選択されているか
  • オフラインになっていないか
  • 印刷の一時停止になっていないか
  • ドライバーは正しくインストールされているか

□ テスト印刷

  • プリンター本体からのテストページ印刷
  • Windowsからのテストページ印刷
  • シンプルなテキストファイルで試行

教員への説明例：

「印刷できない原因を確認しています。
まず、プリンター本体は正常に動作していますので、設定の問題と思われます。
今すぐ必要な資料でしたら、私のPCから代理で印刷いたします。
その間に、先生のPCの設定を確認させていただきます」

1-5. 画面が映らない

チェックフロー図：

画面が映らない
    ↓
[電源確認]
├─ディスプレイ/プロジェクターの電源ランプは?
│  ├─消灯 → 電源ケーブル、コンセント確認
│  └─点灯 → 次へ
    ↓
[ケーブル確認]
├─HDMIケーブルは両端しっかり接続?
│  ├─No → 接続し直す
│  └─Yes → 次へ
    ↓
[入力切替確認]
├─ディスプレイの入力選択は正しい?
│  ├─No → HDMI1/HDMI2等を切替
│  └─Yes → 次へ
    ↓
[PC側出力確認]
├─PC側の映像出力設定は?
│  ├─Windows: Win+P で表示モード確認
│  │  (複製/拡張/セカンドスクリーンのみ)
│  └─Mac: システム設定→ディスプレイ
    ↓
[ケーブル・端子の物理的問題]
├─別のケーブルで試す
├─別のポートで試す
└─別のPCで試す

時間別対応戦略：

残り時間 対応方針
授業開始5分前
  • 深追いせず、代替手段を優先
  • 別のプロジェクター/電子黒板への移動
  • 教員用PCの画面を見せながら授業
授業開始20分前
  • 基本的なチェックを実施（5分）
  • 解決しなければ代替手段
  • 予備機器の準備
授業まで余裕あり
  • 体系的なトラブルシューティング実施
  • 原因の特定と根本的な解決
  • 記録と再発防止策の検討

✅ 試験での出題パターン：

Q: 授業開始5分前、電子黒板に画面が映らない。ケーブルを差し直したが改善しない。最も適切な対応は？

  • A. ✅ 別の教室への移動を提案し、並行して復旧作業を継続
  • B. ❌ 授業開始を遅らせて原因究明
  • C. ❌ 教員に「少し待ってください」と伝える
  • D. ❌ すぐにメーカーサポートに連絡

第2部：複合的問題の判断

2-1. 原因の切り分け方法

三層診断モデル

🖥️ Layer 1: ハードウェア

特徴：

  • 物理的な故障や接続不良
  • 特定の機器のみで発生
  • 再現性が高い

確認方法：

  • 電源、ケーブル接続の目視確認
  • 異音、異臭、発熱の確認
  • LEDランプ、エラー表示の確認
  • 別の機器での動作確認

例：

  • タブレットの画面割れ
  • プロジェクターのランプ切れ
  • キーボードの特定キーが反応しない

💿 Layer 2: ソフトウェア

特徴：

  • 設定や動作の問題
  • 再起動で改善することがある
  • 同じ操作で再現する

確認方法：

  • エラーメッセージの確認
  • 設定内容の確認
  • ログファイルの確認
  • 再起動、再インストールでの改善確認

例：

  • アプリケーションが起動しない
  • ファイルが開けない
  • 設定が保存されない

🌐 Layer 3: ネットワーク

特徴：

  • 複数の機器で同時発生
  • 時間帯で症状が変わる
  • 特定のサービスのみ影響

確認方法：

  • Wi-Fi/有線の接続状態確認
  • IPアドレスの取得確認
  • pingコマンドでの疎通確認
  • 他の端末での動作確認

例：

  • インターネットに接続できない
  • 共有フォルダにアクセスできない
  • 特定のサイトのみ開けない

2-2. 切り分け診断の実践

📝 ケーススタディ1: 複合問題

状況：教員が「デジタル教科書が開けない」と報告。
デジタル教科書はサーバー上にあり、ネットワーク経由でアクセスする。

診断フロー：

STEP1: 現象の詳細確認

  • いつから発生しているか
  • 他の教員も同じ症状か
  • エラーメッセージは表示されるか
  • 以前は正常に開けていたか

STEP2: 切り分け診断

[ネットワーク層の確認]
Q: インターネットは見られますか?
└─Yes → サーバー接続の問題
└─No → ネットワーク全体の問題
     ↓
  [他の端末でも同じ?]
  └─Yes → ネットワーク機器/回線の問題
  └─No → 当該PCのネットワーク設定

[アプリケーション層の確認]
Q: 他のファイルは開けますか?
└─Yes → 特定のファイルの問題
└─No → アプリケーション自体の問題
     ↓
  [再インストールで改善?]
  └─Yes → アプリケーションの不具合
  └─No → OSまたは権限の問題

[ハードウェア層の確認]
Q: 他のアプリケーションは正常に動作しますか?
└─Yes → ハードウェアは正常
└─No → ハードウェア故障の可能性

診断結果の例：

  • Case A: 他の教員も同じ症状 → サーバー側の問題（高優先度、管理者連絡）
  • Case B: 特定のPCのみ → アプリケーション再インストール（中優先度、授業後対応）
  • Case C: 特定のファイルのみ → ファイル破損（低優先度、別ファイルで代替）

2-3. 優先順位の判断

優先度マトリクス：影響度×緊急度

影響度×緊急度 低緊急度 中緊急度 高緊急度
高影響度
（全校、授業中断）		 優先度:中
計画的対応		 優先度:高
当日中対応		 優先度:最高
即時対応
中影響度
（学級、一部機能）		 優先度:低
週次対応		 優先度:中
翌日対応		 優先度:高
当日対応
低影響度
（個人、軽微）		 優先度:最低
月次対応		 優先度:低
週次対応		 優先度:中
翌日対応

判断基準：

影響度の評価：

  • 全校規模で影響 → 高
  • 特定学年/クラスで影響 → 中
  • 個人で影響 → 低

緊急度の評価：

  • 授業中/直前 → 高
  • 当日中に必要 → 中
  • 余裕がある → 低

優先順位の判断例：

シナリオA: 授業中、プロジェクターが故障（1台）

→ 影響度:中、緊急度:高 → 優先度:高

→ 即座に代替機を手配、授業継続を優先

シナリオB: 職員室のプリンターが動かない（授業後）

→ 影響度:中、緊急度:低 → 優先度:低

→ 翌日までに修理、緊急時は別のプリンター案内

シナリオC: 校内ネットワーク全体がダウン（授業中）

→ 影響度:高、緊急度:高 → 優先度:最高

→ 全授業に影響、管理者・業者へ即連絡

2-4. エスカレーション判断

対応レベルの分類：

🟢 レベル1: ICT支援員で対応可能

範囲：

  • 基本的な操作説明
  • ケーブル接続、設定確認
  • 再起動、再インストール
  • アカウント管理（権限内）

対応時間の目安：5〜30分

例：

  • Wi-Fi接続設定
  • プリンター用紙補充
  • パスワードリセット
  • 基本的なソフトの使い方

🟡 レベル2: 管理者・情報担当教員へエスカレーション

範囲：

  • サーバー設定変更
  • ネットワーク機器設定
  • 高度な権限が必要な作業
  • セキュリティに関わる判断

対応時間の目安：30分〜数時間

例：

  • ネットワーク全体の不具合
  • サーバーへのアクセス権限設定
  • 新規アカウントの一括作成
  • セキュリティポリシーの判断

🔴 レベル3: 専門業者へエスカレーション

範囲：

  • 物理的な故障・修理
  • 専門的な設定・構築
  • 保証対象の不具合
  • 法的・セキュリティ上の重大事項

対応時間の目安：数時間〜数日

例：

  • 機器の物理的故障
  • ネットワーク回線の不具合
  • サーバーのハードウェア障害
  • ウイルス感染の疑い

エスカレーション判断フローチャート：

問題発生
    ↓
[自分で対応可能か?]
├─知識・権限がある → レベル1対応
│  └─15分で解決しない
│      ↓
│  [授業への影響は?]
│  ├─重大 → 即エスカレーション
│  └─軽微 → もう15分試行
│      ↓
│      解決しない → エスカレーション
│
└─知識・権限がない
    ↓
[誰にエスカレーション?]
├─設定・権限の問題 → 管理者・情報担当教員
├─物理的故障 → 専門業者
├─ネットワーク全体 → 管理者→業者
└─セキュリティ問題 → 管理者(最優先)

エスカレーション時の報告内容：

  • 症状：何が、どのように動かないか
  • 発生状況：いつ、どこで、誰が、何をしていた時か
  • 影響範囲：何台、どの範囲に影響しているか
  • 実施した対処：何を確認し、何を試したか
  • 緊急度：授業への影響度合い

✅ 試験での出題例：

Q: 校内ネットワークが全て接続不能になった。Wi-Fiルーターの再起動は試したが改善しない。次にとるべき行動は？

  • A. ❌ 自分でネットワーク設定を詳しく調査し、解決策を見つける
  • B. ✅ ネットワーク管理者に連絡し、状況を報告する
  • C. ❌ 全教員に「しばらくお待ちください」と連絡だけする
  • D. ❌ 業者に直接連絡し、修理を依頼する

解説：

  • 全校規模の問題 = 影響度:高
  • インターネット不通 = 多くの授業に影響 = 緊急度:高
  • 権限がない = 自分で対応不可
  • エスカレーションレベル2（管理者）が適切
  • 管理者が状況を判断し、必要なら業者に連絡（レベル3）

第3部：教員への説明とコミュニケーション

3-1. わかりやすい説明

専門用語を避けた説明例：

❌ NG（専門用語） ✅ OK（平易な表現）
DHCPサーバーが応答していません Wi-Fiには繋がっていますが、インターネット接続の設定が取得できていません
DNSの名前解決に失敗しています インターネットのアドレス帳が使えない状態です
HDMIの信号が検出されません PCと画面をつなぐケーブルの信号が届いていません
ドライバーが破損しています プリンターを動かすプログラムに問題があります
802.1X認証に失敗しています 学校のWi-Fiに接続するための認証がうまくいっていません

説明の3要素：

1️⃣ 現状説明（何が起きているか）

「現在、タブレットがインターネットに接続できない状態です」

2️⃣ 原因説明（なぜ起きているか）※簡潔に

「Wi-Fiの設定が正しく保存されていないようです」

3️⃣ 対応説明（どうするか）

「設定を入れ直しますので、3分ほどお待ちください」

説明のNG例とOK例：

❌ NG例1：専門用語過多

「DHCPリース期間が切れていて、IPアドレスの再取得に失敗しているため、デフォルトゲートウェイへの経路が確立できず...」

→ 専門用語が多すぎて理解不能

❌ NG例2：不安を与える

「ちょっと分かりません」

→ 不安を与える、プロとして不適切

❌ NG例3：責任転嫁

「○○が悪いんですよ」

→ 原因の押し付け、関係悪化

✅ OK例：状況・見込み・代替案を提示

「インターネット接続の設定を確認しています。
あと5分ほどで確認できますので、その間に教科書を使った導入をお願いできますでしょうか」

→ 状況・見込み・代替案を提示

3-2. 状況確認の質問技法

5W1H質問法：

質問タイプ 質問例 確認内容
When
（いつ）		 「いつから症状が出ていますか？」
  • 今日初めて / 昨日から / 以前から時々
  • 特定の時間帯 / ずっと
Where
（どこで）		 「他の場所（教室）でも同じですか？」
  • この教室だけ / 特定の階 / 学校全体
  • 特定の場所に移動すると改善
Who
（誰が）		 「他の先生（児童生徒）も同じ症状ですか？」
  • 自分だけ / 何人か / 全員
  • 特定のアカウント / 特定の端末
What
（何が）		 「具体的にどんな症状ですか？」
  • エラーメッセージの内容
  • 画面の表示状態
  • どこまで進んでどこで止まるか
Why
（なぜ）
※聞き方注意		 「何かいつもと違うことをしましたか？」
  • 設定を変更した / アプリを追加した
  • 更新があった
  • ※「なぜそんなことをしたのですか」は禁句
How
（どのように）		 「どのような手順で操作されましたか？」
  • 実際に操作を見せてもらう
  • 再現してもらう

オープン質問 vs クローズド質問：

オープン質問（状況把握に有効）

「どのような症状でしょうか？」
「いつ頃からお困りですか？」

→ 自由に答えてもらい、情報を収集

クローズド質問（切り分けに有効）

「昨日は正常に動いていましたか？（Yes/No）」
「他のタブレットでは正常ですか？（Yes/No）」

→ 原因を絞り込む

質問の組み立て例：

STEP1: オープン質問で状況把握
「どのような問題でしょうか？」
→ 「印刷ができないんです」

STEP2: クローズド質問で切り分け
「昨日は印刷できていましたか？」→ Yes
「他の先生も印刷できませんか？」→ No
「エラーメッセージは表示されますか？」→ No

STEP3: 詳細確認（オープン）
「印刷ボタンを押した後、どうなりますか？」
→ 「反応がない」

STEP4: 原因の仮説と確認（クローズド）
「プリンターは正しく選択されていますか？」
→ 原因特定へ

3-3. 暫定対応と恒久対策の提示

二段階対応の説明法：

パターン1: 授業優先型

【暫定対応】

「まず、今日の授業は○○の方法で進めていただけますでしょうか。
（代替手段を具体的に提示）」

【恒久対策】

「根本的な原因は、授業後にしっかり調査いたします。
同じ問題が起きないよう対策を講じます」

パターン2: 迅速修正型

【暫定対応】

「今すぐできる応急処置で、とりあえず動くようにします。
ただし、これは一時的な対処です」

【恒久対策】

「今週中に正式な設定変更を行い、再発しないようにいたします。
その際、5分ほどお時間をいただきます」

パターン3: 段階的改善型

【暫定対応】

「現在の設定を変更することで、70%程度は改善できます」

【恒久対策】

「完全に解決するには機器の交換が必要です。
業者に見積もりを依頼し、予算措置を検討いたします」

説明時の留意点：

ポイント ❌ 悪い例 ✅ 良い例
過度な約束をしない 「すぐに完全に直します」 「まず応急処置で授業ができるようにします」
見積もり時間に余裕を持つ 「5分で直ります」→ 10分かかる 「10分ほどお時間ください」→ 7分で完了
代替案を複数提示 （提示なし） 案A: 別の機器を使用
案B: 教材を変更
案C: 授業順序を変更
影響範囲を明確に （曖昧な説明） 「この教室だけの問題です」
「他のクラスは正常に使えます」
「来週の授業までには解決します」
フォローアップの約束 （約束なし） 「修理が完了しましたら、ご報告に伺います」
「明日、動作確認をさせてください」

✅ 試験での評価ポイント：

  • ✅ 授業への影響を最小化する提案
  • ✅ 暫定/恒久の両方を説明
  • ✅ 現実的な対応時間の提示
  • ❌ 専門用語での説明
  • ❌ 責任転嫁の表現
  • ❌ 曖昧な対応方針

4-1. ケーススタディ（選択式問題）

問題1: 複合判断

【状況】

2時間目開始10分前、4年1組担任の先生から連絡。
「教室の電子黒板が真っ暗で何も映らない。今日は算数の公開授業で、デジタル教材を使う予定だった」

【あなたの初動確認】

  • 電子黒板の電源は入っている。
  • ケーブルは接続されている。
  • PCの画面は正常に表示されている。

Q: 最も適切な対応の順序は？

  1. A. 入力切替を確認 → 解決しなければプロジェクターを代替機として準備 → 並行して電子黒板の復旧を試みる
  2. B. メーカーサポートに連絡 → 復旧を待つ → 間に合わなければ延期を提案
  3. C. すぐに業者を呼ぶ → 完全に修理してから授業開始
  4. D. 教員に「もう少し待ってください」と伝え、原因究明を最優先
解答と解説を見る

✅ 解答: A

解説：

  • 授業開始まで10分 = 時間的制約が大きい
  • 公開授業 = 延期や中止は困難
  • 入力切替は30秒で確認可能 = まず試す価値あり
  • 代替手段の準備と並行作業 = 授業への影響を最小化
  • B, C, Dは時間がかかりすぎる、または授業を止めてしまう

問題2: 優先順位判断

【状況】

以下の3つの問題が同時に発生しました。あなた（ICT支援員）は1人です。

トラブル1: 1年生の教室で、児童用タブレット5台が起動しない（授業中、すぐに対応が必要）

トラブル2: 職員室のプリンターが印刷できない（通知表の印刷期限が明日）

トラブル3: 校長室のPCがインターネットに接続できない（すぐには使わないが、重要な会議資料を確認したい）

Q: 対応の優先順位として最も適切なのは？

  1. A. トラブル1 → トラブル2 → トラブル3
  2. B. トラブル3 → トラブル1 → トラブル2
  3. C. トラブル2 → トラブル3 → トラブル1
  4. D. トラブル1 → トラブル3 → トラブル2
解答と解説を見る

✅ 解答: A

解説: 優先順位マトリクスによる評価

トラブル 影響度 緊急度 優先度
1（授業中） 中（5台） 高（今すぐ） 最高
2（通知表） 中（職員室） 中（明日） 中
3（校長室） 低（1台） 低（余裕） 低

対応方針：

  • トラブル1: 即座に対応（予備機を渡す等、5分で対処）
  • トラブル2: 当日中に対応（授業の合間に確認）
  • トラブル3: 余裕を持って対応（放課後に対応）

問題3: エスカレーション判断

【状況】

校内の全てのPCとタブレットがインターネットに接続できなくなった。
Wi-Fiの電波は届いているが、Webサイトが表示されない。
ネットワーク機器の再起動を試したが改善しない。

あなたには、ネットワーク機器の詳細設定を変更する権限はない。

Q: 最も適切な対応は？

  1. A. 自分でネットワーク設定を詳しく調査し、解決策を見つける
  2. B. 情報担当教員と管理者に即座に連絡し、状況を報告する
  3. C. 全教員に「しばらくインターネットは使えません」と連絡だけする
  4. D. 業者に直接連絡し、修理を依頼する
解答と解説を見る

✅ 解答: B

解説：

  • 全校規模の問題 = 影響度:高
  • インターネット不通 = 多くの授業に影響 = 緊急度:高
  • 権限がない = 自分で対応不可
  • エスカレーションレベル2（管理者）が適切
  • 管理者が状況を判断し、必要なら業者に連絡（レベル3）

エスカレーション時の報告内容：

  • 症状: 全校でインターネット接続不可
  • 発生時刻: ○時○分頃
  • 影響範囲: 全てのPC、タブレット
  • 実施した対処: ネットワーク機器の再起動（改善なし）
  • 現在の状況: Wi-Fi電波は届くが通信不可

問題5: 複合トラブルの切り分け

【状況】

5年生の理科の授業中、以下の問題が同時に報告されました：

  1. 電子黒板に教材が表示されない
  2. 児童用タブレット10台がインターネットに接続できない
  3. 実物投影機の画像が表示されない

【課題】

これらの問題の原因を切り分け、どのような順序で確認・対応するか、理由とともに記述しなさい。

模範解答例を見る

模範解答：

STEP1: 共通原因の可能性を確認（優先）

まず、「2. タブレット10台がインターネット不通」を確認します。

理由: 複数台が同時に同じ症状 = ネットワーク機器の問題の可能性が高い

確認内容:

  • 他の教室でも同じ症状か
  • 他のクラスのタブレットは正常か
  • → この教室だけなら、教室のアクセスポイントの問題
  • → 全校的なら、ネットワーク全体の問題（管理者に連絡）

STEP2: 授業への影響が大きい問題を優先

次に「1. 電子黒板に教材が表示されない」を確認します。

理由: 教員の説明に必要、授業の進行に直接影響

確認内容:

  • PC画面は正常に表示されているか
  • HDMIケーブルは接続されているか
  • 電子黒板の入力切替は正しいか
  • → 5分で解決しなければ、プロジェクターを代替機として準備

STEP3: 影響度が低い問題は後回し

最後に「3. 実物投影機の画像が表示されない」を確認します。

理由: 今すぐ使わない、代替手段（タブレットのカメラ）がある

確認内容:

  • 実物投影機の電源は入っているか
  • ケーブル接続は正しいか
  • 電子黒板の入力切替を確認
  • → 授業後にゆっくり対応

優先順位の理由：

  • ネットワーク問題は複数に影響 = 根本原因の可能性
  • 電子黒板は授業の中心 = 影響度が高い
  • 実物投影機は補助的 = 代替手段あり

5-2. 試験本番での注意点

選択肢の見極め方：

❌ 誤りの選択肢の特徴：

  • 「すぐに完全に解決します」→ 過度な約束
  • 「原因を徹底的に調査します」→ 時間がかかりすぎ
  • 「メーカーに連絡して待ちます」→ 授業が止まる
  • 「専門用語で詳しく説明します」→ 教員が理解できない
  • 「私の権限では対応できません」→ 何もしない

✅ 正解の選択肢の特徴：

  • 「まず○○で授業を始めていただき...」→ 授業継続優先
  • 「代替手段として△△を使用し...」→ 柔軟な対応
  • 「5分ほどで確認できます」→ 現実的な時間
  • 「分かりやすく言いますと...」→ 平易な説明
  • 「管理者に状況を報告します」→ 適切なエスカレーション

時間配分：

  • A領域全体: 30問 / 30分 = 1問あたり1分
  • 状況判断問題は文章が長い = 1問1分30秒
  • 即答できない問題は一旦飛ばす
  • 全体を一巡してから見直し

解答のコツ：

1️⃣ まず授業への影響度を考える

  • 授業中 > 授業前 > 授業後
  • 公開授業 > 通常授業

2️⃣ 時間制約を確認

  • 「直前」なら代替手段優先
  • 「余裕あり」なら根本解決

3️⃣ 自分の権限を確認

  • 権限内 → 自分で対応
  • 権限外 → エスカレーション

4️⃣ 複数の選択肢で迷ったら

  • より具体的な方を選ぶ
  • より教員に寄り添った方を選ぶ
  • よりリスクが低い方を選ぶ

5-3. 復習チェックリスト

試験前に以下を確認しましょう：

□ 典型的トラブル5つの対応手順を言える

  • 授業直前の機器トラブル
  • ネットワーク接続不良
  • アカウントロック
  • 印刷できない
  • 画面が映らない

□ 切り分け診断の3層（ハード/ソフト/ネットワーク）を説明できる

□ 優先順位マトリクス（影響度×緊急度）を使える

□ エスカレーション3レベルを判断できる

□ 専門用語を平易な言葉に言い換えられる（最低5つは用意）

□ 暫定対応と恒久対策の違いを説明できる

□ 5W1Hの質問技法を使える

□ タイムプレッシャー下での判断基準を持っている

第6部：実践問題集

追加演習問題

問題7

教員から「昨日まで使えていた共有フォルダが開けなくなった」と連絡。
他の教員は正常にアクセスできている。
最初に確認すべき項目として最も適切なのは？

  1. A. サーバーの再起動
  2. B. 当該教員のアカウント権限
  3. C. ネットワークケーブルの接続
  4. D. 共有フォルダの設定変更
解答と解説を見る

✅ 解答: B

解説: 他の教員は正常 = サーバー・ネットワークは正常。特定のユーザーのみ = アカウント権限の問題が最も可能性が高い。

問題8

校内の全てのPCで特定のWebサイトのみが開けない。
他のWebサイトは正常に閲覧可能。
最も考えられる原因は？

  1. A. 各PCのブラウザ設定
  2. B. フィルタリング設定
  3. C. Wi-Fiルーターの故障
  4. D. インターネット回線の不具合
解答と解説を見る

✅ 解答: B

解説: 全PC + 特定サイトのみ = フィルタリング（URLフィルタ）の可能性が最も高い。全体的な接続障害ではない。

問題9

タブレットの画面タッチが反応しない。
電源は入っており、画面は表示されている。
最初に試すべき対処として最も適切なのは？

  1. A. 画面保護フィルムを剥がす
  2. B. タブレットを再起動する
  3. C. タッチパネルの設定を確認
  4. D. すぐに修理業者に連絡
解答と解説を見る

✅ 解答: B

解説: ソフトウェア的な問題の可能性がある場合、まず再起動を試すのが基本。物理的対処（A）や業者連絡（D）は後回し。

問題10

授業中に突然、全児童のタブレットから音が出なくなった。
イヤホンを挿しても音が出ない。
最も考えられる原因は？

  1. A. 全タブレットのスピーカー故障
  2. B. MDM（管理ツール）での一括設定変更
  3. C. 教室の電波状況悪化
  4. D. アプリの不具合
解答と解説を見る

✅ 解答: B

解説: 全端末で同時発生 = 個別故障ではない。MDMでの音量制限やミュート設定が一括適用された可能性が高い。

💡 さらなる演習のために

実際の試験対策では最低20問程度の演習を推奨します。既存講座（第23回、中級編第10回）の演習問題も併せて復習しましょう。

まとめ：状況判断・問題解決の5原則

原則1: 授業ファースト

  • 授業への影響を最小化することが最優先
  • 原因究明 < 授業継続
  • 完全な修理 < 暫定対応+代替手段
  • 「今できること」を即座に提供

原則2: 迅速な切り分け

  • 30秒〜1分で原因の見当をつける
  • ハード/ソフト/ネットワーク
  • 影響範囲（1台/複数/全体）
  • 再現性（常に/時々/初めて）

原則3: 優先順位の明確化

  • 影響度×緊急度で判断
  • 授業中 > 授業前 > 授業後
  • 全校 > 学年 > 学級 > 個人
  • 公開授業 > 通常授業

原則4: わかりやすいコミュニケーション

  • 専門用語を避け、3要素で説明
  • 現状（何が起きているか）
  • 原因（なぜ起きているか）
  • 対応（どうするか）

原則5: 適切なエスカレーション

  • 自分の権限と能力を見極める
  • 15分で解決しない → エスカレーション検討
  • 権限外の作業 → 管理者へ
  • 物理故障 → 業者へ
