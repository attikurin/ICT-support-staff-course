📚 第1章：コンピュータの基本構造

1.1 コンピュータの5大機能

コンピュータは、情報処理を行うための機械であり、以下の5つの基本機能を持っています。これは「コンピュータの5大機能」として知られ、すべてのコンピュータに共通する基本構造です。

📌 5大機能の概要

1. 入力（Input）
  • デバイス例：キーボード、マウス、タッチパネル、カメラ、マイクなど
  • 役割：外部からデータや命令を取り込む機能
  • 学校現場の例：タブレットのタッチ操作、QRコードリーダー
2. 記憶（Storage）
  • デバイス例：RAM（主記憶装置）、SSD/HDD（補助記憶装置）
  • 役割：データやプログラムを保存する機能
  • 学校現場の例：児童生徒の作品ファイル、授業用教材の保存
3. 演算（Arithmetic/Logic）
  • デバイス例：CPU（中央演算処理装置）内の演算装置
  • 役割：数値計算や論理演算を行う機能
  • 学校現場の例：プログラミング実行、画像編集の処理
4. 制御（Control）
  • デバイス例：CPU内の制御装置
  • 役割：各装置に指示を出し、全体を統括する機能
  • 学校現場の例：OSによる各アプリの動作制御
5. 出力（Output）
  • デバイス例：ディスプレイ、プリンター、スピーカーなど
  • 役割：処理結果を外部に伝える機能
  • 学校現場の例：電子黒板への表示、印刷物の出力

💡 教育現場での応用

GIGA端末（タブレットやノートPC）で考えると：

  1. 入力：タッチペンで図を描く
  2. ↓
  3. 記憶：作品がSSDに保存される
  4. ↓
  5. 演算：CPUが画像処理
  6. ↓
  7. 制御：OSがアプリを管理
  8. ↓
  9. 出力：画面に表示、または大型提示装置に投影

この流れを理解することで、「なぜ端末が遅いのか」「どこに問題があるのか」を特定しやすくなります。

1.2 ノイマン型コンピュータの概念

現代のほとんどのコンピュータは「ノイマン型（フォン・ノイマン型）」と呼ばれる基本設計に基づいています。

📌 ノイマン型の特徴

  • プログラム内蔵方式：プログラム（命令）とデータを同じメモリに格納する
  • 逐次制御方式：プログラムを順次実行する（逐次処理）
  • 柔軟性：プログラムの変更により、同じハードウェアで異なる処理ができる

💡 教育現場での意味

  • 同じ端末で、ワープロソフトも表計算ソフトも動く理由
  • ソフトウェアを更新すれば、ハードウェアを変えずに機能拡張できる
  • プログラミング教育で「命令を記憶させる」概念の基礎

📚 第2章：CPU（プロセッサ）の仕組み

2.1 CPUの役割と構造

CPU（Central Processing Unit：中央処理装置）は、コンピュータの「頭脳」に相当します。

📌 CPUの主要構成要素

1. 制御装置（Control Unit）
  • メモリから命令を取り出し、解読する
  • 各装置に適切な制御信号を送る
2. 演算装置（ALU：Arithmetic Logic Unit）
  • 算術演算（加算、減算、乗算、除算）
  • 論理演算（AND、OR、NOTなど）
3. レジスタ（Register）
  • CPU内部の超高速記憶領域
  • 現在処理中のデータや命令を一時保存
4. キャッシュメモリ（Cache Memory）
  • CPUとメインメモリの速度差を埋める高速メモリ
  • L1、L2、L3キャッシュの階層構造

2.2 CPUの性能指標

教育現場で端末を選定する際、CPUの性能を理解することは重要です。

📌 主要な性能指標

1. クロック周波数（Clock Speed）
  • 単位：GHz（ギガヘルツ）
  • 例：2.4GHz、3.6GHz
  • 意味：1秒間に何回処理サイクルを実行できるか
  • 注意：周波数が高い = 速いとは限らない（アーキテクチャに依存）
2. コア数（Core Count）
  • デュアルコア（2コア）、クアッドコア（4コア）、オクタコア（8コア）など
  • 複数の処理を同時に実行できる
  • 教育現場：ブラウザで複数タブ + ビデオ会議 + 録画などマルチタスクに有利
3. スレッド数（Thread Count）
  • ハイパースレッディング技術により、1コアで2スレッド処理可能
  • 例：4コア8スレッド
  • より多くの並行処理が可能
4. キャッシュメモリ容量
  • L1キャッシュ：数十KB（最速）
  • L2キャッシュ：数百KB〜数MB
  • L3キャッシュ：数MB〜数十MB（共有キャッシュ）
  • 大容量ほど処理効率が向上

2.3 主要CPUメーカーと教育端末への適用

🔷 Intel プロセッサ

従来のCeleron/Pentiumシリーズ（〜2023年）
  • シリーズ：Celeron N4020、N4500、N5100など
  • GIGA第1期の主流：Celeron N4500が文部科学省の最低基準
  • 性能：基本的な学習活動に対応
  • PassMark参考値：
    • N4020：1,567点
    • N4500：1,818点
    • N5100：3,298点
【追加】Intel Processorシリーズ（2024年〜）

💡 重要な変更点

  • 背景：2024年以降、IntelはCeleron/PentiumブランドをIntel Processorブランドに統合
  • 主要モデル：Intel Processor N100、N95、N200など
  • GIGA第2期での採用増：N100/N95が実質的なCeleron後継として教育端末に採用され始めている
Intel N100/N95の性能（2025年推奨）
  • 性能：約2.8〜3倍の性能向上（N4500比較）
    • Intel N100：5,157点（PassMark）
    • Intel N95：5,206点（PassMark）
    • 参考：第7世代デスクトップCore i5に匹敵する性能
  • 文部科学省の見解：「Intel Processor N100は、Intel Celeron Processor N4500と同等以上との最低スペック基準を満たす」（2024年4月公式文書）
  • 特徴：
    • 超省電力（10W）
    • 長時間バッテリー駆動
    • コストパフォーマンス優秀
  • 教育用途：文書作成、ブラウジング、プログラミング（Scratch、micro:bit）、オンライン授業に十分

🔶 AMD プロセッサ

  • シリーズ：Ryzen 3、Ryzen 5、Ryzen 7、Athlon
  • 教育向け：Athlon、Ryzen 3/5（コストパフォーマンス重視）
  • 特徴：Intelより価格が抑えられ、マルチコア性能が高い
  • 用途：動画編集、3Dモデリング、STEAM教育
  • 注意：教育現場ではIntelが主流、AMD採用は一部に留まる

📱 ARM プロセッサ（Apple Silicon含む）

  • シリーズ：Apple A/Mシリーズ（iPad）、Qualcomm Snapdragon（一部Chromebook）
  • 教育向け：iPad（A14〜A17、M1、M2チップ）
  • 特徴：低消費電力、高効率、長時間バッテリー駆動
  • 用途：モバイル学習、1人1台端末、フィールドワーク
  • 注意：教育現場ではiPadが主流、Apple Silicon搭載Macは高等教育や特定用途

2.4 教育現場での選定ポイント

用途 推奨CPU 具体例
低学年・基本利用 Celeron N4500、Intel N100、ARM系 文書作成、ブラウジング
中高学年・汎用 Intel N100/N95、Core i3、Ryzen 3 オンライン授業、プログラミング
STEAM教育・専門用途 Core i5以上、Ryzen 5以上 動画編集、3D CAD、データサイエンス
予算制約 Intel N100、AMD Athlon 最高のコストパフォーマンス
バッテリー重視 ARM系（iPad、Chromebook） 長時間駆動、持ち運び

⚠️ Intel N100/N95の登場による影響

  • 2024〜2025年の新規導入では、従来のCeleron N4500よりIntel N100/N95が推奨される傾向
  • 同価格帯で約3倍の性能向上
  • 5年間の使用を考慮すると、将来のソフトウェア要件にも対応可能
