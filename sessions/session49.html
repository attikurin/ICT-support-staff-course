📚 第1章：コンピュータの基本構造

1.1 コンピュータの5大機能

コンピュータは、情報処理を行うための機械であり、以下の5つの基本機能を持っています。これは「コンピュータの5大機能」として知られ、すべてのコンピュータに共通する基本構造です。

📌 5大機能の概要

1. 入力（Input）
  • デバイス例：キーボード、マウス、タッチパネル、カメラ、マイクなど
  • 役割：外部からデータや命令を取り込む機能
  • 学校現場の例：タブレットのタッチ操作、QRコードリーダー
2. 記憶（Storage）
  • デバイス例：RAM（主記憶装置）、SSD/HDD（補助記憶装置）
  • 役割：データやプログラムを保存する機能
  • 学校現場の例：児童生徒の作品ファイル、授業用教材の保存
3. 演算（Arithmetic/Logic）
  • デバイス例：CPU（中央演算処理装置）内の演算装置
  • 役割：数値計算や論理演算を行う機能
  • 学校現場の例：プログラミング実行、画像編集の処理
4. 制御（Control）
  • デバイス例：CPU内の制御装置
  • 役割：各装置に指示を出し、全体を統括する機能
  • 学校現場の例：OSによる各アプリの動作制御
5. 出力（Output）
  • デバイス例：ディスプレイ、プリンター、スピーカーなど
  • 役割：処理結果を外部に伝える機能
  • 学校現場の例：電子黒板への表示、印刷物の出力

💡 教育現場での応用

GIGA端末（タブレットやノートPC）で考えると：

  1. 入力：タッチペンで図を描く
  2. ↓
  3. 記憶：作品がSSDに保存される
  4. ↓
  5. 演算：CPUが画像処理
  6. ↓
  7. 制御：OSがアプリを管理
  8. ↓
  9. 出力：画面に表示、または大型提示装置に投影

この流れを理解することで、「なぜ端末が遅いのか」「どこに問題があるのか」を特定しやすくなります。

1.2 ノイマン型コンピュータの概念

現代のほとんどのコンピュータは「ノイマン型（フォン・ノイマン型）」と呼ばれる基本設計に基づいています。

📌 ノイマン型の特徴

  • プログラム内蔵方式：プログラム（命令）とデータを同じメモリに格納する
  • 逐次制御方式：プログラムを順次実行する（逐次処理）
  • 柔軟性：プログラムの変更により、同じハードウェアで異なる処理ができる

💡 教育現場での意味

  • 同じ端末で、ワープロソフトも表計算ソフトも動く理由
  • ソフトウェアを更新すれば、ハードウェアを変えずに機能拡張できる
  • プログラミング教育で「命令を記憶させる」概念の基礎

📚 第2章：CPU（プロセッサ）の仕組み

2.1 CPUの役割と構造

CPU（Central Processing Unit：中央処理装置）は、コンピュータの「頭脳」に相当します。

📌 CPUの主要構成要素

1. 制御装置（Control Unit）
  • メモリから命令を取り出し、解読する
  • 各装置に適切な制御信号を送る
2. 演算装置（ALU：Arithmetic Logic Unit）
  • 算術演算（加算、減算、乗算、除算）
  • 論理演算（AND、OR、NOTなど）
3. レジスタ（Register）
  • CPU内部の超高速記憶領域
  • 現在処理中のデータや命令を一時保存
4. キャッシュメモリ（Cache Memory）
  • CPUとメインメモリの速度差を埋める高速メモリ
  • L1、L2、L3キャッシュの階層構造

2.2 CPUの性能指標

教育現場で端末を選定する際、CPUの性能を理解することは重要です。

📌 主要な性能指標

1. クロック周波数（Clock Speed）
  • 単位：GHz（ギガヘルツ）
  • 例：2.4GHz、3.6GHz
  • 意味：1秒間に何回処理サイクルを実行できるか
  • 注意：周波数が高い = 速いとは限らない（アーキテクチャに依存）
2. コア数（Core Count）
  • デュアルコア（2コア）、クアッドコア（4コア）、オクタコア（8コア）など
  • 複数の処理を同時に実行できる
  • 教育現場：ブラウザで複数タブ + ビデオ会議 + 録画などマルチタスクに有利
3. スレッド数（Thread Count）
  • ハイパースレッディング技術により、1コアで2スレッド処理可能
  • 例：4コア8スレッド
  • より多くの並行処理が可能
4. キャッシュメモリ容量
  • L1キャッシュ：数十KB（最速）
  • L2キャッシュ：数百KB〜数MB
  • L3キャッシュ：数MB〜数十MB（共有キャッシュ）
  • 大容量ほど処理効率が向上

📚 第3章：メモリとストレージ

3.1 メモリ（RAM：主記憶装置）

RAM（Random Access Memory）は、現在作業中のデータやプログラムを一時的に保存する高速メモリです。

📌 RAMの特性

  • 揮発性メモリ：電源を切るとデータが消える
  • 高速アクセス：CPUが直接高速に読み書き
  • 容量：4GB、8GB、16GB、32GBなど
  • 規格：DDR4（現在の主流）、DDR5（次世代規格）

教育現場での適切なRAM容量

用途 推奨RAM容量 具体例
基本的な学習活動 4GB 文書作成、ブラウジング、Scratchプログラミング
標準的な学習活動 8GB 動画視聴、オンライン授業、簡単な画像編集
高度な学習活動 16GB以上 動画編集、3D CAD、データサイエンス、AI活用

⚠️ 文部科学省 GIGA第2期基準（2024年4月）

  • Windows端末：8GB以上（ブラウザ上での活用が前提かつ活用実態上支障がない場合は4GBも許容）
  • Chromebook：4GB以上
  • iPad：規定なし（OSによる最適化で効率的に動作）

💡 RAMが不足すると起こる現象

  • アプリの起動・切り替えが遅い
  • 複数タブを開くとブラウザが固まる
  • ビデオ会議中に画面共有すると動作が遅くなる
  • 「メモリ不足」のエラーメッセージが表示される
トラブルシューティング
  1. タスクマネージャー（Windows）、アクティビティモニタ（Mac）でメモリ使用率を確認
  2. 不要なアプリを終了する
  3. ブラウザのタブを減らす
  4. 将来的にRAMの増設を検討（可能な機種の場合）

📌 DDR4とDDR5について

  • DDR4：現在の教育端末で主流（最大3200MHz）
  • DDR5：次世代規格（4800MHz〜、高性能モデルでは6400MHz以上も可能）、2025年時点では高価格帯のみ
  • 教育現場での選定：現時点ではDDR4で十分。DDR5は将来的な選択肢として知識を持つことは有用ですが、コスト面から普及は限定的です。

3.2 ストレージ（補助記憶装置）

ストレージは、データやプログラムを永続的に保存する装置です。

ストレージの種類

💾 SSD（Solid State Drive）

  • 構造：フラッシュメモリ（半導体）
  • 速度：⭐⭐⭐⭐⭐ 非常に高速（400MB/s〜2GB/s以上）
  • 耐衝撃性：⭐⭐⭐⭐⭐ 可動部なし、衝撃に強い
  • 騒音：⭐⭐⭐⭐⭐ 完全無音
  • 消費電力：⭐⭐⭐⭐☆ 低い
  • 価格：⭐⭐⭐☆☆ やや高い
  • 寿命：書き込み回数に制限あり（通常5〜10年）
  • 教育現場推奨度：⭐⭐⭐⭐⭐ 強く推奨

💾 eMMC（embedded MultiMediaCard）

  • 構造：フラッシュメモリ（基板内蔵型）
  • 速度：⭐⭐⭐☆☆ 中速（45MB/s〜400MB/s、eMMC 5.0規格）
  • 特徴：SSDより遅いがHDDより速い、小型・省電力
  • 採用例：一部の低価格Chromebook（32GB〜64GB）
  • 注意点：SSDと比較して読み書き速度が遅く、容量も小さい
  • 教育現場での評価：Chromebookはクラウドベースのため、eMMCでも基本利用は可能。ただし、大容量ファイルの保存や頻繁なローカル処理には不向き。

💾 HDD（Hard Disk Drive）

  • 構造：磁気ディスク（機械式）
  • 速度：⭐⭐☆☆☆ 遅い
  • 耐衝撃性：⭐⭐☆☆☆ 衝撃に弱い
  • 騒音：⭐⭐☆☆☆ 動作音あり
  • 消費電力：⭐⭐⭐☆☆ 高い
  • 価格：⭐⭐⭐⭐⭐ 安い（GB単価）
  • 寿命：機械的摩耗で劣化（通常3〜4年）
  • 教育現場推奨度：⭐⭐☆☆☆ 非推奨（GIGA端末では採用なし）

💡 教育現場でSSDが推奨される理由

  1. 起動時間短縮：OS起動が10〜20秒（HDDは1〜2分）
  2. 耐久性：児童生徒が持ち運ぶ際の落下・衝撃に強い
  3. 静音性：授業中の動作音がない
  4. 省電力：バッテリー駆動時間が延びる
  5. 学習体験向上：アプリの起動・保存が速く、ストレスフリー

GIGA端末における適切なストレージ容量

OS 最低基準 推奨容量 特記事項
Chromebook 32GB以上（文科省基準） 64GB クラウド保存前提、eMMC/SSD
iPad 64GB以上（文科省基準） 128GB 動画・写真を多用する場合は注意
Windows 64GB以上（文科省基準） 128〜256GB ローカルアプリ利用が多い場合

⚠️ 容量管理のポイント

  • 定期的なファイル整理（不要ファイルの削除）
  • クラウドストレージの活用（Google Drive、OneDriveなど）
  • 大容量ファイル（動画）は外部ストレージやサーバーに保存
  • ストレージ使用状況の定期チェック
  • 【重要】特にeMMC搭載端末では、クラウド活用が必須

📚 第4章：コンピュータの性能指標

4.1 ベンチマークとは

ベンチマークは、コンピュータの性能を数値化して比較するための標準化されたテストです。

📌 主要なベンチマークツール

Geekbench
  • CPU性能を測定
  • シングルコア・マルチコアスコア
  • クロスプラットフォーム対応
  • 教育現場での活用：MediaTek社が教育向けChromebook選定でGeekbench 6を推奨
PassMark
  • CPU、GPU、メモリ、ストレージの総合評価
  • 教育機関でも広く利用
  • 本講座での参考値：CPU性能比較に使用
CrystalDiskMark
  • SSD/HDDの読み書き速度測定
  • 教育現場のストレージ性能チェックに最適
3DMark
  • GPU（グラフィックス）性能測定
  • 3D教材、VR/AR利用時の参考

💡 ベンチマークの読み方と注意点

  • 参考値として活用：絶対値よりも、比較対象との相対的な差を見る
  • 実用シーンとの相関：実際の使用シーン（ブラウジング、文書作成など）での体感との相関を考慮
  • コストパフォーマンス評価：高スコア = 高額とは限らない
  • 【重要】教育現場では、ベンチマークスコアよりも実際の使用感と予算のバランスが重要

PassMark スコアの目安（教育用途）

用途 PassMark推奨値 参考CPU

    • ストレージ：512GB（SSD、NVMe推奨）
    • GPU：専用グラフィックスカード推奨
  • 参考機種：ハイエンドWindows端末、MacBook Pro

📕 レベル4：専門的・研究用途

  • 対象：STEAM専門コース、情報科専門
  • 主な用途：AI/機械学習、ゲーム開発、VR/AR制作
  • 推奨スペック：
    • CPU：Intel Core i7以上、AMD Ryzen 7以上
    • RAM：32GB
    • ストレージ：1TB（NVMe SSD）
    • GPU：NVIDIA GeForce RTX / AMD Radeon RX
  • 参考機種：ワークステーション級端末

4.3 性能とコストのバランス

教育現場では、「最高性能の端末」ではなく、「目的に合った適切な性能の端末」を選ぶことが重要です。

💡 コストパフォーマンス評価の視点

1. 初期導入コスト
  • 1台あたりの価格
  • 一括導入時の割引・教育機関特別価格
  • 周辺機器・保護ケース込みの総額
2. 運用コスト（TCO：Total Cost of Ownership）
  • 保証期間と保証内容
  • 故障率と修理費用
  • バッテリー交換コスト
  • 保守サポート費用
3. ライフサイクルコスト
  • 端末の使用期間（3年、5年、7年）
  • OSサポート期限
  • 将来的なソフトウェアの動作要件
4. 教育効果への投資対効果（ROI）
  • 学習効率の向上
  • 教員の負担軽減
  • 児童生徒の満足度

TCO（総所有コスト）の具体的計算例

⚠️ 予備機の確保（重要）

  • 文部科学省推奨：整備台数の15%を予備機として確保
  • 根拠：故障率の実態調査（自治体により1%〜44%と大きな格差、2024年朝日新聞調査）
    • 名古屋市：44%（最高）
    • 広島市：1%（最低）
  • 授業継続：平均的な故障率を考慮すると、予備機は必須

予算制約下での優先順位

限られた予算で最大の効果を得るための優先順位：

1️⃣ 必須項目（妥協不可）
  • SSD搭載（HDDは避ける、eMMCは用途次第）
  • 十分なバッテリー駆動時間（8時間以上、文科省基準）
  • 堅牢性（落下テスト合格、防塵防滴、MIL-STD-810G/810H推奨）
  • USB Type-C PD対応（充電規格の統一）
2️⃣ 重要項目（可能な限り確保）
  • 8GB RAM（4GBは将来性が低い）
  • タッチパネル対応（低学年の直感的操作）
  • ペン入力対応（手書き学習の重要性）
  • Wi-Fi 6対応（IEEE 802.11ax、GIGA第2期基準）
3️⃣ 推奨項目（予算に余裕があれば）
  • カメラ性能（フィールドワーク、実験記録）
  • 高解像度ディスプレイ（目の負担軽減）
  • 長期保証・手厚いサポート（3年以上）
  • Intel N100以上のCPU（将来の性能要件対応）
4️⃣ オプション項目（特定用途で必要）
  • 専用GPU（3D、動画編集専門コースのみ）
  • 大容量ストレージ（通常は外部ストレージで対応可）
  • LTE/5Gモバイル通信（校外学習頻度による）

📚 第5章：学校での機器選定のポイント

5.1 機器選定のプロセス

教育現場での端末選定は、体系的なプロセスで進めることが重要です。

📌 ステップ1：ニーズ調査

  • 対象学年と人数
  • 主な使用目的と頻度
  • 教科・単元での具体的な利用シーン
  • 教員のICTスキルレベル
  • 既存のICT環境（ネットワーク、既存端末、周辺機器）
  • 【追加】GIGA第1期の総括（利活用状況、故障・トラブル実態）

📌 ステップ2：要件定義

  • 必須機能のリストアップ
  • 性能要件の明確化（文部科学省最低スペック基準を参照）
  • 予算の上限設定（TCO含む）
  • 導入台数と配備計画（予備機15%を含む）
  • 運用体制（管理者、サポート体制）

📌 ステップ3：候補機種の選定

  • 市場調査（3〜5機種を比較対象に）
  • スペック表の比較
  • 実機デモ・トライアルの実施
  • 他校の導入事例調査
  • 【追加】OSサポート期限の確認（Chromebook AUE、Windows EOL、iPadOS対応年数）

📌 ステップ4：総合評価

  • スペック評価（性能、機能）
  • コスト評価（初期費用、運用費用、TCO）
  • サポート評価（保証、修理体制、研修）
  • 将来性評価（OS更新、拡張性）
  • 【追加】堅牢性評価（MIL規格準拠、落下試験データ）

📌 ステップ5：導入決定と調達

  • 稟議書・予算申請
  • 入札・見積依頼（共同調達の検討）
  • 契約・発注
  • 納品・検収

5.2 チェックリストの活用

機器選定時には、以下のチェックリストを活用して漏れのない評価を行いましょう。

✅ 機器選定チェックリスト（2025年版）

□ ハードウェア仕様
  • ☐ CPU：用途に適した性能か（Intel N100以上推奨）
  • ☐ RAM：8GB以上を推奨（4GBは最低限、Chromebookは4GBでも可）
  • ☐ ストレージ：SSD搭載必須（eMMCは用途次第）、容量は十分か
  • ☐ ディスプレイ：サイズ、解像度、タッチ対応
  • ☐ バッテリー：8時間以上の駆動時間（文科省基準）
  • ☐ 重量：児童生徒が持ち運べる重さか（1.5kg以下、理想は1.0〜1.3kg）
□ 堅牢性・耐久性
  • ☐ 落下テスト合格（MIL-STD-810G/810H推奨、122cmからの落下試験）
  • ☐ 防塵・防滴性能（IP規格）
  • ☐ キーボード・タッチパッドの耐久性
  • ☐ ヒンジ・接続部の強度
  • ☐ 【追加】実機での耐久性評価（他校の故障率データ参照）
□ OS・ソフトウェア
  • ☐ OSの種類とバージョン
  • ☐ 【追加】OSサポート期限
    • Chromebook：AUE（自動更新期限）10年（2021年以降モデル）
    • Windows：Windows 11は2032年まで（予定）
    • iPadOS：通常5〜7年（機種により異なる）
  • ☐ 既存システムとの互換性
  • ☐ 教育用アプリの動作確認
□ ネットワーク・接続性
  • ☐ Wi-Fi規格（Wi-Fi 6（IEEE 802.11ax）推奨、GIGA第2期基準）
  • ☐ Bluetooth対応
  • ☐ USB端子の数と種類（USB Type-C PD必須、文科省基準）
  • ☐ HDMI出力（大型提示装置接続）
□ セキュリティ
  • ☐ OS標準のセキュリティ機能
  • ☐ MDM（モバイルデバイス管理）対応
  • ☐ 生体認証（指紋、顔認証）
  • ☐ 物理的なセキュリティスロット
□ サポート・保証
  • ☐ 保証期間（3年以上推奨）
  • ☐ 修理対応（オンサイト、センドバック、代替機提供）
  • ☐ 教員向け研修サポート
  • ☐ 技術サポート窓口（電話、メール、チャット）
  • ☐ 【追加】バッテリー交換費用と対応期間
□ コスト
  • ☐ 1台あたりの価格
  • ☐ 教育機関割引の有無
  • ☐ 保守費用（年額）
  • ☐ 修理時の追加費用
  • ☐ アクセサリ・周辺機器の費用
  • ☐ 【追加】5年間TCOの試算
□ 将来性・拡張性
  • ☐ メモリ増設の可否
  • ☐ ストレージ交換・増設の可否
  • ☐ OSアップデート対応予定
  • ☐ メーカーの教育市場へのコミットメント
  • ☐ 【追加】予備部品の供給期間

5.3 実機評価の重要性

カタログスペックだけでなく、実際に端末を触って評価することが重要です。

💡 実機評価のポイント

操作性
  • キーボードのタイピング感
  • タッチパッドの反応
  • タッチパネルの精度
  • ペン入力の書き心地
画面品質
  • 色再現性、明るさ
  • 視野角
  • 屋外での視認性
  • ブルーライトカット機能
音質
  • スピーカーの音量・音質
  • マイクの集音性能
  • ビデオ会議での聞き取りやすさ
発熱・騒音
  • 高負荷時の発熱具合
  • ファンの騒音レベル
  • 長時間使用時の快適性
実際の使用シーン再現
  • ブラウザで複数タブを開く
  • ビデオ会議をしながら資料を表示
  • 動画編集ソフトの動作
  • プログラミング環境の動作

⚠️ 児童生徒による評価

  • 実際に使用する児童生徒にも試用してもらう
  • 大人と子どもでは操作感が異なる
  • 重さ、持ちやすさ、画面サイズの適切性
  • 直感的に使えるか（ユーザビリティ）

5.4 導入後の評価と見直し

端末導入後も、継続的な評価と改善が重要です。

📊 導入後評価の項目

技術的評価
  • 実際の性能は期待通りか
  • 故障率・トラブル発生率（自治体平均との比較）
  • バッテリーの劣化状況（3〜5年後の充電持続時間）
  • OSアップデート後の動作
教育的評価
  • 学習目標の達成度
  • 児童生徒の満足度
  • 教員の使いやすさ
  • 授業での活用頻度
運用評価
  • サポート対応の質
  • 修理のスピード
  • 管理の負担
  • ランニングコスト
次回更新への反映（GIGA第3期に向けて）
  • 良かった点、改善点の整理
  • 次世代端末への要望
  • 予算計画の見直し
  • 【追加】AUE/EOL到達時期の把握

5.5 NEXT GIGA（第2期）の主要課題と対策

文部科学省が示す「次期ICT環境整備方針」（2024年6月）より：

⚠️ 課題1：端末故障・バッテリー劣化への対応

実態：GIGA第1期（2020〜2021年導入）の端末が4〜5年経過

対策：
  • 予備機15%の確保
  • バッテリー交換計画の策定
  • 教室内個別充電設備の整備（USB PD対応）
  • 保守契約の見直し

⚠️ 課題2：教職員のICT活用指導力の格差

実態：教員間でICTリテラシーの差が大きい

対策：
  • 段階的な研修プログラムの実施
  • ICT支援員の配置拡充
  • 好事例の横展開

⚠️ 課題3：OSサポート期限の管理

  • Chromebook：AUE（自動更新期限）の確認（機種ごとに異なる）
  • Windows：Windows 10のサポート終了（2025年10月）
  • 対策：計画的な更新スケジュールの策定

  • CPU: Intel Processor N100
  • RAM: 8GB
  • ストレージ: 128GB SSD (NVMe)
  • ディスプレイ: 11.6インチ FHD (1920×1080) タッチ対応
  • 重量: 1.25kg
  • バッテリー: 8時間

  • CPU: Intel Core i5-1235U
  • RAM: 16GB
  • ストレージ: 512GB SSD (NVMe)
  • ディスプレイ: 13.3インチ FHD (1920×1080) IPS タッチ対応
  • 重量: 1.35kg

演習2：既存端末のスペック調査

自校または支援校の既存端末について、以下の情報を調査してください。

【調査項目】

  • 端末の型番・メーカー
  • CPUの種類とクロック周波数
  • RAM容量
  • ストレージの種類（SSD/eMMC/HDD）と容量
  • OSのバージョン
  • 導入年度と使用年数
  • 現在のストレージ使用状況
  • 【追加】OSサポート期限（Chromebook AUE、Windows EOL）
  • 【追加】故障・修理の実績
  • 【追加】バッテリー駆動時間の劣化状況

【調査方法】

  • Windows: 「設定」→「システム」→「詳細情報」
  • Mac: Appleメニュー → 「このMacについて」
  • Chromebook: 「設定」→「Chrome OSについて」、AUEは管理コンソールで確認
  • iPad: 「設定」→「一般」→「情報」

【評価と提案】

  1. 現在の性能で今後3年間使用可能か評価
  2. 文科省GIGA第2期基準との比較
  3. 不足している点を指摘
  4. 改善策（RAM増設、ストレージ増設、買い替え、バッテリー交換）を提案
  5. 【追加】OSサポート期限到達時期と対応策

演習3：予算別端末選定プラン作成

  • 用途: 基本的な学習活動

【シナリオB：標準モデル】

  • 学校: 公立中学校
  • 予算: 1台あたり70,000円以内（周辺機器含む）
  • 台数: 200台（全学年） + 予備機30台（15%）= 合計230台
  • 用途: 標準的な学習活動 + 簡単なプログラミング
  • 総予算: 16,100,000円

【シナリオC：専門コース】

  • 学校: 高校（情報科専門コース）
  • 予算: 1台あたり150,000円以内（周辺機器含む）

【プランに含める内容】

  3. 総コスト内訳（端末+周辺機器+保守+5年間TCO）
  4. OSサポート期限の確認

  6. リスクと対策（故障率、バッテリー劣化、AUE到達時期）
  7. 【追加】予備機確保計画（15%の根拠）

【2025年版の重要ポイント】

  • Intel N100/N95の登場により、低価格帯でも高性能な端末が選択可能に
  • Chromebook AUE 10年への延長により、長期使用計画が立てやすく
  • GIGA第2期では、Wi-Fi 6、USB Type-C PDが標準化
  • 故障率の実態を踏まえた予備機15%確保が重要
  • TCOを意識した総合的なコスト管理が求められる
