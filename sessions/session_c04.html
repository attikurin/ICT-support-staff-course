中級編第4回

プログラミング教育の支援

所要時間: 90分 | レベル: 中級

📘 学習目標

  • 学習指導要領におけるプログラミング教育の位置づけを理解する
  • 各学年・教科で使用される代表的なプログラミング教材の特徴を説明できる
  • プログラミング的思考の育成方法を理解する
  • プログラミング授業における効果的な支援方法を習得する

1. プログラミング教育の基本

1.1 学習指導要領での位置づけ

💡 プログラミング教育の目的

2020年度から実施されている学習指導要領では、以下の3つの柱が示されています：

  • ① プログラミング的思考の育成
  • ② プログラムの働きやよさへの気づき
  • ③ 各教科での学びをより確実にする

※プログラミング言語の習得が目的ではありません

1.2 プログラミング的思考とは

🎯 プログラミング的思考の定義

「自分が意図する一連の活動を実現するために、どのような動きの組合せが必要であり、一つ一つの動きに対応した記号を、どのように組み合わせたらいいのか、記号の組合せをどのように改善していけば、より意図した活動に近づくのか、といったことを論理的に考えていく力」

具体的には：

  • 分解：問題を小さな部分に分ける
  • 抽象化：共通点を見つけてパターン化
  • 順序立て：手順を整理して並べる
  • 試行錯誤：うまくいくまで改善を繰り返す

2. 学年・教科別プログラミング教材

2.1 小学校低学年向け教材

教材名 特徴 推奨学年
Viscuit（ビスケット） 絵を描いて動かす、文字入力不要 小1〜3
プログラミングゼミ ブロックを並べる、キャラクターを動かす 小1〜3
ScratchJr タブレット向け、直感的操作 小1〜2
Code.org ゲーム感覚、段階的学習 小1〜

2.2 小学校中・高学年向け教材

  • Scratch（スクラッチ）
    • 最も広く使われているビジュアルプログラミング言語
    • ブロックを組み合わせてプログラム作成
    • ゲーム、アニメーション、物語など多様な作品制作
    • 推奨：小3〜中学生
  • micro:bit（マイクロビット）
    • 教育用小型コンピューター
    • センサー、LED、ボタンを使った実践的プログラミング
    • 理科・算数との連携がしやすい
    • 推奨：小5〜中学生
  • プログル
    • 教科学習に特化したプログラミング教材
    • 算数「多角形」、理科「電気の利用」など
    • 文部科学省の実証事業で使用

2.3 中学校向け教材

  • 技術・家庭科（技術分野）
    • 「D 情報の技術」でプログラミング必修
    • Scratch、Python、Javascriptなど
    • 計測・制御プログラミング（Arduino、micro:bitなど）
  • JavaScript / Python
    • テキストベースのプログラミング言語
    • より実践的なプログラミング学習
    • Webサイトやアプリケーションの作成

📌 教材選定のポイント

  • 学年・発達段階に応じた難易度
  • 教科の学習内容との関連性
  • 使用機器（PC、タブレット、ロボット）
  • 教員の習熟度とサポート体制
  • 無料 or 有料、オンライン or オフライン

3. 教科でのプログラミング実践例

3.1 算数「正多角形」（小学5年生）

🎯 授業の流れ

  1. 導入：正多角形の性質を復習
  2. プログラミング体験：Scratchで正三角形を描くプログラム作成
  3. 法則の発見：辺の数と角度の関係に気づく
  4. 応用：正方形、正五角形、正六角形を描く
  5. まとめ：プログラミングを通じた数学的理解の深化

使用教材例：プログル（多角形コース）、Scratch

3.2 理科「電気の利用」（小学6年生）

学習内容：

  • 電気を効率よく使うための工夫
  • センサーを使った自動制御
  • 条件分岐（明るさセンサー、人感センサー）

使用教材例：micro:bit、プログル（電気コース）

3.3 総合的な学習の時間

テーマ例：

  • 地域の魅力発信：Scratchでクイズやゲーム作成
  • 環境問題：データ収集とグラフ化プログラム
  • ロボット制御：レゴマインドストーム、MESHなど

3.4 音楽「リズム・パターン」

  • Scratchで音楽を作曲
  • 繰り返し（ループ）の理解
  • 音の長さ、高さのパラメータ調整

3.5 図工「アニメーション制作」

  • Scratchでオリジナルアニメーション
  • ストーリー構成と表現
  • キャラクターの動きの工夫

4. アンプラグドプログラミング

4.1 アンプラグドとは

💡 アンプラグドプログラミング

コンピューターを使わずに、プログラミング的思考を育成する活動です。

メリット：

  • 端末台数が不足している場合でも実施可能
  • 低学年でも取り組みやすい
  • 身体を動かしながら学べる
  • グループ活動で協働的に学べる

4.2 具体的な活動例

  • ロボット人間
    • 一人がロボット役、一人がプログラマー役
    • 「3歩前に進む」「右に90度回転」などの指示カードを並べる
    • ロボット役がその通りに動く
  • お掃除ロボット
    • 床に敷いたマス目の上を動く
    • 矢印カードで動きを指示
    • 最短ルートを考える
  • 順次・分岐・反復カード
    • 朝の準備や給食の配膳をフローチャートで表現
    • 「もし〜なら」の条件分岐を考える
    • 繰り返しの処理を見つける

5. プログラミング授業のトラブル対応

5.1 よくあるトラブルと対処法

トラブル 原因 対処法
Scratchが動かない ネット接続不良、ブラウザの問題 Scratchデスクトップ版を使用、ブラウザ再起動
プログラムが意図通りに動かない ブロックの組み方のミス デバッグの方法を教える（一つずつ実行して確認）
作品が保存できない 未ログイン、容量不足 ログイン確認、ローカル保存、クラウド容量確認
micro:bitが認識されない USBケーブルの接触不良 ケーブル交換、再接続、別のポート試行

5.2 児童生徒の困りごと対応

⚠️ よくある児童生徒の悩み

  • 「どうやって始めればいいか分からない」
    • → 小さな目標を設定、段階的に進める
  • 「エラーが出て動かない」
    • → デバッグの方法を教える、一緒に原因を探す
  • 「友達のようにうまくできない」
    • → 比較ではなく、自分の成長に目を向けさせる
  • 「もっと難しいことがしたい」
    • → 発展課題の提示、自由制作の機会提供

6. ICT支援員の役割

6.1 授業前の準備

  • 環境整備：Scratchアカウント作成、micro:bit動作確認
  • サンプルプログラム準備：参考作品、テンプレート
  • 教員への操作説明：教材の基本操作レクチャー
  • トラブル対応マニュアル作成：よくある質問集

6.2 授業中のサポート

  • デモンストレーション：基本操作の実演
  • 個別サポート：つまずいている児童生徒への助言
  • 技術的トラブル対応：接続不良、動作不良の解決
  • 発展課題の提示：早く終わった児童生徒への対応

🎯 サポートのポイント

  • 答えを教えすぎない：ヒントを出して自分で考えさせる
  • 試行錯誤を促す：失敗を恐れず挑戦する雰囲気づくり
  • 創造性を尊重：正解は一つではないことを伝える
  • 共有を推奨：友達の作品から学ぶ機会を提供

6.3 教員研修のサポート

  • プログラミング教材の研修会開催
  • 指導案作成のサポート
  • 他校の実践事例の情報提供
  • 教員同士の情報共有の場づくり

📝 演習問題

演習1: 教材選定

以下の授業に最も適したプログラミング教材を選び、理由を説明してください。

  1. 小学1年生 生活科「おもちゃづくり」
  2. 小学4年生 算数「垂直・平行と四角形」
  3. 小学6年生 理科「電気の利用」

演習2: トラブルシナリオ

状況：プログラミング授業中、複数の児童から「Scratchが動かない」「保存できない」と同時に相談がありました。教員一人では対応しきれない状況です。ICT支援員としてどのように対応しますか？

  1. 優先順位をどのように判断しますか？
  2. 教員とどのように役割分担しますか？
  3. 再発防止のためにできることは何ですか？

演習3: 授業設計サポート

教員から「初めてプログラミング授業をやるので不安。何を準備すればいいか」と相談されました。小学5年生の算数「正多角形」の単元でScratchを使う予定です。ICT支援員として、どのようなサポートを提案しますか？

📌 まとめ

  • プログラミング教育の目的は「プログラミング的思考」の育成であり、言語習得ではない
  • 学年・教科に応じて適切なプログラミング教材を選定する必要がある
  • Scratch、micro:bit、Viscuitなど、多様な教材が存在する
  • アンプラグドプログラミングはコンピューターなしでも実施可能
  • ICT支援員は技術サポートだけでなく、児童生徒の創造性を引き出す支援が重要
