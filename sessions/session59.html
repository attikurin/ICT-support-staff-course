1.1 クライアント・サーバーモデル

基本概念

教育現場で使用する多くのアプリケーション（Google Classroom、Teams、学習支援ソフト等）は、クライアント・サーバーモデルで動作しています。

クライアント・サーバーモデルの構造
【クライアント】          【サーバー】
 (あなたの端末)           (データセンター)
      │                        │
      │  ①リクエスト送信       │
      │ ──────────────────→    │
      │   "課題一覧を表示"      │
      │                        │
      │  ②レスポンス返信       │
      │ ←──────────────────    │
      │   "データを送信"        │
教育現場での具体例
  • Google Classroom
    • クライアント：教員・生徒のブラウザ/アプリ
    • サーバー：Googleのデータセンター
    • やりとり：課題の配信、提出物の管理、成績の記録
  • 学習支援アプリ（Qubena、スタディサプリ等）
    • クライアント：生徒のタブレット/PC
    • サーバー：学習データを保管・分析
    • やりとり：問題の配信、解答の送信、進捗の記録

1.2 HTTP/HTTPSプロトコル

HTTP（HyperText Transfer Protocol）

Webの世界で情報をやりとりするための「ルール」です。

主なHTTPメソッド
  • GET - データを「取得」する（例：課題一覧を表示）
  • POST - データを「送信」する（例：課題を提出）
  • PUT - データを「更新」する（例：解答を修正）
  • DELETE - データを「削除」する（例：下書きを削除）
HTTPS（HTTP Secure）

HTTPに暗号化を追加したもの。教育現場では個人情報保護のため、HTTPSを使うことが必須です。

  • 🔒 URLが「https://」で始まる
  • 🔒 ブラウザのアドレスバーに鍵マーク
  • 🔒 通信内容が暗号化されている

3.3 ユーザー認証

ユーザー認証の重要性

教育現場では、生徒の個人情報や成績データを扱うため、適切な認証が必須です。

基本的な認証の流れ
  1. 登録 - ユーザーIDとパスワードを登録
  2. ログイン - 登録情報と照合
  3. セッション管理 - ログイン状態を保持
  4. 権限管理 - 教員と生徒で見られる情報を分ける
Flaskでの簡単なログイン機能
from flask import Flask, session, redirect, url_for, request
from werkzeug.security import generate_password_hash, check_password_hash

app = Flask(__name__)
app.secret_key = 'your-secret-key-here'

# 仮のユーザーデータベース
users = {
    'teacher1': generate_password_hash('password123'),
    'student1': generate_password_hash('student456')
}

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        username = request.form['username']
        password = request.form['password']
        
        if username in users and check_password_hash(users[username], password):
            session['username'] = username
            return redirect(url_for('dashboard'))
        else:
            return 'ログイン失敗'
    
    return '''
        <form method="post">
            <input name="username" placeholder="ユーザー名">
            <input name="password" type="password" placeholder="パスワード">
            <button type="submit">ログイン</button>
        </form>
    '''

@app.route('/dashboard')
def dashboard():
    if 'username' in session:
        return f'ようこそ、{session["username"]}さん！'
    return redirect(url_for('login'))

@app.route('/logout')
def logout():
    session.pop('username', None)
    return redirect(url_for('login'))

function onFormSubmit(e) {
  // フォームの回答を取得
  const responses = e.values;
  const timestamp = responses[0];  // タイムスタンプ
  const name = responses[1];        // 生徒名
  const status = responses[2];      // 出席/欠席/遅刻
  
  // スプレッドシートに記録
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('出席記録');
  sheet.appendRow([timestamp, name, status]);
  
  // 遅刻・欠席の場合、教員にメール通知
  if (status !== '出席') {
    MailApp.sendEmail({
      to: 'teacher@school.jp',
      subject: `【出席確認】${name}さんが${status}`,
      body: `${name}さんが${timestamp}に${status}を報告しました。`
    });
  }
}
実践例2：課題の自動集計とグラフ作成
function createScoreSummary() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('成績一覧');
  const data = sheet.getDataRange().getValues();
  
  // 1行目（ヘッダー）を除く
  const scores = data.slice(1).map(row => row[2]);  // 3列目が点数
  
  // 統計を計算
  const average = scores.reduce((a, b) => a + b, 0) / scores.length;
  const maxScore = Math.max(...scores);
  const minScore = Math.min(...scores);
  
  // サマリーシートに書き込み
  const summarySheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('サマリー');
  summarySheet.getRange('A1').setValue('平均点');
  summarySheet.getRange('B1').setValue(average.toFixed(1));
  summarySheet.getRange('A2').setValue('最高点');
  summarySheet.getRange('B2').setValue(maxScore);
  summarySheet.getRange('A3').setValue('最低点');
  summarySheet.getRange('B3').setValue(minScore);
  
  Logger.log(`平均点: ${average.toFixed(1)}`);
}

第5部：公開とデプロイ（20分）

5.1 GitHub Pagesの利用

GitHub Pagesとは

GitHubで管理しているWebサイトを無料で公開できるサービスです。

利用条件
  • ✅ 静的なWebサイト（HTML/CSS/JavaScript）が対象
  • ✅ GitHubアカウントが必要
  • ✅ 無料で使える
  • ⚠️ 公開される情報は誰でも閲覧可能（個人情報は掲載しない）
GitHub Pagesで教材サイトを公開する手順
  1. GitHubリポジトリを作成
    • GitHub（https://github.com/）にログイン
    • 「New repository」をクリック
    • リポジトリ名を入力（例：math-quiz）
    • 「Public」を選択
  2. ファイルをアップロード
    • 「Add file」→「Upload files」
    • HTML、CSS、JavaScriptファイルをドラッグ&ドロップ
    • 「Commit changes」をクリック
  3. GitHub Pagesを有効化
    • リポジトリの「Settings」タブを開く
    • 左メニューから「Pages」を選択
    • 「Source」で「main」ブランチを選択
    • 「Save」をクリック
  4. 公開URLを確認
    • https://[ユーザー名].github.io/[リポジトリ名]/
    • このURLを生徒に共有

5.2 セキュリティの考慮

教育現場でWebアプリを公開する際の注意点
  1. 個人情報の保護
    • ⚠️ 生徒の氏名、住所、電話番号などは公開しない
    • ⚠️ 成績データを公開する場合は、匿名化する
    • ⚠️ ログイン機能がない場合、機密情報は掲載しない
  2. HTTPS通信
    • ✅ GitHub Pagesは自動的にHTTPSに対応
    • ✅ 他のサーバーを使う場合も、必ずHTTPSを使用
  3. アクセス制限
    • 重要なデータは、ログイン機能で保護
    • 学校内ネットワークからのみアクセス可能にする
    • パスワードは複雑なものを設定
  4. 著作権と肖像権
    • 他人の画像や文章を無断で使用しない
    • 生徒の写真を公開する場合は保護者の同意を得る
    • 教材の引用は出典を明記

演習2：Google Apps Scriptで校務自動化（難易度：★★★☆☆）

課題内容

Google Formsで集めた欠席連絡を自動処理するスクリプトを作成してください。

要件
  • Google Formsで欠席連絡フォームを作成
  • 項目：生徒名、学年、クラス、欠席理由
  • 回答があったら自動でスプレッドシートに記録
  • 担任の先生にメールで通知
  • 欠席者数をカウントして別シートに集計
手順
  1. Google Formsを作成
  2. スプレッドシートと連携
  3. スクリプトエディタを開く（拡張機能 → Apps Script）
  4. onFormSubmit関数を作成
  5. トリガーを設定（フォーム送信時に実行）

演習3：Glideで学習記録アプリ作成（難易度：★★☆☆☆）

課題内容

Glideを使って、生徒が自分の学習時間を記録できるアプリを作成してください。

要件
  • Googleスプレッドシートにデータを記録
  • 項目：日付、科目、学習時間（分）、メモ
  • 記録の追加、編集、削除ができる
  • 科目ごとの合計時間を表示
  • グラフで学習時間の推移を可視化
手順
  1. Googleスプレッドシートを作成（列：日付、科目、時間、メモ）
  2. Glide（https://www.glideapps.com/）にアクセス
  3. 「New App」→「From Google Sheet」
  4. レイアウトを「List」に設定
  5. 「Add Item」ボタンを追加
  6. Computed Columnで合計時間を計算

💡 指導のポイント（ICT支援員・講師向け）

学習者がつまずきやすいポイントと対策

1. つまずきポイント：HTML/CSS/JavaScriptの役割の混同

対策：

  • 「家の建築」で例える（HTML=骨組み、CSS=内装、JavaScript=電気設備）
  • 同じページを3つの言語で段階的に作成し、違いを体感させる
  • 最初は1つの技術に集中し、徐々に組み合わせる
2. つまずきポイント：タグの閉じ忘れ・文法エラー

対策：

  • ブラウザの開発者ツール（F12）でエラーを確認する習慣をつける
  • インデント（字下げ）を揃えて、構造を見やすくする
  • Visual Studio CodeなどのエディタのHTML補完機能を活用
3. つまずきポイント：JavaScriptの変数とスコープ

対策：

  • letとconstの使い分けを具体例で説明
  • 関数の内外でのアクセス可否を図解する
  • console.log()で変数の中身を確認する習慣づけ
4. つまずきポイント：Google Apps Scriptのトリガー設定

対策：

  • 手動実行で動作確認してからトリガーを設定
  • 「現在のプロジェクトのトリガー」画面のスクリーンショットを共有
  • トリガーが発火しない場合、実行ログを確認する方法を教える
5. つまずきポイント：Glideのデータ連携

対策：

  • スプレッドシートの1行目（ヘッダー）が重要であることを強調
  • データ型（テキスト、数値、日付）の適切な設定
  • 「Reload data」で最新のスプレッドシートを反映
