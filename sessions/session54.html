📚 第1章：ハードウェアアーキテクチャ

1.1 フォン・ノイマン型アーキテクチャ

現代のほぼすべてのコンピュータは、数学者ジョン・フォン・ノイマンが1945年に提唱した「フォン・ノイマン型アーキテクチャ」に基づいて設計されています。

1.1.1 基本概念

フォン・ノイマン型アーキテクチャの最も重要な特徴は以下の2点です：

  • プログラム内蔵方式(Stored Program)：プログラム(命令列)とデータを同一のメモリ空間に格納する
  • 逐次制御方式(Sequential Control)：メモリに格納された命令を順番に読み出して実行する

この設計により、プログラムを外部から読み込んで実行することが可能となり、現代の汎用コンピュータの基礎が確立されました。

1.1.2 5つの構成要素

フォン・ノイマン型コンピュータは、以下の5つの基本要素で構成されます：

フォン・ノイマン型アーキテクチャの5大要素
構成要素 英語名称 主な役割 学校ICT環境での具体例
演算装置 Arithmetic Logic Unit (ALU) 算術演算(加減乗除)と論理演算(AND/OR/NOT)を実行 CPUのALU部分:計算処理、画像処理、データ分析
制御装置 Control Unit (CU) 命令の解釈と実行順序の制御、各装置への指令送信 CPUの制御部:プログラムの流れ制御、タイミング調整

1.1.3 フォン・ノイマン・ボトルネック

フォン・ノイマン型アーキテクチャには構造的な制約があります：

  • 問題点：CPUとメモリ間のデータ転送速度(バス帯域幅)が、CPU処理速度に追いつかない
  • 影響：CPUが高速化してもメモリからのデータ供給が間に合わず、性能向上が頭打ちになる
対策技術
  • キャッシュメモリ：CPU内部に高速小容量メモリ(L1/L2/L3キャッシュ)を配置し、頻繁に使うデータを保持
  • パイプライン処理：命令実行を複数ステージに分割し並列化
  • マルチコアCPU：複数のCPUコアで並列処理
  • 高速バス：PCIe 4.0/5.0、DDR4/DDR5など高帯域幅インターフェース
🏫 学校現場での重要性

大量の生徒データ処理やマルチメディアコンテンツ再生時に、このボトルネックが顕在化します。機器選定時には、CPU性能だけでなくメモリ速度・容量、ストレージ転送速度のバランスが重要です。

1.2 CPU(中央処理装置)の内部構造

1.2.1 CPUの基本構成

CPU(Central Processing Unit)は、「コンピュータの頭脳」として、すべての演算と制御を担当します。

各要素の詳細：
  • 制御装置(CU)：
    • プログラムカウンタ(PC): 次に実行する命令のメモリアドレスを保持
    • 命令レジスタ(IR): 現在実行中の命令を保持
    • 命令デコーダ: 機械語命令を解釈し、制御信号を生成
  • 演算装置(ALU)：
    • 算術演算: 加算、減算、乗算、除算
    • 論理演算: AND、OR、NOT、XOR
    • シフト演算: ビット単位の左シフト・右シフト
    • 比較演算: 大小比較、等号判定
  • キャッシュメモリ階層：
    • L1キャッシュ: CPUコア直結、最速(1-2クロック)、容量小(32-64KB)
    • L2キャッシュ: コア専用または共有、中速(数クロック)、容量中(256KB-1MB)
    • L3キャッシュ: 全コア共有、やや遅い(十数クロック)、容量大(8-64MB)

1.2.2 命令実行サイクル

CPUは以下の4段階サイクル(フェッチ→デコード→実行→ライトバック)を高速で繰り返してプログラムを実行します。

命令実行サイクルの4ステージ
 ステージ1      ステージ2      ステージ3      ステージ4
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│          │ │          │ │          │ │          │
│ Fetch    │→│ Decode   │→│ Execute  │→│ Write-   │
│ (取得)   │ │ (解読)   │ │ (実行)   │ │  Back    │
│          │ │          │ │          │ │ (書戻し) │
└──────────┘ └──────────┘ └──────────┘ └──────────┘
     │            │            │            │
     ↓            ↓            ↓            ↓
メモリから    命令を解釈    演算実行      結果を
命令読出し    制御信号生成  ALUで処理    レジスタ/
PCインクリ    オペランド    条件分岐判定  メモリへ
メント        取得準備                    書込み
各ステージの詳細：
  1. Fetch(フェッチ/命令取得)：プログラムカウンタ(PC)が指すメモリアドレスから命令を読み出し、命令レジスタ(IR)に格納。PCをインクリメント。
  2. Decode(デコード/命令解読)：命令レジスタの内容を命令デコーダで解析。オペコード(操作の種類)とオペランド(データ/アドレス)を識別。
  3. Execute(実行)：ALUで算術・論理演算を実行。メモリアクセス(Load/Store)を実行。条件分岐判定。
  4. Write-Back(ライトバック/結果書込み)：演算結果をレジスタに書き戻す。または、メモリへ書き込む。フラグレジスタを更新。
パイプライン処理による高速化

現代のCPUは、複数の命令を同時に異なるステージで処理する「パイプライン処理」を採用しています：

時刻 →  T1    T2    T3    T4    T5    T6    T7    T8
命令1  [F]   [D]   [E]   [W]
命令2        [F]   [D]   [E]   [W]
命令3              [F]   [D]   [E]   [W]
命令4                    [F]   [D]   [E]   [W]
命令5                          [F]   [D]   [E]   [W]

F=Fetch, D=Decode, E=Execute, W=Write-Back

これにより、理論上は毎クロックで1命令を完了できます(スーパースカラ設計では複数命令/クロック)。

📚 第2章：主要コンポーネント

2.1 マザーボード(Motherboard)

マザーボードは、コンピュータのすべてのハードウェアコンポーネントを接続する「基板」であり、電源供給、データ通信、制御信号伝達を担います。

主要構成要素

  • CPUソケット：CPUを物理的に搭載する場所（Intel: LGA1700、AMD: AM5等）
  • メモリスロット(DIMM)：RAMモジュールを挿入（DDR4/DDR5対応）
  • チップセット：CPUと周辺デバイスの間でデータ転送を仲介
  • 拡張スロット：PCIe x16(GPU用)、PCIe x1(拡張カード用)
  • ストレージ接続：M.2スロット(NVMe SSD)、SATAポート(HDD/SSD)
  • I/Oパネル：背面外部接続端子（USB、HDMI、LAN、Audio等）

フォームファクター（サイズ規格）

規格 サイズ(mm) 拡張スロット数 用途
ATX 305×244 最大7スロット デスクトップPC、高拡張性
Micro-ATX 244×244 最大4スロット 学校標準PC推奨
Mini-ITX 170×170 1スロットのみ 超小型PC、デジタルサイネージ

2.2 チップセット(Chipset)

チップセットは、CPUと周辺デバイス(メモリ、ストレージ、USB等)の間でデータ転送を仲介する重要な集積回路です。

現代のチップセット構成

現代では多くの機能がCPUに統合され、単一チップセット(PCH: Platform Controller Hub)となっています：

  • CPUが直接管理：メモリコントローラ、PCIe レーン×16〜20（GPU直結）
  • チップセットが管理：追加のPCIeレーン、SATAポート、USBコントローラ、ネットワーク、オーディオ
  • DMI(Direct Media Interface)：CPUとチップセット間の専用高速リンク（DMI 4.0: 8GB/s）

🏫 学校現場での選定基準

  • 教員PC/管理PC：B760(Intel)またはB650(AMD)で必要十分
  • サーバー：Z790/X670E（多数の拡張デバイス接続が必要な場合）

2.3 拡張スロット(Expansion Slots)

2.3.1 PCIe(PCI Express)規格

PCIeは、現代の高速拡張バス規格です。レーン数(x1, x4, x8, x16)で帯域幅が変わります。

世代別転送速度（片方向/レーンあたり）
世代 1レーンあたり x16スロット 登場年
PCIe 3.0 約1GB/s 約16GB/s 2010年
PCIe 4.0 約2GB/s 約32GB/s 2017年

下位互換性：PCIe 4.0カードはPCIe 3.0スロットでも動作（速度は遅い方に合わせる）

2.4 電源ユニット(PSU: Power Supply Unit)

電源ユニットは、家庭用AC 100V(日本)をコンピュータ内部で使用するDC(+12V, +5V, +3.3V)に変換します。

電源容量の計算

推奨電源容量 = 合計消費電力 × 1.5〜2.0

標準的な学校PCの例：
  • CPU(Core i5): 65W
  • GPU(内蔵): 15W
  • マザーボード: 50W
  • RAM(8GB×2): 10W
  • SSD(NVMe): 5W
  • その他: 30W
  • 合計: 175W → 推奨電源: 450W

80PLUS認証と効率

80PLUSは、電源変換効率(AC→DC)を保証する国際認証です：

  • 80PLUS Silver：効率85-88%（学校標準PC推奨）
  • 80PLUS Gold：効率87-90%（サーバー/ワークステーション）
  • 80PLUS Bronze：効率82-85%（一般PC）

効率90%の電源で100W供給する場合、111W(100W÷0.9)をコンセントから消費します。差の11Wは熱として放出されます。

2.5 冷却システム(Cooling System)

コンピュータ内部の熱管理は、性能維持と寿命延長に直結します。

CPUクーラーの構造

  • サーマルグリス：CPUとクーラー間の微小な隙間を埋め、熱抵抗を低減。2〜3年で再塗布推奨。
  • ヒートパイプ：内部の液体が蒸発・凝縮を繰り返して熱を運ぶ。熱伝導率が銅の数十倍。
  • ヒートシンク：表面積を増やして空気へ熱を放出（アルミ製または銅製）
  • 冷却ファン：強制対流で放熱効率向上。PWM制御で回転数調整。

温度管理基準

状態 Intel Core AMD Ryzen 対処
アイドル 30〜50℃ 35〜50℃ 正常
通常負荷 50〜75℃ 50〜75℃ 正常
高負荷 75〜85℃ 75〜85℃ 正常（継続監視）
注意 85〜100℃ 85〜95℃ 冷却改善推奨
危険 100℃ 95℃ サーマルスロットリング

🏫 学校現場での温度監視

  • 推奨ツール：HWMonitor, Open Hardware Monitor, Core Temp
  • 定期点検：夏季前(5月)と冬季前(11月)に温度ログ確認
  • 警告閾値設定：CPU 85℃超で管理者へ通知

2.8 ストレージの実装

ストレージは、データを長期保存する不揮発性記憶装置です。

学校端末で使用されるストレージ比較

種類 読込速度 書込速度 寿命 価格 学校での使用
NVMe SSD(PCIe 4.0) 7000MB/s 5000MB/s 600〜1200TBW 高 教員PC、サーバー
NVMe SSD(PCIe 3.0) 3500MB/s 3000MB/s 600TBW 中 現行GIGA端末主流
SATA SSD 550MB/s 520MB/s 300TBW 低 旧型PC換装

⚠️ ストレージ容量不足の診断と対処

症状
  • 「空き容量が不足しています」という警告
  • アプリのインストール・更新ができない
  • 動作が極端に遅くなる
対処法
  1. 一時ファイル削除：Windows: ディスククリーンアップ実行、Chrome OS: キャッシュクリア
  2. 不要アプリ削除：使用していないアプリをアンインストール
  3. 大きなファイルの特定：WinDirStat(Windows)、DiskUsage(Chrome OS)で容量消費の大きいファイル/フォルダを特定
  4. クラウド移行：Google Drive、OneDriveへファイル移動
  5. 最終手段：ストレージ交換・増設(可能な機種の場合)

GIGA端末の推奨容量：128GB以上(256GB推奨)。64GB以下は運用上支障が出やすい。

📚 第3章：入出力インターフェース

3.1 USB(Universal Serial Bus)

3.1.1 USB規格の変遷と転送速度

USBは世代ごとに性能が向上していますが、命名規則が複雑なため注意が必要です。

USB規格一覧表
マーケティング名称 技術仕様名 最大転送速度 登場年
USB 2.0 USB 2.0 480Mbps (60MB/s) 2000年
USB 3.0 / USB 3.1 Gen 1 USB 3.2 Gen 1 5Gbps (625MB/s) 2008年
USB 3.1 / USB 3.1 Gen 2 USB 3.2 Gen 2 10Gbps (1.25GB/s) 2013年
USB4 USB4 40Gbps (5GB/s) 2019年

重要な注意点：

  • "USB 3.0"と"USB 3.1 Gen 1"は同一(5Gbps)
  • USB4以降はType-Cコネクタのみ
  • Type-Cコネクタは上下対称(リバーシブル)、最大100W給電

3.2 映像出力インターフェース

3.2.1 HDMI

HDMI規格比較
規格 最大帯域幅 解像度・リフレッシュレート 登場年
HDMI 1.4 10.2Gbps 4K@30Hz / 1080p@120Hz 2009年
HDMI 2.0 18Gbps 4K@60Hz / 1440p@144Hz 2013年
HDMI 2.1 48Gbps 4K@120Hz / 8K@60Hz 2017年

学校現場での用途：プロジェクター接続（最も普及）、電子黒板・大型ディスプレイ接続

3.2.2 DisplayPort

DisplayPortの特徴：

  • PC向け高性能規格(HDMIより高帯域幅)
  • MST(Multi-Stream Transport)：1つのDP出力から複数モニターへ接続可能(デイジーチェーン)
  • Adaptive-Sync：可変リフレッシュレート対応

学校現場での用途：教員用高解像度モニター(4K以上)、マルチディスプレイ環境、動画編集用ワークステーション

3.2.3 Thunderbolt

Thunderboltの特徴：

  • オールインワン: データ+映像(DisplayPort)+電源+PCIeを1本で伝送
  • Thunderbolt 4: 40Gbps、最大100W給電、DP 1.4対応
  • デイジーチェーン: 最大6デバイスまで数珠つなぎ接続
  • 外付けGPU(eGPU)対応: PCIe接続で外部グラフィックカード使用可能

⚠️ 注意：すべてのUSB Type-CポートがThunderboltではない。⚡マーク確認必須。

学校現場での用途：Apple環境(MacBook + ドック)、高速ストレージ、外付けGPU

5.1 起動しない(電源が入らない)

    │           ├→ 電源ユニット故障確認

原因別対処法
  • コンセント・電源ケーブル：別機器で通電確認、ケーブル交換
  • 電源ユニット故障：ペーパークリップテスト実施、PSU交換

  • マザーボード故障：ビープコード確認、最小構成起動テスト、保証期間確認

5.2 画面に何も表示されない(電源は入る)

確認順序
  1. モニター電源LED確認(緑=通電、橙=スタンバイ)
  2. モニター入力切替確認(HDMI1/HDMI2等)
  3. ケーブル抜き差し(両端)
  4. 別のケーブル・別のモニターで試行
  5. ビープ音確認（長音連続→メモリ問題、短音複数回→GPU問題）
  6. メモリ抜き差し、1枚ずつ起動テスト
  7. GPU取り外し、マザーボード側映像出力で試行

5.3 異音がする

音の種類別対処
音の種類 発生源 緊急度 対処法
カタカタ音 HDD ★★★★★(最高) 即座にシャットダウン、データバックアップ、HDD交換
キーン音(高音) コイル鳴き ★☆☆☆☆(低) 動作に支障なければ継続使用可、保証期間内なら交換申請
ゴォー音(大音量) 冷却ファン ★★★☆☆(中) ファン清掃、軸受け劣化ならファン交換

5.4 発熱・オーバーヒート

冷却改善手順
ステップ1: 内部清掃
  • エアダスターでホコリを吹き飛ばす（ファンを手で押さえながら噴射）
  • 吸気フィルター取り外し、水洗い乾燥
  • 配線整理(エアフロー改善)

清掃頻度推奨：教室PC 3ヶ月に1回、職員室PC 6ヶ月に1回、サーバー 1ヶ月に1回

ステップ2: サーマルグリス再塗布

2年以上交換していない場合、サーマルグリスが劣化している可能性あり。CPU中央に米粒大(0.1ml程度)塗布。

ステップ3: ファン増設・交換

前面吸気ファン増設、天面排気ファン追加、または大型空冷クーラー・簡易水冷へアップグレード。

📚 第7章：実践演習

7.1 PC分解・組立て演習

演習手順（180分）
  1. 分解フェーズ(60分)：外部ケーブル取り外し → サイドパネル開放 → 拡張カード取り外し → メモリ/ストレージ/CPUクーラー/電源ユニット/マザーボード取り外し
  2. 観察・記録フェーズ(30分)：各部品の型番・仕様を記録、接続インターフェース確認、内部のホコリ堆積状況観察
  3. 組立てフェーズ(60分)：分解と逆順で組み立て、ケーブル接続の確認、ネジの締め忘れチェック
  4. 動作確認フェーズ(30分)：電源投入、POST通過確認、BIOS起動、全コンポーネント認識確認、OS起動確認
📋 分解前チェックリスト
  • ☐ 静電気対策完了（リストストラップ装着または金属製品に触れて放電）
  • ☐ 作業場所の整理整頓（明るく、広い作業台を確保）
  • ☐ 必要工具の準備（プラスドライバー#1/#2、マイナスドライバー、ピンセット、懐中電灯）
  • ☐ 部品保管容器の準備（ネジ・小部品用の小皿やトレイ）
  • ☐ 記録用具の準備（カメラ、メモ用紙、ラベルシール）
  • ☐ 安全確認（電源ケーブル抜線、電源ユニットスイッチOFF）
⚠️ 分解作業の注意事項
  • 力を入れすぎない：コネクタは「カチッ」という音がするまで引く。無理に引っ張るとピンが折れる。
  • ネジの混同防止：取り外したネジは、取り外した部品の近くに置く。型番と長さが異なるネジが混在。
  • ケーブルの向き：コネクタには向きがある。無理に押し込まない。
  • CPUクーラー取り外し：サーマルグリスが固着しているため、ゆっくり回転させながら外す。
  • 静電気破壊：特にメモリ、CPU、マザーボードは静電気に弱い。金属部分を持つ。

📚 第9章：用語集

本講座で使用する主要な技術用語を五十音順に整理しました。

🔤 ア行

アーキテクチャ(Architecture)
コンピュータの基本設計思想と構造。フォン・ノイマン型が代表例。
演算装置(ALU: Arithmetic Logic Unit)
CPU内部で算術演算(加減乗除)と論理演算(AND/OR/NOT)を実行する回路。
eMMC(embedded Multi Media Card)
フラッシュメモリとコントローラを一体化した小型ストレージ。低速だがコストが安い。

🔤 カ行

キャッシュメモリ(Cache Memory)
CPUとメモリの速度差を埋めるための高速小容量メモリ。L1/L2/L3の階層構造。
クロック周波数(Clock Frequency)
CPUの動作速度を示す指標。単位はGHz(ギガヘルツ)。1GHz=10億回/秒。
コア(Core)
CPUの演算実行ユニット。マルチコアCPUは複数の命令を同時実行可能。

🔤 サ行

サーマルグリス(Thermal Grease)
CPUとクーラーの接触面に塗布する熱伝導性ペースト。熱抵抗を低減。
サーマルスロットリング(Thermal Throttling)
CPU温度上昇時に自動的にクロック周波数を下げて発熱を抑制する保護機能。
制御装置(CU: Control Unit)
CPU内部で命令の解釈と実行順序を管理し、各装置へ制御信号を送る回路。

🔤 タ行

チップセット(Chipset)
CPUと周辺デバイス(メモリ、ストレージ、USB等)の間でデータ転送を仲介する集積回路。
TDP(Thermal Design Power)
熱設計電力。CPUが通常動作時に発生する熱量の目安。単位はW(ワット)。
TBW(Total Bytes Written)
SSDの総書き込み量の保証値。例: 600TBW = 600テラバイトまで書き込み可能。

🔤 ナ行

NVMe(Non-Volatile Memory Express)
PCIe接続SSD向けの高速通信プロトコル。SATA(600MB/s)より10倍以上高速。

🔤 ハ行

パイプライン処理(Pipelining)
複数の命令を異なるステージで同時処理する高速化技術。理論上は毎クロック1命令完了。
BIOS/UEFI(Basic Input/Output System / Unified Extensible Firmware Interface)
マザーボードのROMに格納されたファームウェア。ハードウェア初期化とOS起動を担当。
POST(Power-On Self Test)
電源投入直後にBIOS/UEFIが実行するハードウェア診断テスト。
フォン・ノイマン型アーキテクチャ(Von Neumann Architecture)
プログラムとデータを同一メモリに格納し、逐次実行する方式。現代コンピュータの基本設計。

🔤 マ行

マザーボード(Motherboard)
コンピュータの全ハードウェアを接続する基板。別名: メインボード。
メモリコントローラ(Memory Controller)
CPUとメモリ間のデータ転送を管理する回路。現代CPUはCPU内蔵が主流。

🔤 ラ行

RAM(Random Access Memory)
読み書き可能な揮発性メモリ。電源OFF時にデータ消失。主記憶装置として使用。
レジスタ(Register)
CPU内部の超高速小容量記憶回路。演算データや命令アドレスを一時保持。

🔤 アルファベット・数字

80PLUS認証
電源ユニットの変換効率(AC→DC)を保証する国際認証。Bronze/Silver/Gold等のランクがある。
DDR4/DDR5(Double Data Rate 4/5)
現代の主流メモリ規格。DDR5はDDR4の2倍の転送速度。
DIMM(Dual Inline Memory Module)
デスクトップPC用メモリモジュールの標準規格。両面に端子がある。
GIGA端末
GIGAスクール構想で児童生徒に1人1台配備されるPC/タブレット端末。
HDD(Hard Disk Drive)
磁気ディスクを回転させてデータを読み書きする記憶装置。SSDより低速だが大容量。
HDMI(High-Definition Multimedia Interface)
映像・音声をデジタル伝送する標準規格。家電機器との接続で普及。
PCIe(PCI Express)
高速拡張バス規格。GPU、NVMe SSD、拡張カード等の接続に使用。
PSU(Power Supply Unit)
電源ユニット。AC 100VをDC +12V/+5V/+3.3Vに変換する装置。
SATA(Serial ATA)
HDD/SSD接続用の規格。最大600MB/s。M.2 NVMeより低速。
SSD(Solid State Drive)
フラッシュメモリを使用する記憶装置。HDDより高速・低消費電力・耐衝撃性が高い。
TBW → タ行を参照
TDP → タ行を参照
USB(Universal Serial Bus)
汎用データ転送・給電規格。USB 2.0(480Mbps)からUSB4(40Gbps)まで。

📚 まとめ

本講座「第54回：計算機ハードウェア」では、コンピュータの内部構造から実践的な診断・保守管理まで、学校現場のICT支援員・教育ICT講師に必要な専門知識を体系的に学習しました。

本講座で習得した知識とスキル

  • 理論：フォン・ノイマン型アーキテクチャの原理と現代CPUへの応用、ハードウェアコンポーネントの役割と相互作用、入出力インターフェースの技術仕様と互換性
  • 実践：症状別トラブルシューティングの診断手法、定期保守と部品交換のタイミング判断、学校資産としてのライフサイクル管理
  • 管理：修理 vs 買替の費用対効果分析、予算制約下での優先順位付け、教育活動継続を最優先とした緊急時対応

継続的成長へのメッセージ

ハードウェア技術は日進月歩です。本講座で学んだ「基本原理」は不変ですが、「具体的な製品仕様」は数年で刷新されます。

重要なのは以下の3点です：

  • 原理原則の理解：フォン・ノイマン型アーキテクチャ、電気信号の伝達、熱管理の物理法則は変わりません。この基盤があれば、新技術も理解できます。
  • 実践経験の蓄積：座学だけでは真の理解に至りません。実機を触り、トラブルに遭遇し、試行錯誤する中で「現場の感覚」が身につきます。
  • 学び続ける姿勢：新しいUSB規格、新世代CPU、次世代インターフェースが登場するたびに、情報をキャッチアップする習慣を持ちましょう。

  • 授業が止まらない：迅速なトラブル対応で、教員と生徒の時間を守る
  • 最適な環境を提案：技術的知見に基づく機器選定で、限られた予算を有効活用
  • 長期的視点：計画的な保守管理で、ICT環境の持続可能性を確保
