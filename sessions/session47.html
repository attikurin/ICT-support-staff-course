  2. 第2部：課題分析の手法 - SWOT分析、ロジックツリー、なぜなぜ分析、ステークホルダー分析
  3. 第3部：解決策の立案 - 複数案検討、実現可能性評価、リスク対応、KPI設定
  4. 第4部：プレゼンテーション技術 - スライド設計、話し方、時間管理
  5. 第5部：口頭試問対策 - 頻出質問、専門用語、想定問答
  6. 第6部：実践演習 - 模擬課題、相互評価、ロールプレイ

関連講座：第33回、第34回、第36回、第45回、第46回

⚠️ 本講座の重要性

2次試験は、1次試験で習得した知識を実践的に活用できるかが問われます。単なる知識の暗記ではなく、現場の課題を分析し、実現可能な解決策を提案し、説得力を持ってプレゼンテーションする総合力が求められます。

最も重要：リハーサルを繰り返すこと。本講座の内容を理解した上で、最低3回は通しで練習してください。

❌ 不合格になりやすいパターン

  • 現状分析が表面的：データに基づかず、問題の本質に迫っていない
  • 解決策が抽象的：「研修を実施する」だけで、具体的な実施計画が不明瞭
  • 時間配分ミス：現状分析に時間をかけすぎ、肝心の解決策が駆け足
  • 実現可能性が低い：予算や人員の制約を無視した理想論
  • 質疑応答で破綻：専門知識の不足や準備不足が露呈
  • 緊張でパニック：リハーサル不足で本番で頭が真っ白に

✅ 合格者の共通点

  • 徹底した課題分析：SWOT、ロジックツリー等のフレームワークを効果的に活用
  • 具体性：「誰が、いつ、何を、どのように」が明確
  • 実現可能性：予算、人材、時間の制約を考慮した現実的な提案
  • エビデンス：データ、先行事例、研究成果に基づく説明
  • 時間管理：リハーサルを重ね、余裕を持って終了
  • 想定問答：50問以上の質問を想定し、回答を準備

🔍 第2部：課題分析の手法

優れたプレゼンテーションの基礎は、徹底した課題分析にあります。本部では、実務で使える分析フレームワークを習得し、問題の本質を見抜く力を養います。

2.1 現状分析フレームワーク

📊 SWOT分析（教育情報化における活用）

SWOT分析は、内部環境（強み・弱み）と外部環境（機会・脅威）を整理し、戦略を導出するフレームワークです。

  • バランス：各象限に3〜5項目程度が目安
  • 戦略導出：クロスSWOTで実行可能な施策を考える

2.2 問題の構造化

🌳 ロジックツリー（問題分解）

ロジックツリーは、問題を階層的に分解し、根本原因を特定する手法です。

【課題】端末活用が進まない

【課題】端末活用が進まない
   │
   ├─【教員要因】
   │  │
   │  ├─スキル不足
   │  │  ├─基本操作に不安がある
   │  │  ├─授業設計力が不足している
   │  │  └─トラブル対応ができない
   │  │
   │  └─意識・姿勢
   │     ├─従来授業への固執
   │     ├─時間的余裕がない
   │     └─失敗への恐れ
   │
   ├─【環境要因】
   │  │
   │  ├─インフラ
   │  │  ├─ネットワークが不安定
   │  │  ├─充電管理が煩雑
   │  │  └─周辺機器が不足
   │  │
   │  └─支援体制
   │     ├─ICT支援員が不足
   │     ├─相談体制が未整備
   │     └─活用事例が不足
   │
   └─【児童生徒要因】
      ├─スキル差が大きい
      ├─家庭環境の差
      └─健康面への配慮が必要

✅ ロジックツリー作成のコツ

  • MECE：Mutually Exclusive, Collectively Exhaustive（漏れなく、ダブりなく）
  • 3〜4階層：最下層は「具体的な事象」まで分解する
  • 因果関係：上位階層と下位階層の間に論理的つながりを持たせる
  • 優先順位：分解後、最も影響が大きい要因を特定する

⚠️ よくある失敗

  • 分解が浅い：「教員のスキル不足」で止まり、何が不足しているのか掘り下げない
  • ダブりがある：「研修不足」と「学習機会の欠如」が重複している
  • 漏れがある：「教員」と「環境」だけで、「児童生徒」「保護者」の視点が抜けている

🔍 なぜなぜ分析（根本原因の特定）

「なぜ？」を5回繰り返すことで、表面的な問題から根本原因に到達します。

【現象】授業でのICT活用率が20%に留まる
   ↓ なぜ？
【原因1】教員がICTを使った授業設計ができない
   ↓ なぜ？
【原因2】効果的な活用方法を知らない
   ↓ なぜ？
【原因3】研修機会が少なく、事例共有もない
   ↓ なぜ？
【原因4】研修計画が体系化されておらず、
         校内でのナレッジ蓄積の仕組みがない
   ↓ なぜ？
【根本原因】組織的な研修体系とナレッジマネジメントの欠如

✅ なぜなぜ分析のポイント

ポイント 説明
5回の「なぜ」 通常3〜5回で根本原因に到達。必ず5回とは限らない
真因を探る 表面的な対症療法を避け、本質的な問題を特定する
「人」でなく「仕組み」 「〇〇先生がやる気がない」ではなく、「支援体制が不足」と構造的に捉える
検証可能性 「本当にそれが原因か？」をデータや事例で裏付ける

プレゼンでの活用例

「なぜなぜ分析の結果、根本原因は『組織的な研修体系の欠如』であることが分かりました。そのため、個別研修の実施だけでなく、体系的な研修計画とナレッジマネジメントの仕組み構築を解決策の柱とします」

💡 ステークホルダー分析のポイント

  • 推進派・抵抗派を特定：誰が協力者で、誰が反対しそうか
  • キーパーソンへの働きかけ：影響力の大きい人物を重点的にケア
  • 懸念への事前対応：抵抗要因を先回りして解消
  • コミュニケーション計画：誰に、いつ、何を、どのように伝えるか

📌 第2部のまとめ

  • SWOT分析：内部・外部環境を整理し、クロスSWOTで戦略を導出する
  • PEST分析：政治・経済・社会・技術の4つの観点でマクロ環境を理解する
  • ロジックツリー：問題をMECE（漏れなく、ダブりなく）に分解する
  • なぜなぜ分析：5回の「なぜ？」で根本原因を特定する
  • ステークホルダー分析：関係者の期待・懸念を把握し、対応戦略を立てる
  • これらのフレームワークを組み合わせることで、説得力のある課題分析が可能になる

対応方針 説明 具体例
回避

低減

  • □ 文字だけのスライドが3枚以上連続していない
  • □ グラフ・図は大きく、見やすい
  • □ 色は2〜3色に絞られている
  • □ フォントサイズは18pt以上
  • □ スライド番号が入っている
  • □ 全体で10〜15枚程度（10分の場合）

4.2 説得力のある話し方

📢 PREP法の活用

【PREP法の構造】

P (Point)    : 結論
   「校内メンター制度の導入を提案します」

R (Reason)   : 理由
   「持続可能で効果的な研修体制だからです」

E (Example)  : 具体例
   「先行実施のB校では活用率が30%→70%に向上しました」

P (Point)    : 結論（再確認）
   「よって校内メンター制度が最適解です」

PREP法の利点

  • 論理的：結論→理由→具体例の流れで説得力が増す
  • 簡潔：要点が明確で、聞き手が理解しやすい
  • 時間管理：構造が決まっているので、話しやすい
  • 質疑応答でも有効：質問に対する回答もPREP法で

4.4 質疑応答への備え

❓ プレゼン後の質疑応答のコツ

【質疑応答の基本フロー】

1. 質問をしっかり聞く
   └─ メモを取りながら聞く
   └─ 聞き逃したら「恐れ入りますが、もう一度お願いします」

3. PREP法で回答
   P: 結論から述べる
   R: 理由を説明
   E: 具体例を示す
   P: 結論を再確認

4. 質問に感謝
   「ご質問ありがとうございました」

5.4 冷静な対応の維持

😌 メンタルコントロールの技術

口頭試問では、予期しない質問や厳しい指摘を受けることもあります。そのような場合でも、冷静に対応するための技術を身につけましょう。

【困った質問への対応フレームワーク】

1️⃣ 質問の確認(時間を稼ぐ)
   「〇〇についてのご質問でよろしいでしょうか?」
   └─ 深呼吸する時間を作る
   └─ 質問の意図を確認できる

2️⃣ 分かる部分から答える
   「〇〇の観点からお答えしますと...」
   └─ 完璧な回答でなくても、関連する知識を伝える

3️⃣ 分からないことは正直に
   「その点については、現時点で十分な検討ができておりませんので、
    持ち帰って確認させていただけますでしょうか」
   └─ 知ったかぶりは最悪の対応
   └─ 誠実さが評価される

4️⃣ 質問への感謝
   「重要なご指摘をありがとうございます。
    今後の計画に反映させていただきます」
   └─ 防衛的にならない
   └─ 質問を前向きに受け止める姿勢
困った状況 対応方法 NGな対応
質問の意味が分からない 「恐れ入りますが、〇〇という理解でよろしいでしょうか?」
「もう少し詳しくお聞かせいただけますか?」 		• 分かったふりをする
• 的外れな回答をする
想定外の質問 「その観点は考慮しておりませんでした。現時点での考えとしては...」
「重要なご指摘です。〇〇の視点から検討すると...」 		• 焦って支離滅裂な回答
• 「質問が悪い」という態度
厳しい指摘 「ご指摘の通り、その点は課題です。対応策としては...」
「貴重なご意見ありがとうございます。計画に反映いたします」 		• 反論する
• 言い訳を並べる
• 防衛的になる
答えられない専門知識 「その点については、勉強不足で正確にお答えできません。確認してご報告いたします」
「現時点での理解では...」(推測である旨を明示) 		• 知ったかぶり
• 誤った情報を伝える
• あいまいにごまかす
矛盾を指摘される 「ご指摘の通り、説明に不整合がありました。正しくは...」
「言葉足らずでした。補足いたしますと...」 		• 認めない
• さらに矛盾を重ねる
• 話をそらす

🧘 緊張をコントロールする技術

💡 本番で使える緊張緩和テクニック

  1. 深呼吸法:
    • 4秒かけて鼻から息を吸う
    • 7秒息を止める
    • 8秒かけて口から息を吐く
    • 質問を受けてから答える前に実践
  2. ポジティブセルフトーク:
    • 「準備は十分にした」
    • 「質問は攻撃ではなく、関心の表れ」
    • 「分からないことがあっても当然」
  3. 身体のリラックス:
    • 肩の力を抜く(意識的に肩を下げる)
    • 手を軽く握ったり開いたりする
    • 足の裏で床を感じる(グラウンディング)
  4. 視線のコントロール:
    • 質問者の眉間あたりを見る(目を合わせすぎない)
    • 複数の試験官に順番に視線を配る
    • 下を向きすぎない、上を向きすぎない
  5. ペース配分:
    • 早口にならないよう意識する
    • 語尾まではっきり話す
    • 「えーと」「あのー」を減らす(沈黙を恐れない)

🎭 ロールプレイング演習の重要性

口頭試問のシミュレーション方法

  • 模擬面接の実施:同僚や友人に試験官役を依頼し、想定問答を実践
  • 録画して振り返り:自分の話し方、表情、姿勢を客観的にチェック
  • 想定外の質問練習:準備していない質問への対応力を養う
  • 制限時間を設定:各質問への回答時間を1分程度に制限して練習
  • フィードバック:第三者から率直な意見をもらい改善

⚠️ 口頭試問でやってはいけないこと

  • 質問を遮る:最後まで聞かずに答え始める
  • 長すぎる回答:1問に5分も話し続ける(簡潔に1〜2分で)
  • 質問者を批判:「その質問は適切ではない」等の発言
  • 感情的になる:指摘に対して反論・言い訳を繰り返す
  • 専門用語の乱用:相手を煙に巻こうとする態度
  • 準備不足の露呈:「何も考えていませんでした」

6.3 ロールプレイング演習

🎭 実践的なシナリオ演習

7.6 試験中の心構え

🎯 本番で実力を発揮するために

【プレゼン中】
□ 冒頭は暗記した通りに話す(自信につながる)
□ 視線を試験官に向ける(スクリーンばかり見ない)
□ ゆっくり、はっきり話す(早口にならない)
□ 語尾まで明確に発音する
□ 適度な間を取る(沈黙を恐れない)
□ スライドの時刻メモで進捗を確認
□ 時間が足りない時はCouldの内容を省略
□ 熱意を持って話す
□ 笑顔を忘れない
□ 最後まで諦めない

【質疑応答中】
□ 質問は最後まで聞く(遮らない)
□ メモを取りながら聞く
□ 質問の意図を確認する
   「〇〇についてのご質問でよろしいでしょうか?」
□ PREP法で回答する(結論→理由→具体例→結論)
□ 簡潔に答える(1〜2分程度)
□ 分からないことは正直に認める
   「勉強不足で申し訳ございません。確認してご報告いたします」
□ 推測の場合は明示する
   「現時点での理解では...」
□ 質問に感謝する
   「ご質問ありがとうございました」
□ 全員に視線を配る(質問者だけでなく)
□ 冷静さを保つ(感情的にならない)

【トラブル時の対応】
□ PCが動かない → 深呼吸して、試験官に助けを求める
□ 時間が足りない → 優先度の低い部分を省略
□ 質問の意味が分からない → 正直に聞き返す
□ 緊張で頭が真っ白 → 深呼吸、水を一口飲む
□ 間違いに気づく → 素直に訂正する
□ 厳しい指摘 → 「ご指摘ありがとうございます」と受け止める

⚠️ 試験中の絶対NG行動

  • 諦める:途中で投げ出さない。最後まで全力で
  • 感情的になる:厳しい指摘にも冷静に対応
  • 嘘をつく:知らないことは正直に認める
  • 試験官を批判:質問や指摘に対して反論しない
  • 時間無視:大幅な時間オーバーは減点対象
  • 下を向く:視線を下げず、試験官を見る

8.5 不合格事例に学ぶ「やってはいけないこと」

⚠️ 不合格者に共通する失敗パターン

失敗パターン 具体例 改善策
1. 理想論に終始 「AIを活用した完全個別最適化学習」
「全教員がICTマスターになる」
実現可能性を無視した壮大な計画 		現実的で、明日から実践できる施策に絞る。 小規模から始める段階的アプローチ
2. 技術偏重 スライドがシステム構成図だらけ。 ネットワーク帯域の計算式を説明。 教育の話がほとんどない 技術は「手段」。主役は「教育」と「人」。 技術的な話は最小限に
3. 準備不足 想定問答が20問程度。 専門用語を説明できない。 先行事例を知らない 想定問答50問以上、専門用語50語、 先行事例3〜5件を必ず準備
4. 防衛的な態度 質疑応答で反論、言い訳。 「それは想定していません」と逆ギレ。 試験官を見下す態度 どんな指摘も感謝して受け止める。 「貴重なご意見ありがとうございます」
5. 時間管理の失敗 15分かかり、大幅超過。 または7分で終わり、内容不足。 質疑応答の時間がなくなる リハーサルを最低3回。 9分以内で終わる練習を徹底
6. スライドが分かりにくい 文字だらけ、図表が複雑。 1スライドに3つもメッセージ。 配色がバラバラ 1スライド1メッセージを厳守。 文字は最小限、図表はシンプルに。 配色を統一
7. 知ったかぶり 知らない専門用語を適当に説明。 質問に対して曖昧な回答。 後で矛盾が発覚 分からないことは正直に認める。 「確認してご報告します」と誠実に対応

⚠️ これをやったら確実に不合格

  • 遅刻:時間厳守は社会人の基本
  • 服装が不適切:スーツ以外はNG
  • 試験官への無礼:挨拶しない、態度が悪い
  • 途中放棄:最後まで諦めない
  • 盗作・剽窃:他者の資料をそのまま使用
  • 虚偽のデータ:嘘のデータや実績を提示

↓

【Action(改善)】
↓
□ うまくいったことを継続
   効果的だった学習方法、実践方法を続ける

□ うまくいかなかったことを改善
   「なぜうまくいかなかったか」を分析し、次に活かす

□ 次月の目標を設定
   今月の振り返りを踏まえ、次の目標を立てる
