中級編第13回

予算・調達プロセスの理解

所要時間: 90分 | レベル: 中級

📘 学習目標

  • 学校ICT予算の仕組みと財源を理解する
  • ICT機器・ソフトウェアの調達プロセスを説明できる
  • 予算要望書の作成方法と必要な情報を習得する
  • ICT支援員として予算・調達に関わる際の役割を理解する

1. 学校ICT予算の基本

1.1 予算の財源

💡 学校ICT予算の主な財源

  • 国庫補助金：文部科学省のICT関連補助金
    • GIGAスクール構想関連予算
    • ICT活用教育アドバイザー事業
    • 学校DX推進事業
  • 地方交付税措置：国から自治体への財政支援
    • 教育のICT化に向けた環境整備（年間約2,000億円規模）
    • ICT支援員の配置（4校に1人）
  • 自治体独自予算：市区町村の一般財源
  • 寄付・PTA予算：限定的な用途

1.2 予算の種類

予算区分 用途 例
初期整備費 機器の導入 タブレット購入、電子黒板設置
運用・保守費 維持管理 保守契約、故障修理
消耗品費 定期交換品 プロジェクターランプ、充電ケーブル
ソフトウェア費 アプリ・サービス デジタルドリル、校務支援システム
人件費 支援員配置 ICT支援員、ネットワーク管理者

1.3 予算サイクル

🎯 年間予算スケジュール（標準的な例）

  • 6月〜8月：次年度予算の要望取りまとめ
  • 9月〜10月：教育委員会での予算編成
  • 11月〜12月：自治体での予算査定
  • 2月〜3月：議会での予算審議・承認
  • 4月：新年度予算の執行開始

2. ICT機器・ソフトウェアの調達プロセス

2.1 調達の流れ

  1. ニーズ把握：現状の課題、必要性の確認
  2. 仕様書作成：必要な機能、スペックの明確化
  3. 予算確保：見積もり取得、予算要望
  4. 入札・見積もり比較：複数社からの提案
  5. 契約締結：業者決定、契約書作成
  6. 納品・検収：機器納品、動作確認
  7. 支払い：検収後の代金支払い

2.2 調達方法の種類

方法 特徴 適用場面
一般競争入札 公開入札、最低価格で落札 高額案件（自治体の基準額以上）
指名競争入札 指名業者のみ参加 専門性が高い案件
随意契約 特定業者と直接契約 少額、緊急、既存システム追加
見積もり合わせ 複数社から見積もり取得 少額案件

2.3 仕様書作成のポイント

🎯 良い仕様書の条件

  • 目的の明確化：何のために導入するのか
  • 必須要件と推奨要件の分離：
    • 必須：最低限必要な機能・スペック
    • 推奨：あれば望ましい機能
  • 客観的な基準：
    • ○：「画面サイズ10インチ以上」
    • ×：「使いやすいこと」（主観的）
  • オープンな仕様：特定メーカーに限定しない
  • 保守・サポート条件：保証期間、修理対応

3. 予算要望書の作成

3.1 予算要望書に含めるべき項目

項目 内容
件名 何を要望するのか簡潔に
現状と課題 なぜ必要なのか、データで示す
導入目的 何を実現したいのか
期待される効果 教育的効果、業務効率化など
概算金額 見積もり額、内訳
導入スケジュール いつまでに必要か
維持費用 次年度以降の運用コスト

3.2 効果的なプレゼンテーション

予算要望を通すためのポイント

  • 具体的なデータ：
    • 「機器が古い」→「導入から8年経過、故障率30%」
    • 「足りない」→「1クラス40人に対し端末20台しかない」
  • 他校事例：
    • 先進校の実践例
    • 導入効果の具体例
  • 国の方針との整合性：
    • GIGAスクール構想
    • 学習指導要領
  • 費用対効果：
    • 業務効率化による時間削減
    • 学習効果の向上

4. リース契約 vs 購入

4.1 比較表

項目 購入 リース
初期費用 高い（全額一括） 低い（月額・年額払い）
総費用 リースより安い 金利分だけ高い
所有権 学校（自治体） リース会社
更新 自由に判断 契約期間満了後に入れ替え
保守 別途契約必要 含まれることが多い
予算計上 資産購入費 賃借料（毎年計上）

4.2 リース期間の考え方

  • PC・タブレット：5年リースが一般的
  • サーバー：5年リース
  • ネットワーク機器：5〜7年
  • プロジェクター：5年

5. ICT支援員の役割

5.1 情報収集・調査

  • 現状把握：
    • 機器の台数、型番、導入年度
    • 故障頻度、修理履歴
    • 教員・児童生徒の要望
  • 市場調査：
    • 最新機器の情報収集
    • 価格相場の把握
    • 他校の導入事例

5.2 仕様書作成支援

  • 必要なスペックの明確化
  • 技術的な要件の記述
  • 保守・サポート条件の提案
  • 互換性・拡張性の考慮

5.3 検収・動作確認

  • 納品された機器の台数確認
  • 仕様書通りのスペックか確認
  • 動作テスト（起動、接続、アプリ）
  • 不良品の洗い出し

5.4 資産管理サポート

  • 機器台帳の作成・更新
  • シリアル番号、管理番号の記録
  • 配置場所の記録
  • リース満了時期の管理

⚠️ ICT支援員の権限範囲

できること：

  • 情報収集、調査
  • 技術的な助言
  • 仕様書のドラフト作成
  • 検収の補助

できないこと（学校・教育委員会の権限）：

  • 予算の決定
  • 業者との契約
  • 発注行為

6. コスト削減の工夫

6.1 共同調達

  • 複数校・複数自治体での一括調達
  • スケールメリットによる単価削減
  • 仕様の標準化

6.2 オープンソース・フリーソフトの活用

  • LibreOffice（Office互換）
  • GIMP（画像編集）
  • Audacity（音声編集）
  • ※ただし、サポートは自力で対応

6.3 クラウドサービスの活用

  • Google Workspace for Education（無料）
  • Microsoft 365 Education A1（無料）
  • サーバー購入・保守費用の削減

6.4 長寿命化

  • 適切なメンテナンス
  • 故障予防（清掃、適切な保管）
  • 部分的な更新（メモリ増設など）

📝 演習問題

演習1: 予算要望書の作成

タブレット端末を40台追加導入したい場合の予算要望書の骨子を作成してください。以下の項目を含めてください：

  1. 現状と課題
  2. 導入目的
  3. 期待される効果
  4. 概算金額

演習2: リース vs 購入の判断

以下の状況で、リースと購入のどちらが適切か、理由とともに説明してください。

  • タブレット200台を導入
  • 初期予算は限られている
  • 5年後に最新機種に入れ替えたい

演習3: 仕様書の改善

以下の仕様書の問題点を指摘し、改善案を提案してください。

悪い例：「使いやすいタブレット端末を40台購入したい」

📌 まとめ

  • 学校ICT予算は国庫補助金、地方交付税措置、自治体独自予算から構成される
  • 調達プロセスは仕様書作成、入札、契約、検収の流れで進む
  • 予算要望書には現状課題、導入目的、期待効果、概算金額を明記する
  • リース契約は初期費用が低いが総額は高く、定期更新に適している
  • ICT支援員は情報収集、仕様書作成支援、検収サポートを担う
