1-2. ICT活用の3分類

文部科学省によるICT活用の分類

① A：一斉学習（教師による指導の効率化）
  • 目的：教師の教材提示を効率化し、分かりやすい授業を実現
  • 主な機器：電子黒板、プロジェクター、実物投影機、デジタル教科書
  • 具体例：
    • 動画や画像を大きく映して説明
    • 児童の考えを電子黒板に映して共有
    • シミュレーションを使った実験の可視化
② B：個別学習（児童生徒の主体的な学習）
  • 目的：一人ひとりの理解度や進度に応じた学習
  • 主な機器・ソフト：タブレット、PC、ドリル型教材、調べ学習
  • 具体例：
    • 適応学習（AIが難易度を調整）
    • インターネットで情報収集
    • 自分のペースで反復練習
    • eポートフォリオでの振り返り
③ C：協働学習（情報を共有・協働制作）
  • 目的：他者と協力して課題を解決、知識を深める
  • 主な機器・ソフト：共同編集ツール、プレゼンソフト、ビデオ会議
  • 具体例：
    • Googleドキュメントで共同執筆
    • グループでプレゼン資料作成
    • 他校とのオンライン交流
    • 相互評価・ピアレビュー

1-3. 協同学習と探究学習

📌 協同学習（Cooperative Learning）

小グループで協力して学習する形態

重要要素（5つの基本的要素）
  1. 相互依存性：メンバー全員が協力しないと目標達成できない
  2. 個人の責任：各自が役割を果たす責任を持つ
  3. 対面的相互作用：直接顔を合わせて協力する
  4. 社会的スキルの活用：コミュニケーション、リーダーシップ、意思決定
  5. グループの振り返り：活動を振り返り、改善する
主な手法
  • ジグソー法：
    1. 異なる教材を各自が学習（エキスパート活動）
    2. ジグソーグループで互いに教え合う
    3. 全体で共有し、理解を深める
  • Think-Pair-Share：個人で考える→ペアで共有→全体で発表
  • グループ調査：役割分担して調べ、統合してまとめる
ICT活用
  • Google Classroom：課題配布、提出、フィードバック
  • Jamboard：オンライン付箋でブレインストーミング
  • Microsoft Teams：チャネルでグループディスカッション

📌 問題解決学習（Problem-Based Learning: PBL）

実際的な問題から学習を開始する学習方法

特徴
  • 知識を習得してから問題に取り組むのではなく、問題から出発する
  • 現実的で複雑な問題を扱う
  • 自己主導的な学習
プロセス
  1. 問題の提示：現実的な問題状況を提示
  2. 情報収集：必要な情報を自ら収集
  3. 仮説の設定：原因や解決策について仮説を立てる
  4. 検証：仮説を検証する
  5. 解決策の提示：最適な解決策をまとめる
ICT活用例
  • 情報検索：インターネット、データベース
  • データ分析ツール：表計算ソフト、統計ソフト
  • シミュレーション：仮説検証
  • プレゼンテーション：解決策の発表

📌 探究学習（Inquiry-Based Learning）

疑問を持ち、自ら調べ、考え、表現する学習

探究のプロセス（例）
  1. 課題の設定：疑問や問いを立てる
  2. 情報の収集：文献調査、実験、観察、インタビュー
  3. 整理・分析：集めた情報を整理し、考察する
  4. まとめ・表現：レポート、プレゼン、ポスター作成
ICT活用
  • 情報収集：検索エンジン、デジタル図書館、データベース
  • 整理・分析：表計算ソフト、グラフ作成、思考ツール
  • 表現：プレゼンソフト、動画編集、Webページ作成

2-3. プログラミング的思考

📌 プログラミング的思考とは

自分が意図する一連の活動を実現するために、どのような動きの組合せが必要か、どのように改善していけばより意図した活動に近づくかを論理的に考えていく力（文部科学省）

プログラミング的思考の要素
  • 分解：問題を小さな部分に分ける
  • パターン認識：共通点や規則性を見つける
  • 抽象化：本質的な情報のみに注目する
  • アルゴリズム：手順を論理的に組み立てる
  • 評価・改善：結果を検証し、修正する（デバッグ）
小学校でのプログラミング教育
  • 必修化：2020年度から全面実施
  • 目的：プログラミング言語の習得ではなく、「プログラミング的思考」の育成
  • 実施方法：
    • A分類：学習指導要領に例示（算数、理科、総合）
    • B分類：各教科で適宜実施
    • C分類：クラブ活動、特別活動
  • 教材例：Scratch、プログル、Viscuit、micro:bit、レゴ®WeDo、マインドストーム（ロボット）
中学校での技術・家庭科（技術分野）
  • 内容：
    • ネットワークの活用
    • プログラミングによる計測・制御
    • 双方向性のあるコンテンツのプログラミング
  • 使用言語：ビジュアルプログラミング、テキストプログラミング（Python等）
高等学校での情報科
  • 情報I（必履修科目）：
    • プログラミング
    • データサイエンス
    • 情報デザイン
    • 情報社会の問題解決
  • 使用言語：Python等のテキストプログラミング言語

2-4. 情報モラル教育

情報モラルの5つの柱

項目 内容
① 情報社会の倫理 他者への影響を考えて行動する
例：誹謗中傷の禁止、ネットいじめ防止
② 法の理解と遵守 著作権、肖像権、個人情報保護法などの理解
例：他人の画像の無断使用禁止
③ 安全への知恵 危険を予測し、回避する
例：不審なメールを開かない、個人情報を書き込まない
④ 情報セキュリティ パスワード管理、ウイルス対策
例：パスワードの使い回しをしない
⑤ 公共的なネットワーク社会の構築 マナーを守り、建設的に参加
例：デマを拡散しない、情報の真偽を確認

💡 試験頻出：デジタル・シティズンシップ

近年注目されている概念。従来の「禁止・制限」中心の情報モラル教育から、「善き使い手」を育てる方向へシフト。

  • 自律的な判断力：状況に応じて適切に判断できる力
  • 積極的参加：情報社会に主体的に参画
  • 批判的思考：情報を鵜呑みにせず、多角的に検討

発達段階に応じた指導

  • 小学校低学年：基本的なルールとマナー
  • 小学校中学年：ネット利用の影響理解
  • 小学校高学年：情報の信頼性判断
  • 中学校：法的責任と社会的影響
  • 高等学校：情報社会の課題と参画

第3部：カリキュラムと授業設計

3-1. 学習指導要領「3つの柱」

資質・能力の3つの柱（2017年改訂）

学習指導要領では、育成すべき資質・能力を「3つの柱」で整理しています。

柱 内容
① 知識及び技能 「何を理解しているか、何ができるか」
• 各教科の基礎的・基本的な知識
• 技能の習得
• 知識を相互に関連付けて理解
② 思考力、判断力、表現力等 「理解していること・できることをどう使うか」
• 問題発見・解決能力
• 論理的思考、創造的思考
• 表現力、コミュニケーション能力
③ 学びに向かう力、人間性等 「どのように社会・世界と関わり、よりよい人生を送るか」
• 主体的に学習に取り組む態度
• 多様性の尊重
• 協働する力、思いやり

📌 主体的・対話的で深い学び（アクティブ・ラーニング）

3つの柱を育成するための授業改善の視点

① 主体的な学び
  • 学習の見通しを持ち、振り返る
  • 自ら課題を設定し、取り組む
  • ICT活用例：eポートフォリオ、学習ログの可視化
② 対話的な学び
  • 他者と協力して学ぶ
  • 対話を通じて考えを広げ、深める
  • ICT活用例：共同編集、オンライン議論、ビデオ会議
③ 深い学び
  • 知識を関連付けて理解
  • 問題解決の過程を通じて概念を形成
  • ICT活用例：シミュレーション、データ分析、探究学習

3-2. 単元設計と指導案

単元設計の基本

単元とは

一定のまとまりを持った学習内容（例：「植物の成長」「江戸時代の文化」）

単元設計の流れ
  1. 単元目標の設定：3つの柱に沿って具体的に設定
  2. 評価規準の明確化：どのような姿を目指すか
  3. 単元計画：何時間で、どのように学習を進めるか
  4. 教材・ICTの選定：目標達成に適した教材を選ぶ
  5. 評価計画：形成的評価、総括的評価

学習指導案の構成

項目 内容
単元名・題材名 学習内容の名称
単元目標 3つの柱に沿って記述
単元の評価規準 3観点（知識・技能、思考・判断・表現、主体的に学習に取り組む態度）
児童生徒観 学習者の実態、既習事項
教材観 教材の特性、指導の工夫
指導観 指導方針、ICT活用方法
単元計画 全体の流れ（第1時～最終時）
本時の展開 導入・展開・まとめの詳細な流れ、発問、ICT活用場面

新学習指導要領の評価（3観点）

2020年度から、評価の観点が従来の4観点から3観点に整理されました。

評価の観点 評価の内容・方法

  • CBT（Computer Based Testing）：コンピュータで実施するテスト。即座の採点、個別最適化
  • eポートフォリオ：学習の記録を蓄積し、成長を可視化
  • ルーブリック：評価基準を明確化。生徒と共有することで自己評価を促進
  • 相互評価機能：ロイロノート、Googleフォームなどでピアレビュー

3-4. カリキュラム・マネジメント

カリキュラム・マネジメントとは

学校の教育目標の実現に向けて、教育課程を編成し、実施し、評価して改善する一連のサイクル。

3つの側面
  • ① 教科横断的な視点：各教科の内容を関連付け、横断的に指導
  • ② 教育課程の実施とPDCAサイクル：計画→実施→評価→改善
  • ③ 人的・物的資源の活用：地域人材、ICT機器、予算の効果的活用
ICT環境整備との関連
  • 学校のICT活用方針をカリキュラムに位置づける
  • 情報活用能力を各教科で計画的に育成
  • ICT支援員や外部人材の効果的活用
  • 予算確保と計画的な機器更新

第4部：各教科でのICT活用

教育情報化コーディネータには、各教科の特性に応じたICT活用を提案できる知識が求められます。

国語でのICT活用例

算数・数学でのICT活用例

  • ドリル型教材：反復練習、即座のフィードバック

  • 動的幾何ソフト：GeoGebra（図形を動かして性質を発見）

  • 表計算ソフト：グラフ作成、平均・分散の計算

第5部：デジタル教材

5-1. デジタル教科書

デジタル教科書とは

紙の教科書の内容を電子化し、タブレットやPCで使用できるようにしたもの

2つの種類
  • 学習者用デジタル教科書：児童生徒が使用。2024年度から本格導入
  • 指導者用デジタル教科書：教員が電子黒板等で使用
主な機能
  • 拡大：文字や図を大きく表示（視覚支援）
  • 音声読み上げ：読み困難への支援
  • 書き込み：マーカー、付箋、メモ
  • 検索：語句をすぐに探せる
  • リンク：動画、音声、外部サイトへのリンク
  • 保存・共有：書き込みを保存、先生に提出

💡 デジタル教科書のメリット・課題

メリット
  • ✓ アクセシビリティ：読み書き困難な児童生徒への支援
  • ✓ 個別最適化：文字サイズ、音声速度の調整
  • ✓ 教材の豊富化：動画、音声、アニメーション
  • ✓ 荷物軽減：複数教科をタブレット1台で
課題
  • ✗ 紙との併用：当面は紙の教科書も必要（法改正で部分的に代替可能に）
  • ✗ コスト：紙の教科書＋デジタルの費用負担
  • ✗ 目の疲れ：長時間画面を見ることの健康影響
  • ✗ 教員研修：効果的な活用方法の習得

LMS（Learning Management System）とは

学習管理システム：教材配信、課題管理、成績管理、コミュニケーションを一元的に管理するシステム

📌 ドリル型教材

基本機能
  • 反復練習：繰り返し問題を解く
  • 即座のフィードバック：正誤がすぐに分かる
  • 自動採点：教員の負担軽減
  • 進捗管理：学習履歴の記録、可視化
代表的なドリル型教材
  • eライブラリ：小中高の全教科に対応
  • スタディサプリ：映像授業＋ドリル
  • 東京ベーシック・ドリル：東京都教育委員会提供（無料）
活用場面
  • 授業の導入：前時の復習
  • 習熟度別学習：個人のペースで進める
  • 家庭学習：宿題の代わり
  • 長期休暇：学習習慣の維持

5-4. 適応学習（アダプティブラーニング）

適応学習（Adaptive Learning）とは

AIが学習者の理解度を分析し、一人ひとりに最適な問題や学習コンテンツを提供するシステム

仕組み
  1. 診断：学習者の現在の理解度を測定
  2. 分析：つまずきポイント、苦手分野を特定
  3. 最適化：その子に合った難易度の問題を出題
  4. 調整：正答率に応じて難易度を自動調整
代表的な適応学習システム
  • Qubena（キュビナ）：算数・数学に特化。AIが最短ルートを提示
  • atama+（アタマプラス）：中高生向け。AIコーチ機能
  • すらら：無学年式。学年を超えて学習可能
メリット
  • ✓ 個別最適な学び：一人ひとりに合わせた学習
  • ✓ 効率的：必要な内容だけを学習
  • ✓ 教員の負担軽減：採点、分析が自動化
  • ✓ 学習意欲向上：自分に合った難易度で達成感
注意点
  • 基礎的な知識・技能の習得に効果的
  • 思考力や表現力は別途育成が必要
  • 教員による適切な指導・支援は不可欠

第6部：演習問題と試験対策

📝 B領域（学習・教育ブロック）の出題傾向

  • 学習理論：20%（構成主義、社会的構成主義、協同学習）
  • 情報活用能力：25%（3観点、体系表、プログラミング的思考、情報モラル）
  • 学習指導要領・評価：20%（3つの柱、カリキュラム・マネジメント、3観点評価）
  • 教科でのICT活用：25%（具体的な活用事例、デジタル教材）
  • 実践的判断：10%（授業設計、教材選定）

📌 練習問題（例）

問題1：学習理論

問題：「学習者が他者との対話を通じて、社会的に知識を構築していく」という考え方を表す学習理論はどれですか？

  1. 行動主義
  2. 認知主義
  3. 構成主義
  4. 社会的構成主義

解答：d) 社会的構成主義

解説：社会的構成主義は、ヴィゴツキーが提唱した理論で、他者との対話・協働を通じて学習が深まると考えます。協同学習やグループディスカッションは、この理論に基づいています。

問題2：情報活用能力

問題：プログラミング的思考を構成する要素として適切でないものはどれですか？

  1. 分解（問題を小さな部分に分ける）
  2. パターン認識（共通点や規則性を見つける）
  3. 暗記（多くの知識を記憶する）
  4. アルゴリズム（手順を論理的に組み立てる）

解答：c) 暗記（多くの知識を記憶する）

解説：プログラミング的思考は、「分解」「パターン認識」「抽象化」「アルゴリズム」「評価・改善」の要素から構成されます。暗記は含まれません。

問題3：学習指導要領

問題：新学習指導要領で示された「資質・能力の3つの柱」に含まれないものはどれですか？

  1. 知識及び技能
  2. 思考力、判断力、表現力等
  3. 関心・意欲・態度
  4. 学びに向かう力、人間性等

解答：c) 関心・意欲・態度

解説：「関心・意欲・態度」は旧学習指導要領の評価観点です。新学習指導要領では「学びに向かう力、人間性等」に整理されました。

問題4：ICT活用の分類

問題：文部科学省によるICT活用の3分類のうち、「C：協働学習」に該当する活動はどれですか？

  1. 電子黒板で動画を映して説明する
  2. ドリル型教材で個別に反復練習する
  3. Googleドキュメントで共同執筆する
  4. インターネットで情報を検索する

解答：c) Googleドキュメントで共同執筆する

解説：a)は「A：一斉学習」、b)とd)は「B：個別学習」に該当します。c)は他者と協働して制作する活動なので「C：協働学習」です。

問題5：デジタル教材

問題：適応学習（アダプティブラーニング）の説明として最も適切なものはどれですか？

  1. すべての児童生徒が同じ順序で同じ問題を解く
  2. AIが学習者の理解度を分析し、一人ひとりに最適な問題を提供する
  3. 教師が手作業で各児童生徒に異なるプリントを配布する
  4. 習熟度別にクラスを分けて一斉授業を行う

解答：b) AIが学習者の理解度を分析し、一人ひとりに最適な問題を提供する

解説：適応学習は、AIが学習者のデータを分析し、リアルタイムで最適な学習内容を提供するシステムです。Qubena、atama+などが代表例です。

💡 試験対策のポイント

  • 学習理論の人物名：ピアジェ（構成主義）、ヴィゴツキー（社会的構成主義）を覚える
  • 情報活用能力の3観点：新旧の違いを理解（4観点→3観点）
  • 学習指導要領：2017年改訂のポイント（3つの柱、主体的・対話的で深い学び）

  • 教科別活用：各教科の特性に応じたICT活用例を覚える
  • 最新動向：デジタル教科書の本格導入（2024年度～）、生成AIの教育利用など
