中級編第8回

ネットワークとインフラ管理

所要時間: 90分 | レベル: 中級

📘 学習目標

  • 学校ネットワークの基本構成と特徴を理解する
  • Wi-Fi環境の構築と管理方法を説明できる
  • ネットワークトラブルの基本的な診断方法を習得する
  • セキュリティを考慮したネットワーク管理の重要性を理解する

1. 学校ネットワークの基本

1.1 学校ネットワークの構成

💡 典型的な学校ネットワーク構成

階層 構成要素 説明
1 インターネット 外部ネットワーク接続
2 ファイアウォール セキュリティ保護
3 教育委員会ネットワーク 自治体によっては経由
4 学校のルーター/スイッチ 校内ネットワークの中心
5a 有線LAN 教員用PC、サーバー
5b Wi-Fi 児童生徒用タブレット
5c 校務系ネットワーク 成績処理等（分離）

1.2 ネットワーク分離の重要性

校務系と学習系の分離

  • 校務系ネットワーク：成績処理、個人情報を扱うシステム専用
  • 学習系ネットワーク：児童生徒が使用するネットワーク
  • 分離の目的：個人情報漏洩リスクの低減

⚠️ ネットワーク分離のルール

  • 校務系PCと学習系PCは物理的に別々に用意
  • 校務用USBメモリを学習系PCに接続しない
  • 学習系から校務系へのデータ移動は慎重に

1.3 学校ネットワークの特徴

  • 多数の端末の同時接続：40台×複数クラス
  • 一斉アクセス：授業開始時に全員がログイン
  • 動画視聴の多発：帯域幅の圧迫
  • フィルタリング：有害サイトへのアクセス制限

2. Wi-Fi環境の構築と管理

2.1 Wi-Fiの基礎知識

規格 周波数帯 最大速度 特徴
IEEE 802.11n 2.4GHz / 5GHz 600Mbps 旧世代、互換性高
IEEE 802.11ac 5GHz 6.9Gbps 現在主流、高速
IEEE 802.11ax (Wi-Fi 6) 2.4GHz / 5GHz 9.6Gbps 最新、多数同時接続に強い

2.4GHz vs 5GHz

  • 2.4GHz：障害物に強い、遠くまで届く、混雑しやすい
  • 5GHz：高速、混雑しにくい、障害物に弱い

2.2 アクセスポイント（AP）の配置

🎯 適切なAP配置のポイント

  • 教室ごとに1台：40台同時接続を想定
  • 天井設置：見通しが良く、電波が届きやすい
  • 電波干渉の回避：隣接APは異なるチャンネル使用
  • 有線バックホール：APは有線LANで接続

2.3 SSID設計

SSIDの使い分け例

  • 児童生徒用SSID：フィルタリング有効、制限あり
  • 教員用SSID：フィルタリング緩和、高優先度
  • 来客用SSID：インターネットのみ、校内ネットワークにアクセス不可

2.4 帯域制御

  • QoS（Quality of Service）：重要な通信を優先
  • 帯域制限：1端末あたりの通信速度上限設定
  • 時間帯制御：授業時間帯は優先度を上げる

3. ネットワークトラブルの診断

3.1 基本的な切り分け手順

🎯 トラブル切り分けのステップ

  1. 現象の確認：「何ができない」のか具体的に確認
  2. 範囲の特定：1台だけ？クラス全体？学校全体？
  3. 物理層の確認：ケーブル接続、Wi-Fi接続状態
  4. ネットワーク層の確認：IPアドレス取得、疎通確認
  5. アプリケーション層の確認：ブラウザ、アプリの問題

3.2 よくあるトラブルと対処法

症状 考えられる原因 対処法
Wi-Fiに接続できない パスワード間違い、AP満杯 再入力、他のSSIDへ接続、AP再起動
インターネットに繋がらない ルーター障害、回線障害 ルーター再起動、ISP確認
通信が遅い 帯域不足、電波干渉 接続台数確認、チャンネル変更
特定のサイトに繋がらない フィルタリング、サイト障害 フィルタリング設定確認

3.3 診断コマンド（管理者向け）

基本的なコマンド

  • ping：ネットワークの疎通確認
  • ipconfig / ifconfig：IPアドレスの確認
  • nslookup：DNS名前解決の確認
  • traceroute / tracert：経路の確認

📌 ICT支援員ができる基本確認

  • Wi-Fiアイコンの状態確認（接続済み？電波強度は？）
  • 機内モードがONになっていないか
  • ブラウザのキャッシュクリア
  • 端末の再起動
  • 他の端末での動作確認（切り分け）

4. セキュリティ対策

4.1 Wi-Fiセキュリティ

  • WPA3 / WPA2暗号化：通信の暗号化（WEPは使用禁止）
  • 強固なパスワード：推測されにくい複雑なパスワード
  • SSIDステルス：SSIDを隠す（効果は限定的）
  • MACアドレスフィルタリング：登録端末のみ接続許可

4.2 フィルタリング

💡 フィルタリングの目的

  • 有害サイト（暴力、アダルト、ギャンブル等）へのアクセス防止
  • 不適切なSNS利用の制限
  • 違法ダウンロードサイトのブロック

フィルタリングの方法

  • ブラックリスト方式：有害サイトリストでブロック
  • ホワイトリスト方式：許可リストのみアクセス可能
  • カテゴリフィルタリング：サイトをカテゴリ分類してブロック

4.3 ファイアウォール

  • 外部からの不正アクセスをブロック
  • 内部から外部への不適切な通信を制限
  • ポート制御（必要なポートのみ開放）

4.4 定期的なセキュリティチェック

  • パスワードの定期変更
  • 不要なアカウントの削除
  • ファームウェアのアップデート
  • ログの確認（不審なアクセス）

5. クラウドサービスの活用

5.1 クラウド化のメリット

🎯 学校におけるクラウド活用

  • Google Workspace for Education：無料、容量大
  • Microsoft 365 Education：Officeアプリが使える
  • メリット：
    • 校内サーバー不要、保守費用削減
    • どこからでもアクセス可能
    • 自動バックアップ
    • 複数人での共同編集

5.2 オンプレミスとの使い分け

項目 クラウド オンプレミス
初期費用 低い 高い（サーバー購入）
保守 不要 必要
カスタマイズ 制限あり 自由度高
セキュリティ ベンダー依存 自己管理

6. ICT支援員の役割

6.1 日常的な監視・メンテナンス

  • Wi-Fi接続状態の定期確認
  • インターネット速度のチェック
  • アクセスポイントのランプ状態確認
  • ケーブルの抜け・断線チェック

6.2 トラブル対応

  • 第一次切り分け（範囲の特定）
  • 簡単な対処（再起動、設定確認）
  • ネットワーク管理者への報告・エスカレーション
  • 授業への影響最小化（代替手段の提案）

6.3 教員・児童生徒へのサポート

  • Wi-Fi接続方法の説明
  • パスワード入力支援
  • 通信トラブル時の対処法指導
  • フィルタリングの説明

⚠️ ICT支援員の権限範囲

できること：

  • Wi-Fi接続のサポート
  • 端末側の設定確認
  • 簡単なトラブルシューティング

できないこと（専門業者に依頼）：

  • ルーター・スイッチの設定変更
  • ファイアウォールの設定
  • ネットワーク構成の大幅変更

📝 演習問題

演習1: トラブル診断

授業中、クラス全員のタブレットがインターネットに繋がらなくなりました。Wi-Fiには接続されていますが、Webページが開けません。考えられる原因と確認すべき項目を挙げてください。

演習2: 帯域不足への対応

動画視聴の授業中、通信速度が遅く動画が頻繁に止まります。ICT支援員として、どのような対策を提案できますか？（技術的対策と運用面での対策の両方を考えてください）

演習3: セキュリティ設定

学校に来客用Wi-Fiを新設することになりました。どのようなセキュリティ設定が必要ですか？考慮すべき点を3つ以上挙げてください。

📌 まとめ

  • 学校ネットワークは校務系と学習系を分離し、セキュリティを確保する
  • Wi-Fi環境では、適切なAP配置、SSID設計、帯域制御が重要
  • ネットワークトラブルは範囲の特定、物理層からの切り分けで診断する
  • WPA2/WPA3暗号化、フィルタリング、ファイアウォールでセキュリティを確保
  • ICT支援員は日常監視、第一次切り分け、教員・児童生徒サポートを担う
