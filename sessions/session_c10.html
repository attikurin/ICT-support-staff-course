中級編第10回

トラブルシューティング実践

所要時間: 90分 | レベル: 中級

📘 学習目標

  • 学校現場で頻発するICTトラブルの種類と原因を理解する
  • 体系的なトラブルシューティングの手順を習得する
  • 状況に応じた適切な対応と判断ができるようになる
  • トラブル記録と再発防止策の重要性を理解する

1. トラブルシューティングの基本原則

1.1 基本的な考え方

🎯 トラブル対応の5原則

  1. 落ち着く：慌てず冷静に状況を把握
  2. 聞く：具体的に「何が」「いつ」「どのように」起きたか確認
  3. 切り分ける：範囲を特定し、原因を絞り込む
  4. 試す：簡単な対処から順に試す
  5. 記録する：現象、対処法、結果を記録

1.2 切り分けの基本

💡 トラブル切り分けの観点

  • 範囲の特定：
    • 1台だけ？ → 端末の問題
    • 1クラスだけ？ → その教室のネットワークの問題
    • 学校全体？ → サーバー、回線の問題
  • 再現性：
    • 必ず起きる？ → 設定や故障の可能性
    • 時々起きる？ → 負荷、タイミングの問題
  • タイミング：
    • いつから？ → 直前の変更点を確認
    • 特定の時間帯だけ？ → 混雑、定期処理の影響

1.3 対応の優先順位

優先度 状況 対応
最優先 授業に支障、学校全体に影響 即座に対応、代替手段の提示
高 一部の授業に影響 授業時間内に解決を目指す
中 個人の端末トラブル 休み時間や放課後に対応
低 使い方の質問、軽微な不具合 時間のあるときに対応

2. 頻出トラブルと対処法

2.1 ログイン・認証トラブル

【トラブル1】パスワードを忘れた

  • 確認事項：
    • CapsLockがONになっていないか
    • 全角/半角を間違えていないか
    • 最近パスワード変更したか
  • 対処法：
    • パスワードカードの確認
    • 管理コンソールからパスワードリセット
    • 仮パスワード発行、変更を促す

【トラブル2】「アカウントがロックされています」

  • 原因：パスワード誤入力を繰り返した
  • 対処法：
    • 管理コンソールからロック解除
    • 正しいパスワードを確認してから再試行

2.2 ネットワーク接続トラブル

【トラブル3】Wi-Fiに繋がらない

🎯 診断手順

  1. Wi-Fiアイコン確認：表示されているか、電波マークは？
  2. 機内モード確認：ONになっていないか
  3. SSID確認：正しいSSIDを選択しているか
  4. パスワード確認：正しく入力しているか
  5. 他端末での確認：他の端末は繋がるか（切り分け）
  6. 端末再起動：端末を再起動してみる
  7. AP確認：アクセスポイントのランプ状態

【トラブル4】インターネットに繋がらない（Wi-Fiは接続済）

  • 確認事項：
    • 他の端末も同じ症状か
    • 特定のサイトだけか、すべてのサイトか
    • ルーターのランプ状態
  • 対処法：
    • ブラウザのキャッシュクリア
    • DNSの確認（8.8.8.8で試す）
    • ルーター再起動

2.3 端末・アプリトラブル

【トラブル5】タブレットが起動しない

  • 確認事項：
    • 充電はされているか
    • 電源ボタンを長押ししてみる
    • 画面が真っ暗なだけで音は出るか
  • 対処法：
    • 充電ケーブルを接続して10分待つ
    • 強制再起動（機種ごとの方法）
    • 修理が必要な場合は業者に連絡

【トラブル6】アプリが起動しない・フリーズする

  • 対処法の順序：
    1. アプリの再起動
    2. アプリのキャッシュクリア
    3. 端末の再起動
    4. アプリの再インストール
    5. OSのアップデート確認

2.4 授業中の緊急トラブル

【トラブル7】電子黒板に映らない

⚠️ 授業開始直前の対応

チェックリスト（30秒以内）：

  1. 電源が入っているか
  2. ケーブルが抜けていないか
  3. 入力切替が正しいか
  4. PCがスリープ状態でないか

すぐに解決しない場合：

  • 代替手段を提案（プロジェクター、別の教室）
  • 授業を進めてもらい、休み時間に対応

【トラブル8】音が出ない

  • ミュートになっていないか
  • 音量が0になっていないか
  • ヘッドホンが刺さっていないか
  • 出力先の選択ミス（HDMI、スピーカー）

3. ケース別対応シナリオ

3.1 ケース1：授業開始直前のトラブル

状況：授業開始5分前、教員から「タブレットが配れない。充電されていない」と連絡

対応手順：

  1. 状況確認：何台が充電されていないか、充電カートの電源は入っているか
  2. 即時対応：
    • 充電されている端末から優先的に配布
    • 充電されていない端末は充電しながら後で使用
  3. 代替案：ペア学習（2人で1台）に変更できないか教員に相談
  4. 根本対応：授業後、充電カートの接続確認、ケーブルの断線チェック

3.2 ケース2：複数人が同時にトラブル

状況：授業中、10人の児童が「Google Classroomが開けない」と挙手

対応手順：

  1. 範囲の特定：クラス全体か、一部だけか
  2. 切り分け：
    • 他のサイトは開けるか → ネットワークの問題かアプリの問題か
    • 他のクラスは問題ないか → 学校全体の問題か
  3. 優先順位：
    • 授業進行を優先：代替手段（紙のプリント）を提案
    • 問題が起きていない児童は授業継続
  4. 個別対応：ブラウザ再起動、キャッシュクリア

3.3 ケース3：原因不明の不具合

状況：「時々画面が固まる」という曖昧な相談

対応手順：

  1. 詳細なヒアリング：
    • いつ起きるか（特定のアプリ使用時？）
    • どのくらいの頻度か
    • 何をしているときか
  2. 再現確認：実際に目の前で再現してもらう
  3. ログ確認：エラーメッセージ、システムログ
  4. 記録：現象を詳細に記録し、業者に相談

4. エスカレーション（上位への報告）

4.1 エスカレーションが必要な場合

📌 エスカレーションの判断基準

  • 自分の権限・知識で対応できない
  • 30分以上対応しても解決しない
  • 学校全体に影響がある
  • ハードウェア故障の疑い
  • セキュリティに関わる問題

4.2 報告すべき相手

  • 情報担当教員：学校内の判断が必要な場合
  • 教育委員会：ネットワーク、システム全体の問題
  • 保守業者：ハードウェア故障、システム障害
  • 校長・教頭：重大なトラブル、保護者対応が必要な場合

4.3 報告時に伝えるべき情報

  • 現象：何が起きているか具体的に
  • 範囲：何台、どのクラス、いつから
  • 影響：授業に支障があるか
  • 実施した対応：何を試したか
  • 緊急度：すぐに対応が必要か

5. トラブル記録と再発防止

5.1 トラブル記録の重要性

🎯 記録する理由

  • 同じトラブルが起きたときの参照
  • 傾向分析（特定の端末、時期の問題把握）
  • 業者への報告資料
  • 次年度への引き継ぎ

5.2 記録すべき項目

項目 内容
日時 発生日時、対応日時
場所・機器 教室名、端末番号、機器名
報告者 教員名、児童生徒名
現象 具体的な症状、エラーメッセージ
対処内容 実施した対応、結果
原因 判明した原因（推測含む）
状態 解決/未解決/エスカレーション

5.3 再発防止策

  • 定期メンテナンス：発生前に予防する
  • マニュアル整備：よくあるトラブルの対処法を文書化
  • 教員・児童生徒への啓発：正しい使い方の指導
  • 環境改善：根本原因の解消（古い機器の更新など）

6. トラブル対応の心構え

6.1 冷静な対応

  • 焦らず、落ち着いて状況を把握する
  • 教員や児童生徒の不安を和らげる声かけ
  • 「必ず解決します」という姿勢を示す

6.2 コミュニケーション

  • 専門用語を使わず、分かりやすい説明
  • 進捗状況を適宜報告
  • 解決しない場合も、その旨を正直に伝える

6.3 継続的な学習

  • 新しい機器・アプリの情報収集
  • 他の支援員との情報共有
  • メーカーのサポート情報を定期確認

📝 演習問題

演習1: トラブル診断

授業中、児童から「Scratchが動かない」と相談されました。どのような順序で確認・対応しますか？

演習2: 優先順位の判断

以下のトラブルが同時に発生しました。対応の優先順位をつけ、理由を説明してください。

  1. 1台のタブレットが起動しない
  2. 電子黒板が映らず、授業が始められない
  3. Wi-Fiが遅く、動画が止まる
  4. 教員から「パスワードを忘れた」と連絡

演習3: エスカレーション判断

ネットワークトラブルで、以下を試しても解決しません。次にどのような行動を取りますか？

  • 端末の再起動 → 変化なし
  • Wi-Fi再接続 → 接続はできるがインターネットに繋がらない
  • 他の端末でも同じ症状

📌 まとめ

  • トラブル対応は「落ち着く」「聞く」「切り分ける」「試す」「記録する」の5原則
  • 範囲の特定、再現性、タイミングで原因を切り分ける
  • 授業への影響を最優先に考え、代替手段を提案する
  • 自分で対応できない場合は適切にエスカレーションする
  • トラブル記録を残し、再発防止と次回対応の効率化を図る
