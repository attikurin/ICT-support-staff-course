基礎編 第15回

授業支援の実践（初等教育）

所要時間: 90分 | レベル: 基礎

本講座の目的

小学校におけるICT授業支援の実践的なスキルを習得し、児童の発達段階に応じた適切な支援ができるようになることを目指します。

📘 学習目標

  • 小学校の学級担任制の特徴と支援のポイントを理解する
  • 低・中・高学年の発達段階に応じた支援方法を習得する
  • 授業前・授業中・授業後の各段階での具体的な支援方法を理解する
  • プログラミング教育の支援方法を習得する
  • 低学年への特別な配慮事項を理解する
  • 実践的な授業支援の流れを体験的に学ぶ

1. 小学校の特性理解

1.1 学級担任制の特徴

💡 学級担任制とは

小学校では、ほとんどの教科を1人の担任教諭が指導します。この制度下でのICT支援には以下の特徴があります。

支援のポイント

  • 担任教諭との信頼関係構築が最重要
  • 教諭のICTスキルレベルに応じた柔軟な支援
  • 教科横断的な視点でのアドバイス提供
  • 継続的なフォローアップ体制の確立

学級担任制のメリットを活かす

  • 児童一人ひとりの特性を担任が把握している
  • 学習の連続性を意識した支援が可能
  • 生活指導とICT活用の一体的な推進

1.2 発達段階への配慮

🎯 低学年（1〜2年生）

認知的特徴

  • 具体的・直感的な理解が中心
  • 集中力の持続時間が短い（10分程度、課題によっては15分程度）
  • 文字の読み書きが発展途上

※発達心理学では「年齢×1分」が集中力の目安とされますが、興味のある活動では10〜15分程度持続することもあります。授業では5〜10分ごとに活動を切り替える工夫が効果的です。

支援の工夫

  • 視覚的・聴覚的な教材の活用
  • 短時間での活動切り替え
  • 音声読み上げ機能の積極利用
  • 大きなアイコン・ボタンの使用
  • 段階的な操作指導（1つずつ確実に）

🎯 中学年（3〜4年生）

認知的特徴

  • 論理的思考の芽生え
  • 協働学習への適応力向上
  • 個人差の拡大

支援の工夫

  • グループ活動でのICT活用支援
  • 調べ学習のスキル指導
  • タイピング練習の本格化
  • 情報モラルの基礎教育

🎯 高学年（5〜6年生）

認知的特徴

  • 抽象的思考の発達
  • 自律的な学習姿勢の形成
  • ICTスキルの個人差が顕著

支援の工夫

  • 主体的な学びを促す支援
  • プレゼンテーション能力の育成
  • 情報リテラシーの本格的指導
  • 創造的な表現活動の支援

2. 授業前の準備支援

2.1 担任教諭との事前打ち合わせ

💡 確認すべき事項

  • 授業の目標とねらい - 本時で何を学ばせたいのか
  • 授業の流れ - 導入・展開・まとめの時間配分
  • ICT活用場面 - どの場面で、何のために使うのか
  • 使用するアプリ・機能 - 具体的なツール名
  • 児童の実態 - ICTスキルレベル、配慮が必要な児童
  • 期待する支援内容 - 教員が何を求めているか

2.2 機器・環境の事前確認

チェックリスト

項目 確認内容
端末 台数確認、充電状況、動作確認、予備機の準備
ネットワーク Wi-Fi接続確認、速度テスト、同時接続台数の確認
提示装置 接続確認、映像・音声テスト、ケーブル・アダプタの準備
アプリ インストール状況、ログイン確認、ファイル共有設定
教材 デジタル教材の動作確認、ファイルの配布準備

2.3 アカウント・設定の確認

  • ログイン情報 - 児童全員がログインできるか確認
  • 共有設定 - ファイルの共有権限が適切か
  • フィルタリング - 必要なサイトにアクセスできるか
  • アプリの権限 - カメラ、マイク等の使用許可

2.4 トラブル対応の準備

⚠️ 想定されるトラブルと対策

  • 端末が起動しない → 予備機を準備、充電器を用意
  • Wi-Fiに接続できない → 有線LAN接続の準備、モバイルルーター
  • アプリが開かない → 代替アプリの準備、オフライン教材
  • ログインできない → パスワード一覧表、リセット手順の確認
  • ファイルが開けない → 異なる形式での保存、別の共有方法

3. 授業中の支援

3.1 授業開始時の支援

スムーズな授業開始のために

  • 授業開始前に教室に入り、最終確認
  • 提示装置の接続サポート
  • 児童の端末起動・ログイン支援
  • トラブル発生時の迅速な対応

3.2 机間指導の補助

🎯 机間巡視のポイント

  • 困っている児童を見つける
    • 画面を見ずに固まっている
    • 何度も同じ操作を繰り返している
    • 隣の児童に聞いている
  • 適切なタイミングで声をかける
    • 「どうしたの？」「困ってる？」
    • まずは児童の話を聞く
    • 一緒に画面を見ながら考える
  • 自分で解決できるように支援
    • すぐに答えを教えない
    • ヒントを出しながら考えさせる
    • 「ここをクリックしてみようか」

3.3 個別支援のポイント

状況 支援方法
操作が分からない 画面を一緒に見ながら、ゆっくり説明
必要に応じてデモンストレーション
作業が進まない どこまでできているか確認
次のステップを具体的に示す
エラーメッセージ 落ち着かせてから、メッセージを一緒に読む
対処方法を説明
作業が早く終わった 発展課題の提示
他の児童の支援役を依頼

3.4 トラブル対応の実際

トラブル対応の基本姿勢

  • 冷静に対応 - 慌てない、焦らない
  • 授業の流れを止めない - 素早く判断し行動
  • 代替案を提示 - 別の方法で進められるようにする
  • 教員に報告 - 状況を簡潔に伝える

📋 トラブル対応例

状況：5人の児童のタブレットがWi-Fiに接続できない

対応：

  1. まず、他の児童は問題なく進められるか確認
  2. 接続できない5人を別グループにして、紙の資料で活動継続を提案
  3. 並行して、端末の再起動を試す
  4. 改善しない場合は、予備機と交換
  5. 授業後、原因を調査し報告書作成

3.5 教員へのさりげないサポート

  • 教員が操作に困っている様子があれば、そっと近づく
  • 児童に気づかれないよう、小声で助言
  • 操作を代わりに行う場合は、事前に了解を得る
  • 授業の進行を妨げないよう、タイミングに配慮

4. 授業後のフォロー

4.1 片付け支援

  • 端末の回収
    • 充電保管庫への正しい収納を指導
    • 破損・紛失がないか確認
    • 次の授業のために充電開始
  • データの保存確認
    • 作品が保存されているか確認
    • 共有設定が適切か確認
    • 提出物の受け取り確認

4.2 教員との振り返り

💡 振り返りのポイント

  • うまくいった点
    • 「〇〇の場面で、児童が積極的に取り組んでいましたね」
    • 「△△の使い方が効果的でした」
  • 改善点の提案
    • 「次回は〇〇すると、もっとスムーズかもしれません」
    • 「□□の機能を使うと、時間短縮できます」
  • 児童の反応のフィードバック
    • 「〇〇さんが、こんな工夫をしていました」
    • 「△△で困っている児童が何人かいました」

4.3 記録の作成

支援記録に残すべき情報

  • 日時、学年・クラス、教科、単元名
  • 授業の目標とICT活用の目的
  • 使用した機器・アプリ
  • 支援内容（事前準備、授業中の対応）
  • 児童の様子、反応
  • 発生したトラブルと対応
  • 成功事例、改善提案
  • 次回への申し送り事項

4.4 成果の可視化

  • 授業の写真撮影 - 児童の活動の様子（個人情報に配慮）
  • 作品の記録 - 優れた作品のスクリーンショット
  • 事例集の作成 - 校内で共有できる形にまとめる
  • 教員へのフィードバック - 写真付きレポートの提供

5. プログラミング教育の支援

5.1 プログラミング教育の位置づけ

💡 学習指導要領での位置づけ

小学校では、プログラミング的思考を育成することが求められています。必ずしもプログラミング言語を学ぶことが目的ではありません。

プログラミング的思考とは

  • 自分が意図する活動を実現するために
  • どのような動きの組み合わせが必要か
  • どう改善すればより意図した活動に近づくか
  • 論理的に考える力

5.2 代表的なプログラミング教材

教材名 対象学年 特徴
Viscuit 低〜中学年 絵を描いて動かす、直感的操作
Scratch 低学年後半〜高学年（主に3年生以上）
※低学年向けには「Scratch Jr」（5〜7歳対象）もあり		 ブロック型、世界的に普及
Hour of Code 全学年 ゲーム感覚、短時間で体験
micro:bit 高学年 実物のデバイス、センサー活用（加速度、温度、光センサー等内蔵）
※2025年時点ではmicro:bit v2が標準。スピーカー・マイク内蔵

5.3 プログラミング授業の支援例

📋 支援例：5年生 算数「正多角形」×Scratch

授業の目標：正多角形の性質を理解し、プログラムで作図する

事前準備：

  • Scratchの動作確認、サンプルプログラムの準備
  • ワークシートの印刷（角度計算用）
  • デモ用の完成プログラム準備

授業中の支援：

  • 導入：Scratchの基本操作のデモンストレーション
  • 展開：机間巡視で個別支援、つまずきポイントの把握
  • まとめ：作品の保存方法指導

よくあるつまずき：

  • 角度の計算間違い → 電卓の使用を提案
  • ブロックの組み方が分からない → 順序立てて考えるよう助言

5.4 プログラミング支援のコツ

  • すぐに答えを教えない - 試行錯誤が大切
  • エラーを肯定的に捉える - 「なぜうまくいかないか考えてみよう」
  • 児童同士の教え合いを促す - 「隣の人はどうしてる？」
  • 小さな成功を褒める - 「ここまでできたね！」
  • 論理的思考を言語化させる - 「どうしてそう考えたの？」

6. 低学年への特別な配慮

6.1 低学年特有の課題

課題 対応方法
文字入力が困難 音声入力、手書き入力の活用
予測変換機能の利用
定型文の準備
マウス操作が苦手 タッチ操作可能な機器の使用
大きなアイコン
ドラッグ&ドロップの練習
指示が理解できない 1つずつ順番に指示
視覚的な手順書
実演してから実施
集中力が続かない 活動を細かく区切る
達成感を味わえる工夫
楽しい要素を取り入れる

6.2 低学年向けの工夫

  • 視覚的な手順書
    • 写真やイラスト中心
    • 1ステップずつ大きく表示
    • 教室に掲示
  • 「できた！」マークの活用
    • 段階ごとに達成感を味わえる
    • シールや花丸で可視化
  • お助けカードの準備
    • よくある質問への回答を図解
    • 児童が自分で参照できる

6.3 保護者への配慮

低学年では、保護者の理解と協力が特に重要です。

  • ICT活用の目的を丁寧に説明
  • 家庭でのルール作りをサポート
  • 授業での様子を写真付きで報告
  • 健康面への配慮（視力、姿勢）を説明

演習1：事前打ち合わせシミュレーション

以下の状況で、担任教諭にどのような質問をしますか？5つ挙げてください。

状況：3年生 国語「物語を読んで、感想文を書こう」の授業で、タブレットを使いたいとの相談を受けた。

演習2：トラブル対応の判断

以下のトラブルが発生しました。優先順位をつけて、どのように対応しますか？

  • A児：「先生、タブレットが動かない！」（画面がフリーズ）
  • B児：「Wi-Fiがつながらない」（他の児童は問題なし）
  • C児：「さっき作ったのが消えちゃった」（保存忘れ）
  • 担任：提示装置に画面が映らない（授業開始5分前）

演習3：支援計画の立案

  • 学年：小学2年生
  • 教科：生活科
  • 単元：「町たんけん」
  • 活動：タブレットで写真を撮影し、発表資料を作成する
  • ICTスキル：初めてのタブレット使用

📌 まとめ

  • 小学校は学級担任制であり、担任教諭との信頼関係が支援の基盤
  • 低・中・高学年で発達段階が異なり、それぞれに応じた支援が必要
  • 授業前の綿密な準備が、スムーズな授業実施の鍵
  • 授業中は、児童の困りごとを早期に発見し、適切に支援する
  • 授業後の振り返りと記録が、次回の改善につながる
  • プログラミング教育では、試行錯誤を促し、論理的思考を育てる支援が重要
  • 低学年には、視覚的な支援と細かなステップ分けが効果的
